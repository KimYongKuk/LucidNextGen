## 5. 등록된 Windows 서비스

| 서비스명 | 도구 | 용도 | 실행 계정 | 시작 방식 |
|----------|------|------|-----------|-----------|
| LFChatbot-nginx | NSSM | nginx 리버스 프록시 | **landfop** | 자동 (부팅) |
| LFChatbot-backend-blue | NSSM | 운영 백엔드 :8001 | SYSTEM | 자동 (부팅) |
| LFChatbot-backend-green | NSSM | 운영 백엔드 :8002 | SYSTEM | 수동 (배포 시) |
| LFChatbot-backend-dev | NSSM | 개발 백엔드 :8099 (--reload) | SYSTEM | 자동 (부팅) |
| LFChatbot-pm2 | NSSM | PM2 프로세스 복원 (frontend-blue/green) | SYSTEM | 자동 (부팅) |

> **nginx는 landfop 계정으로 실행** — `nginx -s reload` 시그널이 동작하려면 서비스와 같은 계정이어야 함. SYSTEM 계정이면 Access Denied 발생.

Task Scheduler:

| 작업명 | 주기 | 동작 | 상태 |
|--------|------|------|------|
| LFChatbot-AutoDeploy | 1분마다 | main 변경 감지 → deploy.bat | **비활성화** (수동 배포 운용 중) |

## 6. 배포 과정 (deploy.bat v3)

```
[Lock]     .deploy.lock 생성 (동시 실행 방지)
              ↓
[Step 1]   IDLE 슬롯에서 git fetch + reset --hard (최신 코드)
              ↓
[Step 2]   requirements.txt 변경 시만 pip install (변경 없으면 스킵)
              ↓
[Step 3]   pnpm-lock.yaml 변경 시만 pnpm install (변경 없으면 스킵)
           → pnpm build (항상)
              ↓
[Step 4]   IDLE 백엔드 stop (잔여 프로세스 정리) → start
              ↓
[Step 5]   /health 체크 (2초 간격, 최대 30초)
           → 실패 시 IDLE 백엔드 stop, lock 해제, 중단
              ↓
[Step 6]   PM2로 IDLE 프론트엔드 start
              ↓
[Step 7]   nginx config 변경 → nginx -t 검증 → nginx -s reload
           → 무중단 전환 (기존 연결 유지, 새 연결만 IDLE로)
              ↓
[Step 8]   state.txt 업데이트
              ↓
[Step 9]   10초 대기 (connection drain) → OLD 슬롯 정지
              ↓
[Lock]     .deploy.lock 삭제
```

에러 발생 시 → lock 해제 후 중단, 기존 슬롯 서비스 유지.

### deploy.bat 주요 설계

| 항목 | 설명 |
|------|------|
| Lock 파일 | `.deploy.lock`으로 동시 배포 방지 |
| deps 스킵 | requirements.txt / pnpm-lock.yaml 변경 없으면 설치 생략 |
| `call pm2` | Windows에서 pm2.cmd 호출 시 `call` 필수 (없으면 bat 종료됨) |
| `nginx -s reload` | graceful reload, 무중단. nginx 서비스가 **landfop 계정**이어야 동작 |
| `ping -n N` | sleep 대체. `timeout.exe`는 비대화형(Task Scheduler)에서 에러 발생 |

## 7. 롤백 과정 (rollback.bat)

```
1. 이전 슬롯 시작 (코드가 그대로 남아있음, 빌드 불필요)
2. Health check 대기
3. nginx 전환
4. 현재 (문제) 슬롯 정지
→ 15초 이내 완료
```

## 10. 주의사항

### .env.local은 슬롯별로 다르다
```
개발:  NEXT_PUBLIC_API_URL=http://192.168.90.30:8099, PORT=3099
Blue:  NEXT_PUBLIC_API_URL=http://127.0.0.1:8001
Green: NEXT_PUBLIC_API_URL=http://127.0.0.1:8002
```
deploy.bat은 git reset --hard를 사용하지만 .env.local은 .gitignore에 포함되어 보존됨.

### nginx 라우팅 규칙
- `/api/v1/*` → 백엔드 (FastAPI)
- `/api/auth/*` → 백엔드 (인증)
- `/api/history`, `/api/vote` 등 → 프론트엔드 (Next.js API route)
- `/health` → 백엔드
- 나머지 → 프론트엔드

### nginx 서비스 계정
- **반드시 landfop 계정**으로 실행해야 `nginx -s reload`가 동작
- SYSTEM 계정이면 `Access is denied` 에러 발생
- services.msc → LFChatbot-nginx → 로그온 탭에서 변경 가능

### 공유 데이터
ChromaDB, 업로드 파일, 생성 파일은 `C:\Services\LFChatbot_data\`에 1벌만 존재.
Blue/Green 양쪽에서 Junction으로 동일 경로를 참조.
Blue-Green 전환 시 overlap 동안 두 인스턴스가 동시 접근 가능하지만,
SQLite WAL 모드에서 읽기는 안전하고 쓰기는 직렬화됨.

### 개발 프론트엔드
- PM2에서 제거됨 (운영과 분리)
- 파워쉘에서 수동 실행: `npx next dev --turbo -p 3099`
- 백엔드는 NSSM 서비스(LFChatbot-backend-dev)로 자동 실행

### 서버 재부팅 시
모두 자동 시작됨:
- nginx, backend-blue, backend-dev (NSSM)
- frontend-blue (PM2 → LFChatbot-pm2 서비스가 복원)
- Green은 꺼진 상태 (배포 시에만 사용)
- 개발 프론트엔드는 수동 실행 필요

### Nightly Summary
- **dev에서만 실행** (prod는 `NIGHTLY_SUMMARY_ENABLED=false`)
- 매일 23:00 실행 → `docs/summary/` 생성 → main 브랜치에 git commit/push
- `git push origin main` (GitLab) + `git push github main` (GitHub 미러)
- 메일 발송은 dev 백엔드에서 처리

### MCP 서버 (tavily-mcp)
- 글로벌 설치: `npm install -g tavily-mcp`
- mcp_config.json에서 전체 경로 사용: `C:\Users\Administrator\AppData\Roaming\npm\tavily-mcp.cmd`
- npx로 매번 다운받는 방식은 불안정 (네트워크/타임아웃 실패)

## 11. 문제 해결

### 운영 사이트 접속 안 됨 (Bad Gateway)
```powershell
# 1. 활성 슬롯 확인
type C:\Services\LFChatbot_prod\deploy\state.txt

# 2. 해당 슬롯 백엔드 확인
Restart-Service LFChatbot-backend-blue   # (또는 green)

# 3. 해당 슬롯 프론트엔드 확인
pm2 list
pm2 restart frontend-blue               # (또는 green)

# 4. nginx 확인
Restart-Service LFChatbot-nginx

# 5. 그래도 안 되면 롤백
C:\Services\LFChatbot_prod\deploy\rollback.bat
```

### 배포가 Step 6에서 멈춤
deploy.bat에서 `pm2`를 `call pm2`로 호출해야 함. (`pm2.cmd`는 배치파일이라 `call` 없으면 원래 bat이 종료됨)

### deploy.bat 즉시 종료됨
`.deploy.lock` 파일이 남아있을 수 있음:
```powershell
del C:\Services\LFChatbot_prod\deploy\.deploy.lock
```

### nginx -s reload가 Access Denied
nginx 서비스가 SYSTEM 계정으로 실행 중. services.msc에서 landfop 계정으로 변경 후 서비스 재시작.

### 배포 실패
```bash
# 로그 확인
type C:\Services\logs\deploy.log

# 빌드 에러면: IDLE 슬롯에서 직접 확인
cd C:\Services\LFChatbot_prod\green\frontend
pnpm build
```

### tavily 웹검색 안 됨
```powershell
# tavily-mcp 글로벌 설치 확인
tavily-mcp --version

# 없으면 설치
npm install -g tavily-mcp

# 백엔드 재시작
Restart-Service LFChatbot-backend-blue
```
