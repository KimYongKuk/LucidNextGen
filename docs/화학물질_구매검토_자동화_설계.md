# 화학물질 구매 검토 프로세스 자동화 설계

> 작성일: 2026-04-07
> 상태: 설계 완료, 개발 착수 전

## 1. 현행 프로세스 (AS-IS)

```
1. (발주부서→환경팀) 화학물질 구매 검토 신청서 전자결재 상신
2. (환경팀) Works 화학물질 검토에 물질 수동 등록
3. (환경팀 & 안전보건팀) 화학물질 검토 실시
4. (환경팀) 검토 결과를 기반으로 구매 검토 결과서 전자결재 수동 상신
```

**병목**: 2번(수동 등록)과 4번(수동 상신)에서 반복 작업 발생

## 2. 개선 프로세스 (TO-BE)

```
1. (발주부서) 화학물질 구매 검토 신청서 전자결재 상신
2. (자동) 결재 감지 → MSDS 첨부파일 다운로드 → AI 파싱 → Outline 위키에 검토 문서 자동 생성
3. (환경팀 & 안전보건팀) Outline에서 검토 실시 (댓글, 상태 변경)
4. (자동) 검토 완료 시 → 전자결재 기안 API로 검토 결과서 자동 상신
```

## 3. 기술 아키텍처

```
전자결재 상신 (발주부서)
    ↓
① 결재 감지 — v_appr_* VIEW 폴링 (신규 화학물질 검토 신청서 감지)
    ↓
② 첨부파일 메타 조회 — v_appr_doc_attachments VIEW
   (document_id → attach_id, file_name, extension, file_path)
    ↓
③ 첨부파일 다운로드 (복호화)
   GET /api/approval/document/{document_id}/download/{attach_id}
   Cookie: GOSSOcookie=...
    → 복호화된 MSDS 원본 파일 (PDF/Excel/PPT)
    ↓
④ AI 파싱 — MSDS에서 물질정보 구조화 추출
   (물질명, CAS번호, 구성성분, 함유량, H-code, 규제정보 등)
    ↓
⑤ Outline 위키 문서 자동 생성
   - Outline API: documents.create + attachments.create
   - 구조화된 물질정보 본문 + MSDS 원본 파일 첨부
    ↓
⑥ 환경팀 & 안전보건팀 검토
   - Outline 자체 댓글 기능으로 검토 의견 기록
   - 상태 관리: 문서 제목 태그 ([접수] → [환경완료] → [검토완료])
     또는 컬렉션 이동 (접수대기/ → 검토중/ → 검토완료/)
    ↓
⑦ 검토 완료 시 → 전자결재 기안 API 호출
   POST https://api.daouoffice.com/public/v4/approval/document
   - formCode: 화학물질 구매 검토 결과서 양식 코드
   - content: 검토 결과 HTML (AI가 Outline 검토 내용 기반 자동 생성)
   - attaches: MSDS 원본 파일 (MultipartFile)
   - draftEmpNo: 환경팀 담당자 사번
   - callbackUrl: 결재 처리 상태 추적 URL
```

## 4. 사용 API 목록

### 4.1 전자결재 첨부파일 다운로드 (내부 API)

```
GET https://lfon.landf.co.kr/api/approval/document/{document_id}/download/{attach_id}
인증: GOSSOcookie (SSO 세션)
응답: 복호화된 파일 바이너리
```

- 파일은 디스크에 암호화 저장됨 (`/data/TerraceTims/attach/...`)
- 직접 파일 읽기 불가 → 반드시 이 API를 통해 복호화된 파일 획득

### 4.2 Outline 위키 API

```
POST /api/documents.create     — 검토 문서 생성
POST /api/attachments.create   — MSDS 파일 첨부
POST /api/documents.update     — 상태/내용 업데이트
```

### 4.3 전자결재 기안 API (공개 API)

```
POST https://api.daouoffice.com/public/v4/approval/document
Content-Type: multipart/form-data

파라미터:
- clientId (String, 필수): 서비스 인증 ID
- clientSecret (String, 필수): 서비스 보안 KEY
- formCode (String, 필수): 전자결재 연동 서식 코드
- title (String, 필수): 결재 제목
- draftEmpNo (String): 기안자 사번
- content (String, 필수): 본문 HTML
- attaches (MultipartFile[]): 첨부파일
- callbackUrl (String, 필수): 처리 상태 콜백 URL
- partnerDocId (String): 외부 시스템 문서 고유 ID

제약: 월 600회, 첨부 5MB/건
```

### 4.4 Works 연동 API (공개 API) — 선택적

```
POST https://api.daouoffice.com/public/v1/works
Content-Type: application/json

파라미터:
- clientId, clientSecret, token (필수)
- params: { key: value } 형태, key는 Works 관리 > 외부 데이터 가져오기에서 설정한 코드

지원 컴포넌트: 텍스트, 멀티텍스트, 숫자, 드롭박스, 체크박스, 단일선택, 리스트박스, 날짜, 시간
파일 첨부: 지원 안 함 (텍스트 데이터만)
제약: 월 1,500회
```

### 4.5 Works 내부 API (비공식) — 파일 첨부 필요 시

```
# Step 1: 파일 업로드
POST https://lfon.landf.co.kr/api/file?GOSSOcookie={세션}
Content-Type: multipart/form-data
Body: file={파일}
응답: { hostId, filePath, fileName, fileExt, fileSize }

# Step 2: 문서 저장 (파일 참조 포함)
POST/PUT https://lfon.landf.co.kr/api/works/applets/{appletId}/docs[/{docId}]
Content-Type: application/json
Body: { appletId, values: { "{컴포넌트ID}": [{ id:null, path, name, hostId }] } }
인증: GOSSOcookie
```

- 컴포넌트 ID는 자동생성 (`_v8h3l1nnj` 등), 사용자 정의 코드 무시
- 비공식 API — Works 업데이트 시 변경 가능성 있음

## 5. DB VIEW 요구사항

### 5.1 v_appr_doc_attachments (DBA 생성 필요)

```sql
CREATE VIEW v_appr_doc_attachments AS
SELECT
    da.document_id,
    af.id AS attach_id,
    af.name AS file_name,
    af.extension,
    af.path AS file_path,
    af.length AS file_size
FROM go_appr_doc_attaches da
JOIN go_attach_files af ON da.attach_id = af.id
WHERE af.deleted_at IS NULL;

GRANT SELECT ON v_appr_doc_attachments TO ai_reader;
```

### 5.2 관련 테이블 구조 (참고)

```
go_appr_doc_attaches (junction)
    ├── document_id (bigint) — 결재문서 ID
    └── attach_id (bigint) — 첨부 ID

go_attaches (마스터)
    └── id, status, created_at, updated_at

go_attach_files (파일 메타, go_attaches.id와 1:1)
    ├── id (bigint)
    ├── name (varchar) — 원본 파일명
    ├── extension (varchar)
    ├── path (varchar) — 파일시스템 경로 (암호화 저장)
    ├── host_id (varchar) — 스토리지 호스트 식별자 (예: TMUzAwMDAwMEEtOTEw)
    ├── length (bigint) — 바이트 크기
    └── deleted_at (datetime)
```

파일 저장 경로 패턴: `/data/TerraceTims/attach/{company_id}/{module}/{subdir}/{UUID}`
- module: `docs`(결재), `works`(WORKS), `post`(게시판)
- 확장자 없이 UUID 파일명으로 저장, 다운로드 시 `go_attach_files.name`으로 매핑

## 7. 설계 결정 사항

### DB INSERT 정책 준수
- 그룹웨어 DB에 직접 INSERT/UPDATE **하지 않음**
- 모든 연동은 **공식 API** 또는 **READ-ONLY VIEW** 통해 수행
- 패키지 유지보수 리스크 최소화

### WORKS 앱 대체 → Outline 위키
- WORKS 화학물질 검토 앱의 파일 첨부 연동이 공식 API로 불가
- Outline 위키로 검토 프로세스 대체
  - 장점: 공식 API, 파일 첨부 가능, 시멘틱 검색 연동 (기존), 이력 관리
  - 단점: 상태 관리가 WORKS만큼 체계적이지 않음 (태그/컬렉션으로 우회)

### 파일 암호화 대응
- 그룹웨어 첨부파일은 디스크에 암호화 저장
- `/data/TerraceTims/attach/` 직접 읽기 불가
- 반드시 내부 다운로드 API 통해 복호화된 파일 획득

### Works 공개 API vs 내부 API
| 구분 | 공개 API (api.daouoffice.com) | 내부 API (lfon.landf.co.kr) |
|------|------|------|
| 인증 | clientId + clientSecret + token | GOSSOcookie |
| 파일 첨부 | **X** (텍스트만) | **O** |
| 월 호출 제한 | 1,500회 | 없음 |
| 안정성 | 공식, 버전 보장 | 비공식, 변경 가능 |
| 컴포넌트 키 | 사용자 정의 코드 (`msds_test`) | 자동생성 ID (`_v8h3l1nnj`) |

## 8. 향후 확장 가능성

- **과거 검토 이력 검색**: Outline 시멘틱 검색으로 "이전에 톨루엔 검토한 적 있나?" 즉시 조회
- **규제 자동 체크**: MSDS 파싱 시 유해물질 DB와 자동 대조
- **다른 부서 확장**: 동일 패턴으로 다른 전자결재→검토 프로세스에 적용 가능
