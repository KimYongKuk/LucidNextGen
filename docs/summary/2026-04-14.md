# 일일 개발 보고서 — 2026-04-14

## 오늘의 요약
그룹웨어 서비스 메뉴 플로팅 위젯과 전용 embed 모드를 추가하여 SSO 연동 서비스에 대한 접근성을 개선했습니다. 동시에 Worker LLM 호출의 throttling 폴백 로직을 전면 개편하고, 캘린더 API 호출 시 사용자별 SSO 인증을 적용하여 권한 오류를 해결했습니다. 또한 공유 도구(PDF/DOCX/차트)를 BaseWorker 기본값으로 승격하여 모든 워커에서 문서 생성이 가능하도록 개선했습니다.

## 주요 변경사항

### [추가] [Widget] 그룹웨어 서비스 메뉴 플로팅 위젯
- **기능**: SSO 연동 서비스(e-HR, EHS, wa, Lucid, Pilot, Hub 등)에 직접 접근 가능한 플로팅 메뉴 위젯 구현
- **특징**: 
  - 사번 자동 판별 → 소속 조직(엘앤에프/엘앤에프플러스) 자동 조회
  - 조직별 메뉴 필터링 (7개 초기 메뉴 데이터)
  - 새 탭 SSO 이동 (window.open)
- **DB**: MySQL `service_menu` 테이블 신규 생성 (메뉴 추가/삭제 시 코드 배포 불필요)
- **백엔드**: `GET /api/v1/service-menu` API 추가 (사번→회사→메뉴 필터링)
- **프로세스 캐싱**: 직원 소속 회사는 거의 변경되지 않으므로 캐싱 적용

### [수정] [Fallback] Worker LLM 호출 us↔global inference profile 자동 전환
- **문제**: ThrottlingException 발생 시 catch 없이 전파 → orchestrator_producer 미처리 예외 종료
- **해결**: Inference Profile prefix 양방향 전환 (`us.anthropic.*` ↔ `global.anthropic.*`)
  - 별도 일일 토큰 한도 사용으로 throttling 회피
  - 같은 리전(us-east-1)에서 prefix만 변경하므로 리전 전환 불필요
- **적용 범위**: 모든 Bedrock 호출 경로 (Worker stream, IntentClassifier, bedrock_service 전체)
- **자동 복구**: 다음 날 KST 09:00 (UTC 00:00) 자동 복구 (AWS 일일 한도 리셋 시점 일치)

### [추가] [Embed/GW] 그룹웨어 전용 embed 모드
- **구조**: iframe 방식으로 Next.js 채팅 UI 재사용 (위키 embed와 동일 패턴)
- **인텐트 필터링**: groupware_embed 모드에서 그룹웨어 관련 에이전트만 허용 (MAIL, APPROVAL, BOARD, CALENDAR, RESERVATION, IT_SUPPORT, ACCT_SUPPORT, WEB_SEARCH, DIRECT)
- **사번 추출**: GO.session().employeeNumber 클라이언트 추출 (JSP 변수 및 DB 조회 방식 모두 불가)
- **iframe lazy-load**: 플로팅 버튼 클릭 시 iframe 생성 (GO.session() 초기화 보장)
- **userId 안정화**: useState initializer로 마운트 시 1회만 URL에서 추출 (useRef/useCallback 조합 실패)
- **nginx CSP**: `/embed/` 경로에 frame-ancestors 헤더 추가 (다우오피스 iframe 허용)

### [수정] [Calendar] 캘린더 사용자별 SSO 인증
- **문제**: 모든 사용자의 캘린더 API 호출이 서비스 계정(wg0403)으로 이루어져 타 사용자 캘린더 접근 시 권한 오류 발생
- **해결**: 사용자 본인의 LFON GOSSOcookie를 사용하도록 전환
  - 사용자 쿠키 우선 사용 → 401/403 시 서비스 계정 폴백
  - JSP에서 GOSSOcookie 추출 → AES 암호화 → URL 파라미터 전달
  - middleware에서 복호화 → 쿠키 저장 → 백엔드로 전달
- **데이터 흐름**: RequestContext에 gosso_cookie 필드 추가 → CalendarWorker에서 도구 args에 주입 → calendar_mcp_server에서 사용
- **JSP 배포**: 별도 가이드 문서(`docs/jsp_sso_gosso_patch.md`) 제공 (LFON 서버 관리자 배포 필요)

### [수정] [BaseWorker] 공유 도구(PDF/DOCX/차트) 기본값 승격
- **배경**: VisualizationWorker 제거 후 4개 워커만 override → 나머지 워커에서 문서 생성 불가
- **개선**: opt-in → opt-out 방식으로 전환
  - BaseWorker 기본값에 공유 도구 포함 (create_line_chart, create_bar_chart, create_pie_chart, create_multi_chart, create_document_pdf, create_table_spec_pdf, create_document_docx)
  - 4개 워커의 중복 override 제거
- **시스템 프롬프트**: DirectResponseWorker, XlsxWorker에 문서/차트 생성 도구 사용 지침 명시

## 기술적 세부사항

### Inference Profile 폴백 메커니즘
```
Worker stream_response()
  └─ agent.astream_events() 호출
      └─ ThrottlingException 발생
          └─ is_throttling_error() 감지
              └─ swap_inference_prefix(us.* ↔ global.*)
                  └─ activate_fallback() (이후 요청도 자동 전환)
                      └─ agent 재생성 + 재시도
```

**핵심 결정**: on-demand(`anthropic.*`) 직접 호출은 sonnet-4-6에서 ValidationException 발생 → inference profile 필수. EU/APAC 프로필은 계정에 미활성화 → us/global만 사용 가능.
