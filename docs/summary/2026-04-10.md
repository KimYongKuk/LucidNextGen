# 일일 개발 보고서 — 2026-04-10

## 오늘의 요약
이미지 공유 후 후속 질문 시 AI가 이미지 맥락을 잃어버리는 문제를 해결했습니다. 프론트엔드에서 메시지 히스토리에 `[이미지 첨부됨]` 힌트 태그를 추가하고, 백엔드 시스템 프롬프트에 이미지 맥락 유지 규칙을 적용하여 AI가 이전 분석 결과를 활용하도록 개선했습니다.

## 주요 변경사항

### [수정] Frontend — 메시지 히스토리 이미지 태그 추가
**파일**: `frontend/hooks/use-simple-chat.ts`
- 이미지가 포함된 사용자 메시지에 `[이미지 첨부됨]` 텍스트 태그 자동 삽입
- 메시지 히스토리 전송 시 AI가 이미지 존재 여부를 명확히 인식하도록 개선

### [수정] Backend — 시스템 프롬프트 이미지 맥락 유지 규칙 추가
**파일**: `backend/app/agents/workers/base_worker.py`
- `[이미지 첨부됨]` 태그 인식 로직 추가
- 이전 분석 결과 기반 후속 질문 답변 가이드 추가
- "이미지가 없습니다" 거부 응답 방지 규칙 적용

## 기술적 세부사항

### 문제 분석
- **증상**: 사용자가 이미지 공유 후 "다시 분석해줘", "좀 더 자세히" 등 후속 질문 시 AI가 "이미지를 받은 적 없습니다"로 응답
- **근본 원인**: 
  - 메시지 히스토리에 이미지 base64가 포함되지 않음 (토큰 폭발 방지)
  - AI가 이전 분석 텍스트는 인식하나, "현재 이미지 부재"에 과도하게 집중
  - 시스템 프롬프트에 이미지 맥락 유지 가이드 부재

### 해결 방식
**텍스트 태그 기반 접근** (base64 포함 대신)
```javascript
// 프론트엔드: 이미지 존재 여부를 텍스트로 표시
const hasImages = msg.role === 'user' && msg.parts
  ?.some((part) => part.type === 'file' && part.mediaType?.startsWith('image/'));
const content = hasImages ? `[이미지 첨부됨]\n${textContent}` : textContent;
```

**백엔드**: 시스템 프롬프트에 이미지 맥락 유지 규칙 통합
- `[이미지 첨부됨]` 태그 감지 시 이전 분석 결과 활용
- 거부 응답 대신 누적된 분석 맥락 기반 답변

### 설계 결정
- **토큰 효율성**: base64 재포함 대신 텍스트 태그 방식으로 비용 최소화
- **기능 제약**: 새로운 관점의 분석은 원본 이미지 재첨부 필요 (의도된 동작)
- **안정성**: stale closure 이슈 없음 확인 — 콜백 참조 구조 정상
