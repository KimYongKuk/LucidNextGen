# 일일 개발 보고서 — 2026-03-24

## 오늘의 요약
OutlineWorker의 파일→위키 문서 생성 기능을 완성하고, HANDOFF 마커 감지 실패 및 파일 참조 인텐트 오분류 버그를 수정했습니다. 추가로 워크스페이스 인스트럭션 기반 인텐트 분류를 개선하고 내부 DB 스키마 노출 방지 가드레일을 추가했습니다.

## 주요 변경사항

### 1. [추가] OutlineWorker, MCP — 파일→위키 문서 생성 기능
- **기능**: PDF/PPTX/DOCX 업로드 파일에서 텍스트+이미지 자동 추출 → Outline Wiki 문서 자동 게시
- **신규 모듈**: `file_extractor.py` — 파일 형식별 파싱 및 마크다운 변환
- **신규 MCP 도구 3개**:
  - `extract_file_for_wiki`: 파일 파싱 → 마크다운 + 이미지 스테이징
  - `upload_image_to_outline`: 이미지 → Outline API 업로드 → URL 반환
  - `create_wiki_document`: 문서 생성 API 호출
- **OutlineWorker 확장**: `max_agent_steps` 24→40, 쓰기 도구 3개 추가, 시스템 프롬프트에 DOCUMENT CREATION WORKFLOW 섹션 추가
- **이미지 처리**: 원본 모드(빠른 처리) / 정제 모드(Vision API로 이미지 설명 생성) 지원

### 2. [수정] Orchestrator, IntentClassifier — HANDOFF 마커 감지 실패 + 파일 참조 인텐트 오분류
- **버그 1 (HANDOFF 감지 실패)**: `_extract_text()`가 list content 미처리
  - Bedrock Converse API에서 반환하는 `[{"type": "text", "text": "..."}]` 형태 content 처리 로직 추가
  - HANDOFF/NO_RESULTS 마커 감지 이제 정상 작동
- **버그 2 (파일 참조 오분류)**: "업로드한 파일을 outline 위키용 markdown으로 변환" → outline 키워드로 OutlineWorker 라우팅
  - IntentClassifier Step 1에 업로드 파일 명시 참조 패턴 추가: `(업로드한 파일|올린 파일|첨부 파일|이 파일|그 파일|방금 파일)` + `has_files=True` → `USER_FILES` 우선 분류

### 3. [수정] IntentClassifier — 워크스페이스 인스트럭션 기반 인텐트 분류 개선
- **개선 사항**: Classifier가 workspace_instructions 컨텍스트를 인텐트 분류에 반영
- **구현**: 
  - CLASSIFIER_PROMPT에 `workspace_instructions` 필드 추가
  - WORKSPACE RULES 섹션 확장 — 인스트럭션 기반 라우팅 가이드라인 추가
  - `classify()` 메서드에서 인스트럭션 앞 500자만 프롬프트에 전달 (토큰 효율성)
- **효과**: "전자결재 조회용 워크스페이스"에서 "자주보전 활동 정리해줘" 요청 시 올바른 Worker로 라우팅

### 4. [수정] BaseWorker — 내부 DB 스키마 노출 방지 가드레일
- **추가 사항**: `build_system_prompt()` 마지막에 가드레일 문구 추가
  - "응답에 내부 DB 뷰 이름, 컬럼명, SQL 쿼리, 테이블 구조를 절대 포함하지 마세요"
- **배경**: 사용자가 워크스페이스 인스트럭션 생성 요청 시 LLM이 시스템 프롬프트의 스키마 정보를 응답에 포함하는 문제 방지

## 기술적 세부사항

### 파일 추출 아키텍처 (`file_extractor.py`)
- **PDF**: PyMuPDF `get_text("dict")`로 블록 순서 보존, 폰트 크기 기반 헤딩 감지, `extract_image(xref)`로 이미지 바이너리 추출
- **PPTX**: shape 재귀 순회 → 슬라이드별 마크다운, `shape.image.blob`으로 이미지 추출, 표→마크다운 테이블
- **DOCX**: `paragraph.style.name` 기반 헤딩, `inline_shape` → relationship → blob으로 이미지 추출, `doc.tables` → 마크다운 표
- **이미지 필터링**: 최소 5KB, 최대 15개/문서
- **스테이징**: `backend/data/outline_staging/{uuid}/`에 임시 저장, 1시간 후 자동 정리
- **Vision 처리**: Bedrock Claude Haiku로 이미지 설명 생성 (정제 모드에서만)

### 대화형 워크플로우
```
사용자: "이 파일 위키에 올려줘" + 파일 업로드
  → LLM: list_collections 호출 → 컬렉션 목록 제시
사용자: "2번" (컬렉션 선택)
  → LLM: extract_file_for_wiki → upload_image_to_outline (N회) → create_wiki_document
  → LLM: "위키에 문서를 생성했습니다. [바로가기](링크)"
```

### 인텐트 분류 개선 사항
- **기존**: workspace_name, workspace_description만 참고 → instructions는 Worker 단계에서 주입
- **개선**: Classifier 단계에서 instructions 앞 500자를 컨텍스트로 포함 → 워크스페이스 목적에 맞는 전문 Worker 라우팅
- **토큰 효율**: 평균 인스트럭션 2,124자 중 앞 500자만 사용 → ~170토큰 추가 비용으로 분류 정확도 향상
