## 주요 변경사항

### [수정] Response — Worker 마커 텍스트 노출 버그 근본 해결
- **문제**: `<!--WORKER:name-->` HTML 주석이 간헐적으로 채팅 UI에 텍스트로 노출
- **해결**: 텍스트 콘텐츠 삽입 방식 → 메시지 객체 필드 분리
  - `ChatMessage` 타입에 `workerName?: string` 필드 추가
  - `use-simple-chat.ts`에서 텍스트 마커 제거, 메시지 객체에 직접 저장
  - `response.tsx`에서 prop으로 수신하여 렌더링 누출 완전 차단
  - 레거시 마커도 fallback으로 처리하여 하위 호환성 유지

### [수정] Intent — quick_classify 구조 개선 + 프롬프트 슬림화
- **구조 개선**: Pairwise 충돌 체크 → Scan-all 패턴으로 리팩토링
  - Step 1: 100% 확실한 케이스 (URL, 명시적 액션)
  - Step 2: 도메인 인텐트 키워드 스캔 (mail, approval, board, xlsx)
  - Step 3: 2개 이상 매칭 시 LLM 위임 (오분류 방지)
  - Step 4: Web search fallback (도메인 미매칭 시만)
- **프롬프트 슬림화**: ~180줄 → ~100줄 (quick_classify 처리 예시 제거)
  - 단순 메일/결재/게시판/web_search 예시 27개 제거
  - 뻔한 disambiguation 규칙 통합
  - LLM 전용 인텐트(PPT, corp_rag) 예시만 유지
- **효과**: "전자결재 관련 게시글 찾아줘" 오분류 수정 (APPROVAL → BOARD)

### [수정] Mail — 브리핑 팝업 읽지 않은 메일 하위 폴더 포함

### [수정] Notification — 알림 모달 로딩 경험 혁신
- **즉시 모달 오픈**: 데이터 기다리지 않고 로딩 UI 먼저 표시
- **타이핑 애니메이션**: "Today's Briefing" 제목이 한 글자씩 등장 (50ms/글자)
- **병렬 데이터 로딩**: `/fast` (공지+결재) + `/mail` 엔드포인트 신설, `Promise.all`로 동시 수행
- **전체 로딩 후 표시**: 모든 데이터 준비 완료 시 한꺼번에 렌더링
- **건수 제한**: 모든 섹션 LIMIT 5 → 3으로 변경
- **섹션 순서**: 공지사항 → 메일 → 전자결재

- 실패 서버명 로그 출력으로 디버깅 용이성 향상

### [추가] NightlySummary — 일일 개발 요약 자동화 스케줄러
- **실행 시각**: 매일 23:00 KST (CronTrigger, timezone=Asia/Seoul)
- **처리 흐름**:
  1. CHANGELOG.md에서 당일 섹션 파싱
  2. 링크된 `docs/history/*.md` 파일 + 당일 날짜 파일 수집
  3. Bedrock Haiku로 종합 요약 보고서 생성 (max_tokens=2000)
  4. `docs/summary/YYYY-MM-DD.md` 저장
  5. git add → commit → push (실패해도 계속)
  6. Markdown → HTML 변환 후 이메일 발송
- **환경변수**: `NIGHTLY_SUMMARY_ENABLED`, `NIGHTLY_SUMMARY_HOUR`, `NIGHTLY_SUMMARY_RECIPIENT`
- **수동 실행 API**: `POST /nightly-summary/run-now` (테스트/특정 날짜 재생성)
- **기존 인프라 활용**: APScheduler, EmailService, BedrockService (새 의존성 없음)

## 기술적 세부사항

### Worker 마커 분리 아키텍처
메타데이터를 텍스트 콘텐츠에 혼합하는 방식의 근본적 문제를 해결:
- **이전**: HTML 주석 마커 → 렌더링 타이밍에 따라 누출 위험
- **이후**: 메시지 객체의 별도 필드 → 렌더링 파이프라인 완전 분리
- **하위 호환**: 기존 DB 메시지는 `workerName` 미포함 → 모든 아티팩트 패턴 활성화 (기존 동작 유지)

### Intent Classifier 스캔-올 패턴
기존 N² 확장 문제를 선형 구조로 개선:
- **Pairwise 예외 제거**: mail+approval, board+approval 등 쌍별 규칙 불필요
- **다중 매칭 처리**: 2개 이상 인텐트 동시 매칭 시 LLM 위임 (disambiguation)
- **Web search 분리**: 도메인 인텐트 미매칭 시만 처리 (false overlap 방지)
- **확장성**: 새 인텐트 추가 시 Step 2에 키워드만 추가하면 됨
