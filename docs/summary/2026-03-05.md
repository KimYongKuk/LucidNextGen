## 주요 변경사항

### [수정] [Intent] 후속 질문 인텐트 오분류 수정
- **문제**: 짧은 후속 질문(예: `"근무지"는?`)이 이전 대화 맥락을 무시하고 엉뚱한 인텐트로 분류
- **해결**: LLM 분류 시 최근 4개 메시지 대화 히스토리 전달, FOLLOW-UP 룰 추가로 맥락 기반 분류 지원
- **파일**: `backend/app/agents/intent_classifier.py`, `orchestrator.py`
- **상세**: 각 메시지 150자 truncate로 프롬프트 크기 제한, 짧은 후속 질문은 이전 턴 인텐트 유지

### [추가] [Branding] 커스텀 로고 적용
- **범위**: 사이드바 헤더, AI 응답 아이콘(로딩:SVG/완료:PNG), 브라우저 favicon
- **파일**: `frontend/public/logo.svg`, `logo.png`, `app-sidebar.tsx`, `message.tsx`, `layout.tsx`

### [수정] [Mail] 메일 검색 하이브리드 개선(v4)
- **문제**: 한국어 제목 매칭 불가, 하위폴더 메일 누락
- **해결**: 2단계 병합 검색 — SQL LIKE(전체 메일함 preview) + Java MIME(최근 1000건 제목), Inbox 하위폴더 포함
- **파일**: `backend/data/jsp/lucid_mail.jsp`
- **상세**: SQL injection 방지 위해 단일따옴표 이스케이프, 1차 SQL로 preview 본문 검색 후 2차 MIME으로 제목 보완

### [수정] [Intent] 차트/PPT 생성 요청이 web_search로 오분류
- **문제**: visualization/ppt 키워드가 quick_classify에서 미처리되어 web_search로 폴백
- **해결**: quick_classify에 visualization/ppt 키워드 패턴 추가, 생성 워커가 자체 tavily_search로 데이터 조사+생성 일괄 처리
- **파일**: `backend/app/agents/intent_classifier.py`

### [수정] [Response] Worker 마커 텍스트 노출 버그 수정
- **문제**: `<!--WORKER:name-->` HTML 주석이 간헐적으로 UI에 텍스트로 노출
- **해결**: 텍스트 콘텐츠 삽입 방식을 메시지 객체 필드(`workerName`)로 분리, 렌더링 누출 근본 해결
- **파일**: `frontend/lib/types.ts`, `use-simple-chat.ts`, `message.tsx`, `response.tsx`
- **상세**: 레거시 마커도 fallback으로 처리하여 하위 호환성 유지

### [수정] [Mail] 브리핑 팝업 안 읽은 메일 Inbox 하위폴더 포함
- **문제**: 메일 규칙으로 `Inbox.하위폴더`에 분류된 메일이 브리핑에 미표시
- **해결**: JSP unread 쿼리에 `Inbox.*` 하위폴더 포함, total_count 반환 추가
- **파일**: `backend/data/jsp/lucid_mail.jsp`

### [수정] [Intent] quick_classify 구조 개선 + 프롬프트 슬림화

- **파일**: `backend/app/agents/intent_classifier.py`
