## 주요 변경사항

### 🆕 [Desktop] Tauri v2 데스크톱 앱 신규 구축
**모듈**: `desktop/`

- **시스템 트레이 상주**: 백그라운드 실행, 트레이 메뉴로 열기/퀵 채팅/종료 제어
- **글로벌 단축키**: `Ctrl+Space`로 퀵 채팅 팝업 토글 (OS 레벨)
- **퀵 채팅 UI**: 680x480 alwaysOnTop 팝업, SSE 스트리밍 직접 호출
- **마크다운 렌더링**: `react-markdown` + `remark-gfm`으로 테이블, 코드블록, 리스트 지원
- **FOLLOW_UP 버튼**: HTML 주석 파싱(`<!--FOLLOW_UP:[...]-->`) → 클릭 가능한 pill 버튼으로 자동 질문 전송
- **풀 채팅 창**: 1200x800 리사이즈 가능, 기존 Next.js 웹 iframe 래핑

**주요 파일**:
- `desktop/src-tauri/src/lib.rs`: Rust 코어 (글로벌 단축키, 트레이, 창 관리)
- `desktop/src-tauri/tauri.conf.json`: 창 2개 설정, 트레이, 번들 설정
- `desktop/src/views/QuickChat.tsx`: 퀵 채팅 UI 컴포넌트
- `desktop/src/components/ChatMessage.tsx`: 마크다운 메시지 렌더링

### 🔧 [XlsxWorker] tavily_search 도구 결과 잘림 버그 수정
**모듈**: `backend/app/agents/workers/xlsx_worker.py`

**문제**: XlsxWorker의 `secured_ainvoke` 래핑이 전역 캐시된 `tavily_search`에 적용되어:
- 모든 웹검색 결과가 8,000자로 잘림
- ⚠️ 경고 메시지가 LLM에 "오류"로 해석됨
- 사용자에게 "검색 도구에 일시적인 오류가 발생했습니다" 응답

**해결**:
1. `prepare_tools()`에서 `tavily_search` 래핑 제외 (`SKIP_WRAPPING` frozenset)
2. 이미 래핑된 경우 `_unwrapped_ainvoke`로 원복 (서버 재시작 없이 즉시 적용)
3. `_truncate_tool_result`에서 tavily_search 원본 반환 (이중 안전장치)

**근본 원인**: MCP 도구 전역 캐시(TTL 1시간)의 in-place 수정으로 Worker 간 상호 영향

## 기술적 세부사항

### Tauri 데스크톱 앱 아키텍처
```
Tauri App (Windows, ~10MB)
├── Rust Core (tauri v2)
│   ├── 글로벌 단축키: Ctrl+Space → 퀵 채팅 토글
│   ├── 시스템 트레이: 메뉴 기반 제어
│   └── 창 관리: main(1200x800), quick(680x480, alwaysOnTop)
├── WebView (React + Vite)
│   ├── QuickChat: SSE 스트리밍, 마크다운 렌더링, FOLLOW_UP 버튼
│   └── FullChat: 기존 Next.js 웹 iframe
└── 백엔드: 기존 FastAPI 서버 (변경 없음)
```

### tavily_search 버그 수정 로직
```python
# SKIP_WRAPPING으로 외부 도구 제외
SKIP_WRAPPING = frozenset(["tavily_search", ...])

# 이미 래핑된 경우 원복
if hasattr(tool, "_unwrapped_ainvoke"):
    tool.ainvoke = tool._unwrapped_ainvoke

# 이중 안전장치: tavily_search는 truncation 스킵
if tool_name == "tavily_search":
    return result  # 원본 반환
```
