## 오늘의 요약
브리핑 모듈의 수신문서 접수대기 정확도를 개선하고, 시각화 기능을 대폭 확장했습니다. 전자결재 시스템과의 데이터 일치성을 높이는 한편, SVG 인포그래픽과 Mermaid 다이어그램을 추가하여 정성적·구조적 시각화 능력을 강화했습니다.

## 주요 변경사항

### [수정] Briefing — 수신문서 접수대기 정확도 개선
- **모듈**: `backend/app/services/notice_service.py`, `backend/metadata/MCP_GW_APPR.md`
- **주요 개선**:
  - `accessible_depts` JOIN을 통해 사용자의 모든 접근 가능 부서 조회 (소속 부서 + 부서 문서함 담당자 지정 부서)
  - `reception_status = 'WAITING'` 필터로 실제 접수 상태 반영 (레거시 `is_assigned` 제거)
  - `NULLS LAST` 정렬로 CANCEL/RETURN 문서가 최상단 표시되는 문제 해결
- **신규 VIEW**: `v_appr_user_accessible_depts` (부서 문서함 접근 권한 기반)
- **주의사항**: `reception_status` 값은 WAITING(접수대기), RECEIVED(접수완료), RECV_RETURNED(접수반려)

### [추가] VisualizationWorker — SVG 인포그래픽 + Mermaid 다이어그램
- **모듈**: `backend/app/agents/workers/visualization_worker.py`, `backend/app/mcp_servers/svg_generator/`
- **신규 기능**:
  - **SVG Visual**: LLM이 생성한 자유도 높은 SVG 인포그래픽/KPI 대시보드를 채팅 인라인으로 렌더링
  - **Mermaid Diagram**: 마크다운 ```mermaid 코드 블록을 자동 감지하여 플로우차트, 시퀀스, 간트, ER 다이어그램 렌더링
- **시각화 도구 3종 체계**:
  1. Charts (Recharts) — 정량 데이터 (추이, 비교, 비율)
  2. Mermaid — 구조화된 다이어그램
  3. SVG Visual — 커스텀 인포그래픽 (최후 수단)
- **보안**: 백엔드 regex 정제 + 프론트엔드 DOMPurify 이중 방어
- **프론트엔드 신규 컴포넌트**: `svg-display.tsx`, `mermaid-diagram.tsx` (확대/다운로드/코드복사 기능 포함)

### SVG Visual 아키텍처
- **생성**: VisualizationWorker → `create_svg_visual` MCP tool
- **정제**: MCP Server에서 regex 기반 보안 정제 (script/on*/javascript:/foreignObject 제거)
- **전송**: SSE `svg_visual` 이벤트로 스트리밍
- **렌더링**: 프론트엔드 DOMPurify SVG 프로파일로 이중 정제 후 인라인 렌더링

### Mermaid 통합
- 마크다운 코드 블록 방식으로 모든 Worker에서 자연스럽게 사용 가능
- `response.tsx`에서 `language-mermaid` 감지 → `MermaidDiagram` 컴포넌트 렌더링
- Mermaid.js v11.12.2 (neutral 테마, Malgun Gothic 폰트)
- 스트리밍 중에는 코드 블록으로 표시, 완료 후 렌더링
