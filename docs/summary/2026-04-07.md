# 일일 개발 보고서 — 2026-04-07

## 오늘의 요약
워크스페이스 인텐트 오버라이드를 제거하여 라우팅 구조를 개선했고, 파일 전문 전달 기능을 추가하여 전체 텍스트 기반 요약/번역이 가능해졌다. 또한 화학물질 구매 검토 프로세스 자동화 설계를 완료하여 전자결재-MSDS 파싱-위키 등록-검토-기안 API 상신까지의 엔드-투-엔드 자동화 방안을 수립했다.

## 주요 변경사항

### 1. [수정] IntentClassifier / DirectWorker — 워크스페이스 인텐트 오버라이드 제거
- **파일**: `backend/app/agents/intent_classifier.py`, `backend/app/agents/workers/direct_worker.py`
- **변경 내용**:
  - `intent_classifier.py` 381행의 `direct → user_files` 강제 전환 로직 제거
  - `DirectResponseWorker`에 `search_workspace_docs` 공유 도구 추가
  - 워크스페이스 컨텍스트는 `BaseWorker.build_system_prompt()`에서 모든 워커에 자동 주입
- **효과**: 인텐트 강제 변경 없이도 각 워커가 워크스페이스를 인식하며, "PDF 만들어줘" 같은 시각화 요청이 `DirectResponseWorker`에서 올바르게 처리됨

### 2. [추가] 설계 — 화학물질 구매 검토 프로세스 자동화
- **상태**: 설계 완료, 개발 착수 전
- **프로세스 흐름**:
  1. 전자결재 신청서 상신 감지 (v_appr_* VIEW 폴링)
  2. 첨부 MSDS 파일 다운로드 (복호화)
  3. AI 파싱으로 물질정보 구조화 추출
  4. Outline 위키에 검토 문서 자동 생성 (구조화 정보 + 원본 파일 첨부)
  5. 환경팀/안전보건팀이 Outline에서 검토 실시
  6. 검토 완료 시 전자결재 기안 API로 결과서 자동 상신
- **API 활용**: 
  - 내부 API: 첨부파일 다운로드 (복호화)
  - Outline 공식 API: 문서 생성/업데이트
  - 전자결재 공개 API: 기안 상신 (월 600회 제한)
- **DB 정책**: INSERT 0건, 공식 API 및 READ-ONLY VIEW만 사용
- **선행 작업**: VIEW 생성, clientId/clientSecret 발급, formCode 설정, 월간 건수 확인 등 6개 항목

### 3. [추가] UserFilesWorker — 파일 전문 전달 기능 (Fulltext)
- **파일**: `backend/app/services/chromadb_service.py`, `backend/app/mcp_servers/rag_server.py`, `backend/app/agents/workers/user_files_worker.py`
- **변경 내용**:
  - 파일 업로드 시 추출된 전체 텍스트를 `data/fulltext/{file_id}.txt`에 디스크 보관
  - `get_uploaded_file_content` MCP 도구 추가 (50,000자 제한)
  - 시스템 프롬프트에 전문형(요약/번역) vs 검색형(키워드) 분기 규칙 추가
- **효과**: 기존 청크 검색(~3,000자)으로 불가능했던 전문 요약/번역이 가능해짐 (약 33페이지 커버)
- **하위 호환성**: 기존 ChromaDB 검색 경로는 변경 없음, fulltext는 추가 경로

## 기술적 세부사항

### 워크스페이스 컨텍스트 주입 메커니즘
```
BaseWorker.build_system_prompt()
  ├─ workspace_instructions 추가
  ├─ workspace_memory 추가
  ├─ workspace_files 목록 추가
  └─ 검색 도구 안내 (has_files or workspace_has_files 조건)
     → 모든 워커가 워크스페이스 인식 가능
```
인텐트 오버라이드 제거로 각 워커의 역할이 명확해지고, 워크스페이스는 **환경**으로서 모든 워커에 투명하게 주입됨.

### 파일 전문 전달 아키텍처
```
기존: 파일 → 청킹 → 임베딩 → ChromaDB → 검색(3개 청크) → LLM
개선: 파일 → 전체 텍스트 → data/fulltext/ 저장 (병렬)
                        └→ 청킹 → 임베딩 → ChromaDB (기존 유지)

요청 시 분기:
  - 전문형 요청 → get_uploaded_file_content → fulltext 직접 전달
  - 검색형 요청 → search_user_files → ChromaDB 검색
```

### 화학물질 자동화 설계의 핵심 기술 결정
- **파일 암호화 대응**: 그룹웨어 첨부파일은 디스크에 암호화 저장되므로 내부 다운로드 API 필수
- **WORKS 대체 선택**: 공개 API의 파일 첨부 미지원으로 Outline 위키 선택 (공식 API, 파일 첨부 가능)
- **DB 정책 준수**: 그룹웨어 DB에 직접 INSERT/UPDATE 금지, 공식 API만 사용
