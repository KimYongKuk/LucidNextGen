## 주요 변경사항

### 🆕 [추가] VOC Wiki 자동 축적 시스템
- **모듈**: VOC Wiki Service
- **기능**: IT VOC(v_works_app_934_data) DB의 해결 사례를 매일 자동으로 L&F Wiki에 축적
- **동작**: 매일 06:00 KST에 CronTrigger로 실행, Haiku LLM이 신규 건을 기존 문서에 병합하여 중복 없이 지식 보강
- **구조**: 시스템/주제별 계층 구조로 자동 분류 및 목차 갱신
- **신규 파일**: `voc_wiki_service.py`, `voc_wiki_scheduler.py`, 마이그레이션 스크립트
- **주의**: 초기 적재(`run_initial_load(months=1)`) 및 컬렉션 ID 환경변수 설정 필요

### 🆕 [추가] OpenAI 호환 Chat Completions API
- **모듈**: OpenAPI Bedrock Service
- **기능**: 사내 서비스가 OpenAI SDK를 그대로 사용하여 Bedrock 기반 Claude 모델 호출 가능
- **엔드포인트**: `/v1/chat/completions`, `/v1/models`

- **지원**: 스트리밍/논스트리밍, Sonnet/Haiku 모델, 토큰 사용량 추적
- **신규 파일**: `openapi_compat.py`, `openapi_bedrock_service.py`, `api_key_auth.py`
- **환경변수**: `OPENAPI_AWS_*`, `OPENAPI_KEYS`, 모델 매핑 설정 필요

### 🔧 [수정] 대화 요약 임계치 상향 + 프롬프트 강화
- **모듈**: BaseWorker
- **문제**: 옵션 A/B/C 같은 구조화된 선택지가 조기 요약으로 소실
- **변경**:
  - 임계치 상향: 6메시지/5K자 → 12메시지/15K자
  - 요약 프롬프트 강화: 선택지/옵션 개별 보존, 결정 사항 명시 기록
  - 구조화 데이터 판정 확대: 옵션/번호 매긴 목록 패턴 추가 감지
- **효과**: 6턴 원문 유지로 컨텍스트 손실 방지, 비용-맥락 균형 확보

### 🔧 [수정] MCP 캐시 리프레시 행(hang) 문제
- **모듈**: MCPAdapter
- **근본 원인**: `excel_server` 명령어 PATH 미해결 → 영구 실패를 일시 실패로 취급 → 60초마다 대량 서브프로세스 스폰 → 리소스 경쟁 → 무한 대기
- **수정**:
  - `excel_server` 절대 경로로 변경 (LocalSystem 계정에서도 찾을 수 있도록)
  - 영구 실패 블랙리스트 추가 (FileNotFoundError, PermissionError 등)
  - 개별 서버 타임아웃 10초 적용 (ExceptionGroup 언래핑 포함)
  - TTL 단축 조건 변경: 블랙리스트 제외한 일시적 실패만 트리거
- **효과**: 채팅 요청 행 현상 해결, 서버 안정성 향상

### 🔧 [수정] Outline 임베드 채팅 userId anonymous 버그
- **모듈**: Outline, Frontend
- **문제**: iframe 환경에서 사용자 로그가 "anonymous"로 기록
- **원인**: `embed-chat.tsx`에서 URL 파라미터(`empno`)를 `useSimpleChat` 훅에 전달하지 않음
- **수정**:
  - `UseSimpleChatOptions`에 `userId` 옵션 추가
  - 우선순위: 외부 주입 > SSO 쿠키 > "anonymous"
  - `embed-chat.tsx`에서 `userId` prop 전달
- **효과**: 임베드 채팅 로그 정상 기록

### 🔧 [수정] Outline 컬렉션 문서 수 부정확
- **모듈**: OutlineMCP
- **문제**: `list_collections`이 Outline API의 캐시된 `documentCount` 반환 → 실제 수와 불일치
- **수정**: `collections.documents` API를 병렬 호출하여 실제 문서 트리 순회 및 정확한 카운트 반환
- **폴백**: 조회 실패 시 기존 `documentCount` 사용, `documentCountApproximate` 플래그 추가
- **효과**: LLM의 정확한 문서 수 인식으로 응답 신뢰도 향상

## 기술적 세부사항

### VOC Wiki 자동 축적 시스템 아키텍처
```
CronTrigger (매일 06:00 KST)
  ↓
VOC DB 조회 (asyncpg, 증분 동기화)
  ↓
Haiku LLM 배치 분류 (시스템/주제, 20건 단위)
  ↓
시스템/주제별 그룹핑
  ↓
Haiku LLM 병합 (기존 문서 + 신규 건)
  ↓
Outline API 문서 생성/업데이트
  ↓
부모 목차 갱신 및 MySQL sync_log 저장
```

**문서 계층 구조**:
- 컬렉션 → 시스템별 부모 문서(목차) → 주제별 자식 문서
- LLM이 내용 기반 자율 분류 (DB 시스템 컬럼은 hint로만 사용)
- 기존 주제 목록을 context로 제공하여 분
