# 일일 개발 보고서 — 2026-03-10

## 오늘의 요약
토큰 사용량 모니터링 시스템을 구축하고, 문서 생성 기능을 Word(DOCX)까지 확대했으며, Bedrock Prompt Caching을 도입하여 반복 호출 비용을 60% 절감했습니다. 동시에 프롬프트 경량화(59% 압축)와 조직도 계층 조회 개선으로 전반적인 성능과 비용 효율성을 향상시켰습니다.

## 주요 변경사항

### 1. 토큰 사용량 모니터링 시스템 [Dashboard]
- **추가**: `token_usage_log` 테이블 신규 생성 및 TokenUsageService 싱글톤 구현
- **기능**: 모든 LLM 호출(Sonnet/Haiku)의 input/output 토큰을 자동 수집, 모델별/워커별/사용자별 대시보드 시각화
- **아키텍처**: 비동기 버퍼 기반 batch INSERT (5초 또는 20건 flush), 응답 지연 없음
- **대시보드**: KPI 카드(총 토큰/Input/Output), 모델별 도넛 차트, 일별 추이 라인차트, 워커별 사용량 테이블
- **배경**: CloudWatch가 cross-region inference 환경에서 메트릭을 부정확하게 기록하는 문제 해결

### 2. 조직도 계층 조회 개선 [OrgChart]
- **수정**: `부서ID` + `부서경로` 컬럼 기반 2단계 계층 조회 패턴 도입
- **효과**: 상위 부서 검색 시 하위 부서 전체 포함, 형제 부서 정확한 분리
- **검증**: 인사팀/IT센터/총무팀 등 조직 구조별 정확한 계층 반환 확인

### 3. Word(DOCX) 문서 생성 기능 [Visualization]
- **추가**: `docx_generator` MCP 서버 신규 구현 (python-docx 기반)
- **기능**: 마크다운 → DOCX 변환, 3가지 스타일(technical/report/simple) 지원
- **포맷**: 대제목/섹션 제목/코드 블록/테이블/블록인용 등 전문적 서식 제공
- **인텐트**: "워드로/Word로/DOCX로" 패턴 자동 인식, VisualizationWorker에서 PDF/DOCX 선택 가능

### 4. PDF/DOCX 인라인 미리보기 [Frontend]
- **추가**: DocumentViewerPanel 컴포넌트로 채팅 오른쪽 패널에서 바로 미리보기
- **PDF**: 브라우저 내장 PDF 렌더러 (iframe, `Content-Disposition: inline`)
- **DOCX**: `docx-preview` 라이브러리로 HTML 변환 렌더링, 다크모드 지원
- **UX**: 다운로드 링크에 Eye 버튼 추가, XLSX 뷰어와 독립적 상태 관리

### 5. PDF 품질 개선 [PDF Generator]
- **수정**: 여백/색상/표 렌더링/코드 블록 DOCX와 통일
- **개선 사항**:
  - 좌우 여백: 15mm → 25mm
  - 표 헤더 색상: 회색 → 파란색 (43,108,176)
  - 표 텍스트: cell → multi_cell (자동 줄바꿈)
  - 코드 블록: 어두운 배경 → 밝은 회색, 95자 잘림 제거
  - 부제목 지원 추가
  - 제목 크기: 20pt → 22pt, H2: 14pt → 15pt

### 6. 프롬프트 경량화 [BaseWorker & Metadata]
- **수정**: 메타데이터 스키마 전체 59% 압축 (55.9KB → 23KB)
  - MCP_GW_BOARD: 214줄 → 97줄 (-55%)
  - MCP_GW_APPR: 591줄 → 150줄 (-75%)
  - MCP_GW_WORKS_IT: 217줄 → 93줄 (-57%)
  - MCP_GW_WORKS_ACCT: 190줄 → 88줄 (-54%)
  - MCP_ORG_CHART: 226줄 → 137줄 (-39%)
- **방식**: SQL 예제 축소, 공통 컬럼 분리, FAQ 삭제, 날짜 규칙 압축
- **Agent Step 조정**: xlsx 50→30, mail 30→24 (실 사용 패턴 분석 기반)

### 7. 요청별 토큰 사용량 추적 [Observability]
- **추가**: LLM 호출마다 input/output 토큰 수집, `chat_log_new.metadata` JSON에 저장
- **구현**: `on_chat_model_end` 이벤트에서 usage_metadata 추출, 워커별 누적 토큰 계산
- **로그**: 실시간 서버 로그 출력 `[WorkerName] [TOKEN #N] in=X out=Y (cumul: ...)`
- **분석**: SQL `JSON_EXTRACT(metadata, '$.input_tokens')` 등으로 워커별/일별 분석 가능

### 8. Bedrock Prompt Caching [BaseWorker]
- **추가**: `CachedChatBedrockConverse` 서브클래스로 system prompt 캐싱 구현
- **효과**: Agent loop 내 2회차부터 입력 토큰 90% 절감 (4,000 토큰 → 400 토큰)
- **비용**: 도구 호출 3회 기준 system prompt 반복 비용 60% 절감
- **메트릭**: `cache_read_tokens`/`cache_write_tokens` DB 저장, 모니터링 대시보드에 반영
- **적용 범위**: DirectWorker, 모든 Worker 기본 클래스 (단발 호출 제외)

### 9. LUCID_AI_IDENTITY 기능 목록 업데이트 [BaseWorker]
- **수정**: 16개 → 18개 기능으로 확대
