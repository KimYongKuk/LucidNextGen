# 일일 개발 보고서 — 2026-03-19

## 오늘의 요약
L&F Wiki 연동 기능을 완성하여 사내 위키 문서 검색/조회 기능을 추가했고, PPT 생성 품질을 대폭 개선하며 새로운 시각 요소 3종을 도입했습니다. 동시에 OrgChart와 MailWorker의 안정성 문제 2건을 수정하여 LLM 에러 루프를 방지했습니다.

## 주요 변경사항

### [추가] [Embed] L&F Wiki 임베딩용 경량 채팅 페이지
- **모듈**: Frontend (`/embed` 라우트)
- **내용**: 사이드바/헤더/모달 제거한 경량 채팅 페이지 추가
- **특징**:
  - `chat_mode=outline_embed`로 인텐트를 `OUTLINE + DIRECT`만 허용
  - 기존 SSO 쿠키 또는 URL 파라미터(`?empno=xxx`)로 인증
  - 인증 실패 시 401 응답 반환 (iframe 깨짐 방지)
  - 세션 자동 생성, DataStreamProvider만 포함
- **파일**: `frontend/app/embed/layout.tsx`, `frontend/app/embed/page.tsx`, `frontend/components/embed-chat.tsx` 신규 작성

### [추가] [OutlineWorker] L&F Wiki 연동 기능
- **모듈**: Backend (MCP 서버 + Worker)
- **내용**: L&F Wiki REST API 연동으로 사내 위키 문서 검색/조회/요약 기능 제공
- **MCP 도구 5개**:
  - `search_documents`: 키워드 기반 문서 검색
  - `list_recent_documents`: 최근 수정/생성 문서 목록
  - `get_document`: 특정 문서 전체 내용 조회
  - `list_collections`: 컬렉션(카테고리) 목록
  - `list_collection_documents`: 컬렉션 내 문서 트리 조회
- **특징**:
  - Sonnet 모델 사용 (토큰 압축 활성화)
  - 서비스 계정 기반 전체 컬렉션 열람
  - 문서 본문 12,000자 truncation (LLM 컨텍스트 효율)
  - 검색 결과 최대 25건 제한
- **파일**: `backend/app/mcp_servers/outline_server/server.py` 신규, `backend/app/agents/workers/outline_worker.py` 신규

### [추가] [PPTWorker] PPTX 생성 퀄리티 대폭 개선
- **모듈**: Backend (MCP 서버 + Worker)
- **Shape 3종 추가**:
  - `render_callout_box`: 강조 박스 (인사이트/경고/요약용, 4개 프리셋 스타일)
  - `render_kpi_card`: KPI 카드 (대시보드용 핵심 수치, 트렌드 표시)
  - `render_divider`: 시각 구분선 (콘텐츠 영역 분리)
- **차트 스타일 강화**:
  - `series_colors`: 시리즈별 색상 지정 (HEX 배열)
  - `data_labels`: 데이터 값 표시 여부
  - `legend_position`: 범례 위치 제어 (bottom/right/top/left/none)
  - `number_format`: 숫자 포맷 지정
- **레이아웃 패턴 10종 추가**: 테이블+인사이트, 차트 전폭+콜아웃, 2단 컬럼, KPI 대시보드, 차트+텍스트, 프로세스 플로우, 데이터 하이라이트, 섹션 오프너, 테이블+차트, 텍스트 전용
- **시스템 프롬프트 개편**: 차트 JSON 예시, 디자인 규칙, 패턴 선택 가이드, 콘텐츠 밀도 규칙 추가
- **버그 수정**: 흰색 텍스트 상속 방지, Scatter 차트 XyChartData 처리
- **파일**: `backend/app/mcp_servers/ppt_generator/server.py` 수정, `backend/app/agents/workers/ppt_worker.py` 수정

### [수정] [OrgChart MCP] PostgreSQL 부서ID 컬럼 대소문자 fold 에러 수정
- **모듈**: Backend (MCP 서버)
- **문제**: PostgreSQL identifier folding으로 `부서ID` → `부서id` 변환되어 "column does not exist" 에러 발생
- **영향**: 계층 조회 Step 1 실패로 LLM이 10회 tool call 소진 (49초, 116K 토큰 낭비)
- **해결**: SQL 실행 전 `부서ID` → `"부서ID"` 자동 따옴표 치환 (regex 기반)
- **파일**: `backend/app/mcp_servers/org_chart_mcp_server.py` 수정

### [수정] [MailWorker] .eml 파서 스트리밍 방식 전환
- **모듈**: Backend (JSP 유틸리티)
- **문제**: 파일 전체를 메모리에 로드하여 5MB 제한 발생, 첨부파일 크기가 크면 본문 추출 불가
- **해결**: 스트리밍 파서로 전환 (헤더 + 텍스트 파트만 선택적 읽기)
- **개선**:
  - 메모리 사용량: 파일 크기 무관 → 수십KB 고정
  - 5MB 제한 제거
  - 첨부파일 크기와 무관하게 메일 본문 추출 가능
- **헬퍼 메서드 3개 추가**: `readStreamLine()`, `readStreamLimited()`, `scanMultipartStream()`
- **파일**: `backend/data/jsp/lucid_mail.jsp` 수정
