## 주요 변경사항

### [Security/Chat] 채팅 엔드포인트 JWT 인증 의무화
- **문제**: `ChatRequest.user_id`가 body 필드로 검증 없이 주입되어, DevTools로 사번 변조 시 타인 메일/결재/일정 조회 및 세션 삭제 가능
- **해결**: HttpOnly 쿠키 기반 JWT 인증 Dependency(`get_current_user`) 신설, 6개 채팅 엔드포인트 적용
- **효과**: 
  - 7개 Worker(Mail/Approval/Calendar/Reservation/Outline/NAS/ITSupport)가 코드 변경 없이 자동으로 본인 데이터 격리
  - DELETE/PATCH/PIN 엔드포인트에 소유권 검증(403) 추가
  - body/query `user_id` 필드는 Optional 유지(1~2주 후 제거 예정)
- **프론트엔드**: Next.js API Route에서 Cookie 헤더 forward, `use-simple-chat.ts`에 `credentials: 'include'` 추가

### [Security/Widget] 그룹웨어 iframe 위젯 AES 암호화 토큰 인증
- **문제**: 기존 iframe URL `?empno=A2304013` 평문 방식으로 F12에서 사번 조작 가능, JWT 쿠키 인증 도입으로 위젯 동작 불능
- **해결**: 
  - `backend/app/api/dependencies/widget_auth.py` 신설 — AES-ECB 암호화 토큰 검증
  - `chat.py`에 `get_chat_user` 통합 인증 함수 추가 — `X-Widget-Auth` 헤더 또는 JWT 쿠키 검증
  - 7개 채팅 엔드포인트 모두 이 dependency로 전환
- **위젯 구현**:
  - `lucid-chat-widget-gw.js`에 `buildIframeSrc()` 헬퍼로 URL 생성 일원화
  - `embed/gw/page.tsx`가 URL `token` 추출 후 `X-Widget-Auth` 헤더로 전달
  - JSP 측에서 Java scriptlet으로 토큰 생성(키: `WIDGET_SHARED_KEY`)
- **보안**: 타임스탬프 5분 유효 검증으로 replay 방어, legacy `?empno=` fallback 과도기 호환

### [XlsxWorker/Planner] 엑셀 편집 전체 커버리지 + 라우팅 교정
- **문제**: "시트 추가 + 기존 시트 복사" 요청에서 Planner가 `user_files → xlsx` 2-task로 분할하여 환각 오염, 도구 부족으로 기술적 처리 불가
- **해결**:
  - **modify_xlsx 15 ops로 확장** (기존 7 + 신규 8):
    - 데이터: `update_cells`, `apply_formula`
    - 시트: `add_sheet`, `delete_sheet`, `rename_sheet`, `copy_worksheet`
    - 행·열: `insert_rows`, `insert_columns`, `delete_rows`, `delete_columns`
    - 서식: `format_range`, `merge_cells`, `unmerge_cells`
    - 차트·피벗: `create_chart`, `create_pivot_table`
  - openpyxl `wb.copy_worksheet`로 서식·수식·병합까지 통째 복사
  - `_normalize_color` 유틸로 다양한 색상 포맷(`#FFFF00`, `FFF` 등) 허용
  - **Planner CORE RULES에 rule 9 추가** — 엑셀 편집은 xlsx trivial, user_files 분리 금지
  - XlsxWorker 프롬프트 15 ops 안내로 확장
- **검증**: 오늘 실패 시나리오 재현(서식+수식+병합 포함 시트 복사) PASS

### [Planner/Workspace] `workspace_instructions` Planner 입력 주입
- **문제**: 워크스페이스 운영자가 쓴 시스템 프롬프트(검색 순서·핸즈오프 체인 지시)를 Planner가 읽지 못함
- **해결**: `PLANNER_USER_TEMPLATE`에 `Workspace instructions` 섹션 추가, `plan()` 호출부에서 `context["workspace_instructions"]` 전달
- **효과**: 공용 워크스페이스 UI에서 "사례집→VOC→RAG→웹 순으로 검색" 같은 지시를 쓰면 Planner가 이를 읽고 다중 워커 DAG 자동 구성 가능

### [Workspace/공용] 공용 워크스페이스 (Public Workspace)
- **기능**: 운영자(A2304013)가 본인 워크스페이스를 전 사용자에게 읽기 전용으로 공개
- **구현**:
  - `workspaces.is_public` TINYINT 컬럼 추가
  - `GET /v1/workspaces/public` 신규 엔드포인트
  - 읽기(단건/파일목록)는 소유자 또는 `is_public=1` 시 허용, 쓰기는 소유자 전용
  - `is_public` 전환은 운영자만 가능(백엔드 이중 검증)
- **격리**: 세션·채팅·메모리(`workspace_memory` (workspace_id, user_id) UNIQUE)는 자연스럽게 사용자별 격리
- **UI**: 사이드바에 "내 워크스페이스"/"공용 워크스페이스" 섹션 분리, "My Workspace" 한글화
