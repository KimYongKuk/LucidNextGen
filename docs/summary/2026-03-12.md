# 일일 개발 보고서 — 2026-03-12

## 오늘의 요약
AWS Bedrock 일일 토큰 한도 초과 시 자동으로 리전을 전환하는 폴백 시스템을 구현하여 서비스 안정성을 강화했습니다. 동시에 XlsxWorker의 토큰 사용량을 438K에서 약 60K 수준으로 대폭 절감하고, 파일 업로드 및 이미지 처리 관련 버그를 수정하여 전반적인 시스템 안정성을 개선했습니다.

## 주요 변경사항

### [추가] Bedrock 리전 폴백 시스템
- **모듈**: `backend/app/core/region_fallback.py` (신규)
- **기능**: us-east-1 쓰로틀링 감지 시 us-west-2로 자동 전환, UTC 자정 자동 복구
- **상세**:
  - RegionFallbackManager 싱글톤으로 폴백 상태 관리
  - cross-region 모델 ID(us.anthropic.*)를 on-demand 모델 ID(anthropic.*)로 자동 변환
  - 폴백 활성화/복구 시 관리자 메일 알림 (별도 스레드에서 비동기 발송)
  - 환경변수: `AWS_FALLBACK_REGION`, `ADMIN_ALERT_EMAIL`, `REGION_FALLBACK_COOLDOWN_SEC` 추가

### [수정] XlsxWorker 토큰 최적화 2차
- **모듈**: `backend/app/agents/workers/base_worker.py`, `xlsx_worker.py`
- **성과**: 438K → ~60K 토큰 절감 (약 86% 감소)
- **상세**:
  - AIMessage tool_calls args 압축 추가 (data 배열 300자 제한)
  - `COMPACT_KEEP_RECENT_PAIRS` 2→1로 변경 (최근 1개 tool call만 원본 유지)
  - 비문자열 결과 타입 안전성 개선 (`_truncate_tool_result`)

### [수정] Upload 모듈 한글/특수문자 파일명 PPTX 업로드 실패
- **모듈**: `backend/app/services/upload_service.py`
- **원인**: 임시 파일 경로에서 원본 파일명 처리 오류
- **해결**: 상대경로→절대경로 전환, 임시 파일 경로에서 원본 파일명 제거

### [수정] Chat 모듈 대용량 이미지 Bedrock 전송 실패
- **모듈**: `backend/app/services/chat_service.py`
- **원인**: 5MB 초과 이미지 전송 불가
- **해결**: Pillow 기반 자동 JPEG 압축 (해상도 축소 + quality 하향)

## 기술적 세부사항

### Bedrock 리전 폴백 아키텍처
```
Primary (us-east-1, cross-region quota)
    ↓ [쓰로틀링 감지]
Fallback (us-west-2, on-demand quota)
    ↓ [UTC 자정 또는 수동 reset]
Primary 복귀
```

- **BedrockService**: `self.client` property가 폴백 상태에 따라 적절한 boto3 client 반환
- **Workers**: 요청마다 LLM 생성 시 `region_name` 동적 결정
- **IntentClassifier**: 싱글톤이므로 `_ensure_correct_region()`으로 상태 변경 감지 후 LLM 재생성
- **Lazy 생성**: 폴백이 발생하지 않으면 fallback client 생성 비용 없음

### XlsxWorker 토큰 최적화 메커니즘
- **AIMessage args 압축**: `_compact_tool_call_args()` 함수로 대형 배열/딕셔너리를 요약 문자열로 대체
  - `List[List]` → `"[1000행 x 5열 데이터 생략]"`
  - 긴 문자열 → 300자 제한
- **압축 범위 축소**: 최근 1개 tool call 쌍만 원본 유지, 이전 호출은 모두 압축
- **타입 안전성**: 모든 결과를 문자열로 정규화하여 LangGraph 호환성 유지

## 후속 과제

1. **Bedrock 리전 폴백 모니터링**: 폴백 발생 빈도 및 복구 시간 모니터링, 필요시 cooldown 시간 조정
2. **XlsxWorker 성능 검증**: 대규모 엑셀 작업(1000행 이상)에서 토큰 절감 효과 실제 측정 및 LLM 정확도 영향 평가
3. **이미지 압축 품질 튜닝**: 5MB 초과 이미지 압축 시 해상도/quality 파라미터 최적화 (사용자 피드백 수집)
4. **파일명 인코딩 테스트**: 다양한 언어(중국어, 일본어, 이모지 등) 파일명에 대한 회귀 테스트
