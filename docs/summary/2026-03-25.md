# 일일 개발 보고서 — 2026-03-25

## 오늘의 요약
OutlineWorker의 위키 게시 파이프라인을 3단계에서 1단계로 통합하고, 사용자별 컬렉션 접근 제어를 전사 도구에 적용하여 보안을 강화했습니다. 동시에 PDF/PPTX 이미지 OCR 판정 로직을 개선하고 PPTX media_type 버그를 수정하여 운영 안정성을 높였습니다.

## 주요 변경사항

### [수정] OutlineWorker — 위키 게시 파이프라인 통합
**태그:** `OutlineWorker`, `MCP`

- 기존 3개 도구(extract + upload×N + create) → `publish_file_to_wiki` 단일 엔드포인트로 통합
- 이미지 병렬 업로드로 처리 속도 개선
- 정제 모드(refine mode) 제거로 파이프라인 단순화

### [추가] OutlineWorker — 컬렉션 접근 제어
**태그:** `OutlineWorker`

- **사번 기반 권한 조회**: Outline PostgreSQL DB에서 `emp_code` → 접근 가능 컬렉션 ID + 쓰기 권한 조회
- **도구별 필터링 적용**:
  - 읽기 도구(검색, 조회, 목록): 접근 불가 컬렉션 사전 차단 및 결과 필터링
  - 쓰기 도구(문서 생성): 쓰기 권한 있는 컬렉션만 허용
- **권한 판정 로직**: Outline 동일 기준 적용 (admin 역할, 팀 공개, 직접/그룹 멤버십)
- **캐시 전략**: 5분 TTL로 권한 조회 성능 최적화
- **하위 호환성**: `OUTLINE_DATABASE_URL` 미설정 시 기존 동작 유지

### [수정] PDFVisionService — Vision OCR 판정 로직 개선
**태그:** `PDFVisionService`

- **사각지대 2건 해결**:
  1. 텍스트 30~99자 + 큰 스크린샷 이미지 → Vision OCR 대상 추가 (기존: 스킵)
  2. 텍스트 < 30자 + 이미지 없음(빈 페이지) → Vision 스킵 (기존: 호출)
- **`is_page_complex()` 개선**: 페이지 면적 20% 이상 차지하는 이미지 감지 추가
- **`_detect_media_type()` 신규**: 매직바이트 기반 PNG/JPEG/GIF/WEBP 자동 감지

### [수정] PDFVisionService — PPTX media_type 버그 수정
**태그:** `PDFVisionService`

- **원인**: `extract_text_from_image()`에서 media_type을 `"image/jpeg"`로 하드코딩
- **증상**: PPTX 이미지(PNG 원본)를 JPEG로 Bedrock에 전송 → ValidationException 발생 → 에러 무시로 OCR 실패
- **결과**: 운영서버에서 PPTX OCR이 사실상 동작하지 않음
- **수정**: 이미지 포맷 자동 감지로 올바른 media_type 전송

### [수정] ChromaDB — PPTX 이미지 해시 중복 제거 및 모델 로드 안정화
**태그:** `ChromaDB`

- **PPTX 중복 제거**: MD5 해시 기반으로 템플릿 로고/배경 이미지 중복 제거
- **파일당 이미지 제한**: `PPTX_MAX_IMAGES_PER_FILE(20)` 적용
- **모델 로드 안정화**: `low_cpu_mem_usage=True` 설정으로 Windows 환경 메모리 초과 방지

## 기술적 세부사항

### OutlineWorker 접근 제어 아키텍처
```
prepare_tools(user_id, emp_code)
  ├─ Outline DB 권한 조회 (asyncpg 풀)
  │  └─ readable_ids, writable_ids 캐시 (TTL: 5분)
  └─ 도구별 래핑
     ├─ Pre-check: 입력 collection_id 사전 차단
     ├─ Post-filter: 응답 결과 컬렉션 ID 필터링
     └─ your_permission 필드 추가 (LLM 안내용)
```

**권한 판정 쿼리**: Outline 테이블(`user_permissions`, `group_permissions`, `group_users`) 직접 조회로 admin/팀 공개/직접 멤버십/그룹 멤버십 판정

### Vision OCR 판정 개선 로직
```
텍스트 ≥ 100자 → fast-path (스킵)
텍스트 < 100자 →
  ① 페이지 20%+ 큰 이미지 → Vision ✨ NEW
  ② 의미 있는 이미지 + complex_drawings > 30 → Vision
  ③ complex_drawings > 100 → Vision
  ④ 텍스트 < 30자 + 이미지 있음 → Vision
  ⑤ 텍스트 < 30자 + 이미지 없음 → 스킵 ✨ NEW
  ⑥ 그 외 → 텍스트만
```

## 후속 과제

- **운영 배포 모니터링**: PPTX media_type 수정으로 인해 기존에 없던 Vision API 호출 발생 → 비용 및 응답 속도 모니터링 필요
- **Vision OCR 성능 개선**: Bedrock Sonnet Vision 복잡 이미지당 ~15초 소요 — 향후 한글 지원 우수 OCR 전용 서비스 출시 시 교체 검토
- **권한 변경 반영 지연**: 컬렉션 접근 제어 캐시 TTL 5분으로 인한 최대 5분 지연 — 필요시 캐시 무효화 메커니즘 추가 검토
