## 주요 변경사항

### 1. [수정] [Orchestrator] 워크스페이스 우선 실행 + 전문 워커 폴백
**파일**: `backend/app/agents/intent_classifier.py`, `backend/app/agents/orchestrator.py`

- **문제**: 워크스페이스 문서가 있으면 CORP_RAG, IT_SUPPORT 등 전문 워커가 user_files로 강제 오버라이드되어 호출되지 않음
- **해결책**: 
  - Phase 1.8 (workspace-first 라우팅) 추가: 전문 인텐트를 user_files로 교체하되 **원래 인텐트 보존**
  - Phase 4.5 (workspace 폴백) 추가: user_files에서 NO_RESULTS 감지 시 원래 전문 워커로 **자동 폴백**
  - workspace-first 대상 제한: CORP_RAG, IT_SUPPORT, ACCT_SUPPORT, WEB_SEARCH, BOARD만 적용 (MAIL, APPROVAL 등 외부 시스템은 제외)
- **효과**: 최악의 경우 3단계 순차 실행(user_files → 전문 워커 → fallback) 가능하나, 대부분 1단계에서 종료

### 3. [수정] [Architecture] 공유 도구함 + 시각화 3모드 리팩토링
**파일**: 차트 생성기, 모든 워커, 상태 정의, 프론트엔드 컴포넌트 등 광범위

#### 3-1. VisualizationWorker 제거 및 공유 도구함 도입
- **변경**: VisualizationWorker 삭제 → shared_tool_names 프로퍼티로 공유 도구 분배
- **대상 에이전트**: DirectWorker, WebSearchWorker, UserFilesWorker, CorpRAGWorker (opt-in)
- **공유 도구**: create_line/bar/pie/multi_chart, create_document_pdf, create_table_spec_pdf, create_document_docx
- **효과**: 모든 에이전트가 직접 차트/PDF/DOCX 생성 가능, 워커 간 핸드오프 제거

#### 3-2. 시각화 3모드 체계
| 모드 | 기술 | 용도 | 특징 |
|------|------|------|------|
| **Recharts 차트** | JSON + PNG 자동 저장 | 데이터 차트 (라인/막대/파이/복합) | 인터랙티브, 5개+ 데이터 포인트 |
| **인라인 SVG** | LLM이 `<svg>` 태그 직접 출력 | 플로우차트, 인포그래픽, 타임라인 | 모든 에이전트 사용 가능, 실시간 렌더링 |
| **HTML 위젯** | `<lucid-html>` 래퍼, iframe 렌더링 | 비교표, 카드 그리드, 대시보드 | CSS 변수 테마 대응, 실시간 높이 갱신 |

#### 3-3. 차트 MCP 통합 모드
- **변경**: `output_mode` (display/file) 파라미터 제거
- **통일**: 항상 Recharts JSON (프론트엔드 렌더링) + PNG 자동 저장 (PDF 임베딩용)
- **효과**: LLM의 모드 선택 부담 제거, 일관된 출력

#### 3-4. SVG 스트리밍 렌더링 (svg-stream-block.tsx)
- **방식**: div + dangerouslySetInnerHTML (iframe 제거)
- **정제**: DOMPurify 동적 로드 (SSR 에러 방지)
- **최적화**: SvgRenderer memo로 동일 HTML 시 DOM 재생성 방지 (떨림 방지)
- **UI**: 투명 배경, hover 시 오버레이 메뉴 (Copy/Download/Expand)
