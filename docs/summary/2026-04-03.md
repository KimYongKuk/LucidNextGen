# 일일 개발 보고서 — 2026-04-03

## 오늘의 요약
모든 워커에서 업로드 파일에 접근 가능하도록 파일 컨텍스트 공유 기능을 구현했으며, VOC 위키 스케줄러의 이벤트 루프 블로킹 문제를 해결하고 동기화 누락 방지 로직을 개선했다. 특히 스케줄러 안정화로 과거 3/1~4/3 전량 447건의 누락분을 백필 완료했다.

## 주요 변경사항

### [추가] BaseWorker — 파일 컨텍스트 자동 공유
- **모듈**: `backend/app/agents/workers/base_worker.py`
- 모든 워커에서 `search_user_files`, `search_workspace_docs` 도구 자동 주입
- 파일 존재 시 시스템 프롬프트에 "UPLOADED FILES" 섹션 자동 추가
- 기존 파일 접근 도구 보유 워커는 중복 방지로 건너뜀
- 토큰 증가: 파일 존재 시 약 200토큰 추가

### [수정] IntentClassifier — 파일 우선 라우팅 개선
- **모듈**: `backend/app/agents/intent_classifier.py`
- `has_files=True` + 분석/요약/정리/확인 키워드 → USER_FILES 즉시 반환
- 파일 명시 없이 "분석해줘"만 말해도 파일 우선 접근 가능

### [수정] VocWikiScheduler — 이벤트 루프 블로킹 해결
- **모듈**: `backend/app/services/voc_wiki_service.py`, `backend/app/utils/voc_wiki_scheduler.py`
- boto3/MySQL 동기 호출을 `ThreadPoolExecutor`로 격리하여 이벤트 루프 블로킹 제거
- `asyncpg.create_pool()` 연결 타임아웃 추가 (15초)
- `misfire_grace_time` 3600 → 60으로 축소 (재시작 시 불필요한 스케줄 실행 방지)
- `_execute()` 예외 처리 추가로 서버 프로세스 보호

### [개선] VocWikiScheduler — 동기화 누락 방지
- **모듈**: `backend/app/services/voc_wiki_service.py`
- since 로직 개선: `last_sync_date + 1일` → `last_sync_date` (1일 오버랩)
- 동기화 중간 크래시 시 당일 미처리 VOC 누락 방지
- LLM 병합 프롬프트가 중복 자동 처리
- 과거 누락분 백필: 3/1~4/3 전량 447건 재처리 완료 (신규 4건 생성, 66건 업데이트)

## 기술적 세부사항

### 파일 컨텍스트 공유 아키텍처
- BaseWorker의 `stream_response()` 메서드에서 `has_files` 또는 `workspace_has_files` 플래그 감지 시 파일 검색 도구 자동 추가
- `build_system_prompt()` 메서드에서 파일 존재 여부에 따라 동적으로 시스템 프롬프트 구성
- 개별 워커 수정 불필요 (베이스 레벨 구현으로 모든 워커 자동 적용)

### VOC 위키 스케줄러 안정화 구조
- **문제 원인**: APScheduler의 async job이 이벤트 루프에서 직접 실행되는데, 내부 boto3 동기 호출이 이벤트 루프 전체를 블로킹
- **해결책**: 
  - `_call_haiku_safe()` 래퍼 메서드로 boto3 호출을 ThreadPoolExecutor에서 실행
  - `_get_last_sync_date()`, `_save_sync_log()` 등 MySQL 호출도 `run_in_executor`로 격리
  - 전용 스레드 풀 사용 (`max_workers=3, thread_name_prefix="voc_wiki"`)
- **누락 방지**: 1일 오버랩으로 재시작 후에도 자동 복구 가능

## 후속 과제
