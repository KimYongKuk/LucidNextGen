## 주요 변경사항

### [추가] Logging — 서버 로그 파일 출력

- 10MB 단위 로테이션, 최대 5개 백업 파일 유지 (~50MB 디스크 사용)
- `bat/tail_log.bat`으로 별도 터미널에서 실시간 로그 모니터링 가능 (PowerShell `Get-Content -Wait`)
- UTF-8 인코딩 명시, `fileno()`/`isatty()` 위임으로 uvicorn/subprocess 호환성 확보

### [수정] Intent — 결재 Follow-up 인텐트 오분류 수정
- **모듈**: `intent_classifier.py`, `orchestrator.py`, `a2a_streaming.py`, `chat_log_service.py`
- "WA전표품의" 등 결재 양식명이 `acct_support`로 오분류되던 버그 해결
- quick_classify에 결재 양식명 키워드 추가 (전표품의, 품의서, 사전지출승인 등)
- CLASSIFIER_PROMPT에 양식명 목록 명시, acct_support에서 "WA" 제거
- **previous_intent 전달 체인**: DB → a2a_streaming → orchestrator → classifier로 이전 턴 intent 명시적 전달
- `chat_log_service.get_last_intent()` 메서드 추가로 follow-up 판단 강화

### [수정] BaseWorker — Haiku 대화 요약 기본화
- **모듈**: `base_worker.py`, `ppt_worker.py`, `visualization_worker.py`, `xlsx_worker.py`, `direct_worker.py`
- 기존 3개 워커(XlsxWorker, PPTWorker, VisualizationWorker)에 중복 구현된 요약 코드 ~300줄 제거
- BaseWorker에 `_summarize_history_if_needed()`, `_format_messages_for_summary()` 메서드 추가
- **모든 워커**에 멀티턴 요약 기본 적용 (메시지 6개 AND 5,000자 이상 시 Haiku로 요약)
- `summarization_prompt` 프로퍼티로 워커별 도메인 특화 요약 프롬프트 오버라이드 가능
- PPTWorker에 `compact_previous_results=True` 추가 (ReAct loop 토큰 누적 방지)

### [수정] XlsxWorker — ReAct loop 토큰 최적화
- **모듈**: `xlsx_worker.py`, `base_worker.py`
- **Approach A**: 개별 tool result를 8,000자로 제한 (`_truncate_tool_result()`)
  - 극단적 대량 데이터(500행 엑셀 등) 방어, 사용자에게 apply_formula 사용 안내
- **Approach B**: 이전 step tool result 압축 (`_compact_tool_messages()`)
  - 최근 2개 tool call 쌍은 원본 유지, 이전 쌍은 200자로 축약
  - tool_call_id 페어링 유지, state_modifier를 callable로 전환하여 매 LLM 호출 전 압축
- **예상 토큰 절감**: 82% (콜당 40만 토큰 → 7만 토큰)
- `compact_previous_results` 프로퍼티 기본값 False → XlsxWorker만 True로 오버라이드

### [추가] Upload — PPTX 슬라이드 이미지 OCR
- **모듈**: `chromadb_service.py`
- PPTX 파일 업로드 시 텍스트/표/차트/그룹shape/이미지 모두 추출
- **Shape 처리 로직** (재귀적):
  - 표 → 마크다운 테이블 형식
  - 차트 → 카테고리/시리즈/값 직접 추출
  - GROUP shape → 자식 shapes 재귀 탐색
  - PICTURE shape → 이미지 바이트 수집 (10KB 미만 스킵)
- 이미지는 Claude Vision API로 OCR 처리, `asyncio.gather`로 병렬 호출
- 파일당 최대 20개 이미지 OCR (비용 제한)

### [수정] 알림 — 브리핑 수신문서 임시저장 문서 노출 버그 수정
- **모듈**: 결재 알림 쿼리 (`v_appr_dept_received`)
- `appr_status != 'TEMPSAVE'` 필터 추가로 임시저장 문서가 브리핑에 노출되지 않도록 수정

## 기술적 세부사항

### 로깅 아키텍처
- **_TeeWriter 클래스**: `sys.stdout`/`sys.stderr` 래핑으로 `print()` 출력도 로그 파일에 복제
- Windows 파이프 방식(tee) 불가 → Python 내부 로깅으로 전환
- `fileno()`, `isatty()` 위임으로 uvicorn/subprocess 호환성 확보

### 인텐트 분류 개선
- **previous_intent 전달 체인**: 세션의 마지막 intent를 DB에서 조회하여 분류 프롬프트에 컨텍스트 제공
- **Rule 10 (FOLLOW-UP) 강화**: 이전 턴 intent 명시적 전달로 follow-up 판단 정확도 향상
- quick_classify의 양식명 매칭으로 대부분 follow-up은 LLM 호출 없이 즉시 분류
