### 3. [추가] Image — 업로드 이미지 영구 보존
**모듈**: `Image` | **파일**: `upload.py`, `chat.py`, `chat_log_service.py`, `preview-attachment.tsx`

- 이미지를 디스크에 저장: `data/user_uploads/{YYYY-MM-DD}/{user_id}/{uuid}.ext`
- 메타데이터에 `stored_filename` 필드 추가하여 DB에 참조 저장
- 히스토리 로드 시 이미지 복원: 백엔드 다운로드 엔드포인트 `GET /api/v1/image/download/{filename}`
- 프론트엔드 렌더링: 백엔드 URL은 일반 `<img>` 태그, data URL은 Next.js `<Image>` 컴포넌트 사용
- 파일명에 `user_id` 포함하여 관리자 추적 용이
- 캐시 헤더: `Cache-Control: public, max-age=31536000, immutable` (UUID 기반 불변)

### 4. [추가] Archive — Output 파일 아카이브 시스템 + 업로드 폴더 구조 개선
**모듈**: `Archive` | **파일**: `file_archive.py` (신규), `base_worker.py`, `xlsx_worker.py`, `upload.py`

**Output 파일 아카이브 (복사 방식)**:
- MCP 도구 실행 후 생성 파일을 자동으로 아카이브 디렉토리에 복사
- 아카이브 경로: `data/file_archive/{YYYY-MM-DD}/{user_id}/{file_type}/{filename}`
- 원본 파일은 그대로 유지 (LLM 참조, 다운로드 엔드포인트 호환성)
- 아카이브 대상 도구: PDF, PPT, 차트(4종), XLSX 생성 도구
- BaseWorker의 `_wrap_tools_for_archive()` 메서드로 도구 결과 래핑
- XlsxWorker는 보안 래핑 후 아카이브 래핑 순서 엄수

**업로드 폴더 구조 개선**:
- 이미지: `data/user_uploads/{YYYY-MM-DD}/{user_id}/{uuid}.ext`
- 일반 파일: `data/user_uploads/{YYYY-MM-DD}/{user_id}/{filename}` (기존 session_id 기반에서 변경)
- 다운로드 엔드포인트 하위 호환: `/` 포함 시 신규 경로, 없으면 레거시 경로 탐색

### 5. [수정] Upload — 7일 파일 보관 및 자동 정리 시스템
**모듈**: `Upload` | **파일**: `upload.py`, `chromadb_cleanup.py`, `file_cleanup.py`, `use-session-cleanup.ts`

- 모든 업로드 파일 원본을 디스크에 7일간 보관 (`UPLOAD_RETENTION_HOURS=168`)
- ChromaDB 세션 컬렉션 보관 기간: 24시간 → 7일 (`SESSION_RETENTION_HOURS=168`)
- 프론트엔드 즉시삭제 제거: `sendBeacon` 기반 cleanup 로직 완전 삭제
- 백엔드 스케줄러 기반 자동 정리로 전환 (안정성 향상)
- `file_cleanup.py`에 per-target retention 지원: 출력 파일(1년) vs 업로드 파일(7일)
- ChromaDB 고아 디렉토리 정리: `_cleanup_orphan_dirs()` 함수로 segments 테이블에 없는 UUID 디렉토리 자동 삭제
- 초기 상태 고아 디렉토리 ~200개(7.86GB) → 다음 스케줄러 실행 시 정리
