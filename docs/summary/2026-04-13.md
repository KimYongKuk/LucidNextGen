## 주요 변경사항

### 1. [추가] Widget/nginx — 다우오피스 그룹웨어 플로팅 위젯 연동
- **배경**: 다우오피스 공식 커스터마이징 불가 → JSP 커스텀 포인트(`custom_index_header.jsp`) 활용
- **구현**:
  - nginx 정적 파일 서빙 설정 (`/widget/` location 추가, HTTPS/HTTP 양쪽)
  - 위젯 JS SSE 이벤트 매핑 수정 (백엔드 형식 대응)
  - SPA 환경 MutationObserver 추가로 body 교체 시 자동 재삽입
  - 위젯 on/off 파일명 전환 운영 (`.off` 확장자로 간편 제어)
- **특징**: 다우오피스 서버 재시작 불필요, 그룹웨어 업데이트 시 JSP 3줄만 재삽입

### 2. [추가] ChromaDB — BM25+시멘틱 하이브리드 검색
- **문제**: Lot 번호/코드 검색 실패 (시멘틱 서치만으로는 정확 키워드 구별 불가)
- **해결**:
  - BM25 키워드 검색 병행
  - Reciprocal Rank Fusion (RRF) 알고리즘으로 두 결과 합산
  - BM25 캐시 매니저 (5분 TTL, 스레드 안전)
  - 환경변수로 가중치 조절 가능 (`HYBRID_SEMANTIC_WEIGHT`, `HYBRID_BM25_WEIGHT`)
- **성능**: 캐시 적용 시 검색 지연 0.1~1.4ms (추가 오버헤드 거의 0)

### 3. [수정] OutlineSync — Webhook+청크 기반 동기화 전면 개편
- **기존 문제**: 30분 폴링 + Haiku 요약 → GPU OOM 크래시(매일 아침), 본문 앞 3,000자만 검색 가능
- **개선**:

  - asyncio.Queue 단일 소비자 순차 처리 (GPU OOM 해결)
  - 4시간 폴백 delta sync (놓친 webhook 보완)
- **API**: 단건 재인덱싱, delta sync, 전체 재인덱싱 엔드포인트 추가

## 기술적 세부사항

### 하이브리드 검색 아키텍처
```
쿼리 ─┬─→ ChromaDB 시멘틱 검색 (top limit×3)  ─┐
      └─→ BM25 키워드 검색 (top limit×3)       ─┤
                                                 ↓
                                    RRF 점수 합산 → top limit 반환
```
- 정규식 토크나이저로 `PP2509-138` 같은 코드 보존
- min_relevance 필터: 시멘틱은 cosine similarity, BM25는 정확 매칭으로 통과

### Outline Webhook 처리 흐름
```
문서 변경 → Webhook 수신 → asyncio.Queue
  → 단일 Worker:
    1. 기존 청크 삭제 (document_id 기준)
    2. 본문 청크 분할 (한국어 문장 경계 인식)
    3. 청크별 임베딩 → ChromaDB upsert (소배치 8건)
  → 완료 (~3초)
```
- 서명 검증: `OUTLINE_WEBHOOK_SECRET` 환경변수
- 중복 제거: 같은 document_id가 큐에 있으면 스킵

### 그룹웨어 위젯 서버 구조
```
사용자 → 다우오피스 내장 nginx(80/443) → Tomcat(8080)
         ↓
    custom_index_header.jsp 삽입 포인트
         ↓
    https://lucid.landf.co.kr/widget/lucid-chat-widget.js
```
- JSP 변수 `${userId}` 활용으로 로그인ID 자동 전달
- SPA(Backbone.js) 구조로 최초 1회 로드 후 모든 화면에서 위젯 유지
