# 일일 개발 보고서 — 2026-03-31

## 오늘의 요약
자체 로그인 인증 시스템, 회의실 예약 Worker, Context-Aware 도구 상태 메시지, WORKS VOC 자동 등록 기능을 완성했습니다. 사용자 인증 체계를 SSO와 ID/PW 병행으로 확대하고, 챗봇의 자산 예약 및 IT 서비스데스크 연동 기능을 추가하여 플랫폼의 업무 자동화 범위를 크게 확장했습니다.

## 주요 변경사항

### [추가] [Auth] 자체 로그인 인증 시스템
- **기능**: SSO와 병행하는 ID/PW 기반 자체 로그인, JWT 토큰 세션 관리, 이메일 기반 셀프 비밀번호 설정, 로그아웃
- **구현**:
  - Backend: `users` + `setup_tokens` 테이블, 7개 인증 API (login, register, change-password, verify-token, request-setup, setup-password, decrypt)
  - Frontend: 로그인/비밀번호 설정 페이지, 미들웨어 JWT 검증 (jose 라이브러리), 쿠키 4개 관리 (empno, login_id, user_name, auth_token)
  - 사이드바에 사용자 정보(login_id) 표시 및 로그아웃 버튼 추가
- **보안**: bcrypt 해싱(rounds=12), 1회용 설정 토큰(24시간 유효), httpOnly auth_token 쿠키, TIMS DB 사원 정보 조회 기반 셀프 가입
- **인증 우선순위**: SSO empno URL 파라미터 > JWT auth_token 쿠키 > empno 쿠키 (레거시 호환)

### [추가] [ReservationWorker] 회의실/자산 예약 Worker
- **기능**: LFON REST API 연동으로 회의실/공용차량 조회, 예약 현황 확인, 내 예약 조회, 예약 등록/취소
- **MCP 도구** (6개):
  - `get_sites`: 사업장 목록
  - `get_rooms`: 회의실 목록
  - `get_daily_reservations`: 날짜별 예약 현황
  - `get_my_reservations`: 내 예약 조회 (전 사업장 병렬, asyncio.Semaphore(20) 동시성 제한)
  - `create_reservation`: 예약 등록 (충돌 감지 후 대안 제시)
  - `cancel_reservation`: 예약 취소
- **인증**: 서비스 계정 SSO 로그인 + 쿠키 캐싱, 401/403 시 자동 재로그인, POST/DELETE에 Origin/Referer 헤더 필수
- **보안**: `prepare_tools()`에서 employee_number 강제 주입, 시스템 프롬프트에 확인 규칙, 예약 전 충돌 확인 필수
- **사번 매핑**: `v_user_info_mapping` 재사용 (사번 → GO user.id)
- **에러 처리**: API 응답 JSON 파싱으로 실제 에러 메시지 사용자 전달

### [수정] [A2AStreaming] 도구 상태 메시지 Context-Aware 개선
- **문제**: 정적 메시지만 표시 → 사용자가 AI의 실제 작업 내용 파악 어려움
- **해결**:
  - `TOOL_CONTEXT_GENERATORS`: 도구별 lambda 함수로 파라미터(검색어/키워드)에서 동적 메시지 생성
  - 예: `tavily_search` → "'VPN 접속 오류'에 대해 웹에서 검색하고 있습니다..."
  - 지원 도구: tavily_search, search_hr/ac/it/safety_docs, search_user_files, search_workspace_docs, search_it_voc, search_mail, fetch
- **중복 억제**: 도구 이름 기준 → 메시지 내용 기준으로 변경 (같은 도구도 다른 쿼리면 둘 다 표시)
- **도구 완료 메시지**: `TOOL_COMPLETION_MESSAGES`로 도구별 차별화 메시지
- **헬퍼**: `_truncate()` 함수로 검색어 20자 초과 시 "..." 추가

### [추가] [ITSupportWorker] WORKS 서비스데스크 VOC 자동 등록
- **기능**: IT 질문 답변 후 사용자 승인 시 WORKS(앱릿 934)에 VOC 자동 등록 + 담당 부서원 자동 배정
- **MCP 도구**: `register_works_voc` (employee_number, title, details, system_name)
- **내부 로직**:
  - `v_user_info_mapping`에서 요청자 정보 조회
  - 시스템명 → 담당 부서 자동 매핑 (SAP→ERP파트, LFON→DA파트, 보안→보안기술팀 등)
  - 담당 부서 멤버 조회 → SSO API로 VOC 생성 + 담당자 배정
  - SSO 실패 시 OpenAPI 폴백 (담당자 없이)
- **상태 전환**: VOC 생성 후 자동으로 접수 → 담당자지정 상태 전환
- **보안**: `prepare_tools()`에서 employee_number 강제 주입, SSO 쿠키 프로세스 수명 캐싱
- **API 이원화**: SSO API(담당자 지정 가능) vs OpenAPI(폴백)

### [수정] [IntentClassifier] Follow-up 인텐트 유지 규칙
- **문제**: 멀티턴 대화에서 "등록해줘", "진행해" 등 follow-up 메시지가 quick_classify 미매칭 → direct/clarify로 잘못 분류
- **해결**: quick_classify 미매칭 + previous_intent 존재 시 이전 인텐트 유지
- **적용**: IT 지원 → WORKS 등록, 결재 → 상신, 메일 → 발송 등 연속 작업 시 주제 유지
