# 일일 개발 보고서 — 2026-04-06

## 오늘의 요약
캘린더/예약 Worker의 통합 테스트 결과를 반영하여 LLM 시간대 분석 오류를 서버 측 검증으로 해결하고, NAS 파일 탐색 기능을 신규 구현했으며, Outline Wiki 검색을 하이브리드 시멘틱 방식으로 고도화하고 텍스트 직접 문서 생성 도구를 추가했다. 또한 핸드오프 루프 방지를 위한 기본 프롬프트 개선을 적용했다.

## 주요 변경사항

### [수정] CalendarWorker / ReservationWorker — 통합 테스트 버그 수정 및 기능 보강
- **find_available_rooms 도구 추가**: LLM의 시간대 교차 분석 오류(32건 예약 데이터에서 빈 회의실 오판정) → 서버 측에서 겹침 판정 코드로 확정적 계산 (`NOT (req_end <= r_start OR r_end <= req_start)`)
- **create_reservation 충돌 사전 검증**: 예약 직전 재조회로 item_id 시간 겹침 체크, 충돌 시 상세 정보 반환
- **get_my_calendars 사용자 API 기반 변경**: `/api/calendar/feed` → `/api/calendar/user/{goUserId}/calendar` (본인 캘린더 정상 반환)
- **get_calendar_events URL 인코딩 수정**: `+09:00`의 `+`를 `%2B09:00`으로 인코딩
- **create_event API 호환 수정**: `timeType` 값 정정(`"normal"` → `"timed"`/`"allday"`), `type: "normal"` 필드 추가
- **attendee_names 사내 참석자 이름 검색**: 콤마 구분 이름 입력 → DB 검색 → GO user ID + email 자동 연결
- **일정+회의실 동시 등록**: CalendarWorker에 `find_available_rooms`, `create_reservation`, `execute_org_chart_query` 도구 추가
- **인텐트 우선 규칙**: calendar + reservation 동시 매칭 시 CalendarWorker 우선 처리

### [추가] NASWorker — Synology NAS 파일 탐색 및 다운로드
- **MCP 도구 4개 (읽기 전용)**:
  - `list_nas_directory`: 폴더 내 파일/하위폴더 목록
  - `search_nas_files`: 파일명 키워드 재귀 검색 (max_depth 1~5, 최대 50건)
  - `download_nas_file`: 파일 다운로드 → `data/nas_download/{date}/{uuid}_{filename}`
  - `get_nas_file_info`: 파일/폴더 메타정보 (크기, 수정일)
- **이중 보안 방어**: MCP 서버 레벨 path traversal 차단 + Worker 레벨 재검증 + 감사 로깅
- **환경변수**: `NAS_WEBDAV_URL`, `NAS_USERNAME`, `NAS_PASSWORD`, `NAS_ALLOWED_PATHS` (화이트리스트), `NAS_WORKER_ENABLED`
- **Intent 분류**: `Intent.NAS` 추가, 키워드(NAS, 공유폴더, 부서간공유, 시놀로지 등) 자동 감지

### [추가] OutlineWorker — 하이브리드 시멘틱 검색
- **검색 방식**: Outline 키워드 검색 + ChromaDB 시멘틱 검색(BGE-m3-ko) → Reciprocal Rank Fusion(RRF) 병합
- **동기화 파이프라인**: Outline API → Haiku 문서 요약(200~300자) → BGE-m3-ko 임베딩 → ChromaDB upsert
- **자동 동기화**: 30분 주기 증분 동기화 (변경분만 처리)
- **수동 트리거 API**: `POST /api/v1/admin/outline-sync/trigger` (백그라운드), `POST /api/v1/admin/outline-sync/run` (동기)
- **환경변수**: `OUTLINE_SYNC_ENABLED`, `OUTLINE_SYNC_INTERVAL_MINUTES`, `OUTLINE_HYBRID_SEARCH`

### [추가] OutlineWorker — 텍스트 직접 문서 생성/수정 도구
- **create_document**: 마크다운 텍스트 → 위키 문서 직접 생성 (파일 업로드 불필요), 쓰기 권한 검증 포함
- **update_document**: 기존 문서 수정 (전체 교체 또는 끝에 추가)
- **워크플로우 가이드**: 파일→위키(A), 텍스트→위키(B), 문서 수정(C) 3가지 명시

### [수정] BaseWorker — 핸드오프 타겟 프롬프트 추가
- **is_handoff_target=True일 때**: "할 수 있는 것만 수행, 못 하는 부분 무시" 지시 주입
- **핸드오프 루프 방지**: WebSearchWorker 등이 선행 데이터 수집 역할 명확화

## 기술적 세부사항

### 캘린더/예약 통합 아키텍처
```
사용자 요청(일정+회의실)
  ↓
[Intent 분류] calendar+reservation 키워드 → CalendarWorker 우선
  ↓
[CalendarWorker]
  ├─ execute_org_chart_query: 팀/파트 인원 조회
  ├─ get_my_calendars: 본인 캘린더 조회 (user API 기반)
  ├─ get_calendar_events: 시간대별 일정 조회
  ├─ find_available_rooms: 빈 회의실 필터링 (서버 측 시간 겹침 계산)
  ├─ create_event: 일정 생성
  └─ create_reservation: 회의실 예약 (충돌 사전 검증)
```

**핵심 개선**: LLM이 32건 예약 데이터에서 시간 겹침을 잘못 판정하는 문제 → 서버 측에서 코드로 확정적 계산하여 ✅/❌ 분류 후 반환
