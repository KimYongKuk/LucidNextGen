## 슬라이드 2: 캘린더/예약 에이전트

**"일정 잡아줘" 한 마디로 일정 등록부터 회의실 예약까지**

| 기능 | 설명 |
|------|------|
| 일정 조회 | 내 일정, 동료 공개 일정, 기간별 조회 |
| 일정 등록/수정/삭제 | 참석자·반복·알림 포함, 반복 일정 개별/전체 삭제 |
| 빈 시간 찾기 | 팀원 일정 교차 분석 → 공통 가능 시간대 도출 |
| 회의실 검색 + 예약 | 빈 회의실 조회 → 일정 등록과 회의실 예약 동시 처리 |

**워크플로우 예시:**
"개발팀 회의 잡아줘" → 조직도에서 구성원 확인 → 각자 일정 수집 → 공통 빈 시간 도출 → 빈 회의실 조회 → 사용자 확인 후 일정+회의실 한 번에 등록

**보안:** 사번 자동 주입(변경 불가), 본인 캘린더만 쓰기 가능, 비공개 일정 마스킹, 변경 전 사용자 확인 필수

## 슬라이드 3: NAS 에이전트

**공유폴더를 채팅으로 탐색·검색·업로드**

| 기능 | 설명 |
|------|------|
| 폴더 탐색 | NAS 경로의 파일/폴더 목록 조회 |
| 파일 검색 | 키워드 기반 재귀 검색 (최대 5단계 깊이, 50건) |
| 파일 다운로드 | NAS 파일을 로컬로 다운로드 |
| 파일 업로드 | AI가 생성한 파일(PDF, PPT, Excel 등)을 NAS에 업로드 |
| 폴더 생성 | NAS에 새 폴더 생성 |

**활용 예시:**
- "부서간공유에서 2026년 교육일정 찾아줘" → 검색 결과 표시 → 선택 다운로드
- "이 차트 만들어서 부서간공유/보고서에 올려줘" → 차트 생성 → NAS 업로드

**보안:** 허용 경로 화이트리스트, 경로 순회(..) 차단, AI 출력 폴더만 업로드 허용, NAS 자체 ACL 적용, 전 도구 감사 로그

## 슬라이드 4: L&F Wiki 통합

**사내 지식을 자연어로 검색하고, 문서를 만들고, Wiki 안에서 바로 대화한다**

| 기능 | 설명 |
|------|------|
| 문서 검색 | 키워드 + 시맨틱 하이브리드 검색 (의미 기반 매칭 포함) |
| 문서 열람/생성/수정 | 자연어로 Wiki 문서 CRUD |
| 파일 → Wiki 발행 | PDF/PPT/DOCX를 마크다운 변환하여 Wiki에 자동 게시 |
| 임베드 위젯 | Outline Wiki 화면 내 AI 채팅창 내장, Wiki 문맥에 특화 |
| 접근 권한 연동 | Outline 권한 체계 그대로 반영, 미허용 문서는 검색 결과에서 제외 |

**임베드 위젯:**
Wiki를 보다가 궁금하면 그 자리에서 AI에게 질문 → AI가 관련 문서 검색·요약 → 링크 클릭 시 Wiki 내에서 바로 이동

**하이브리드 검색:**
키워드 매칭 + 벡터 시맨틱 검색 결과를 RRF 알고리즘으로 병합하여 정확도 향상 (예: "와이파이" 검색 → "무선 네트워크 접속 장애" 문서도 포함)

## 슬라이드 5: VOC 자동 문서화

**IT 지원 사례가 매일 자동으로 Wiki 지식이 된다**

| 항목 | 내용 |
|------|------|
| 데이터 소스 | WORKS VOC 시스템 — 조치 완료 건 |
| 실행 주기 | 매일 오전 6시 자동 |
| 분류 체계 | AI가 16개 카테고리로 자동 분류 (SAP, 네트워크, VPN, 보안, HW, SW 등) |
| 문서 처리 | 기존 주제 문서 있으면 병합(중복 제거+보강), 없으면 신규 생성 |
| 발행 | Wiki 자동 게시 + 시스템별 목차 문서 자동 업데이트 |
| 알림 | 처리 결과 메일 발송 |

**자동 생성되는 Wiki 구조:**
```
L&F IT 지식베이스
├── SAP 지원 사례 모음 (목차 — 자동 생성)
│   ├── SAP - 로그인/권한 문제
│   └── SAP - 트랜잭션 오류
├── 네트워크 지원 사례 모음
│   └── 네트워크 - VPN 접속 장애
└── ... (AI가 분류에 따라 자동 확장)
```

**핵심 가치:** 지식이 사람에서 시스템으로 이전, 반복 문의 자가 해결, IT 지원 부하 감소
