# 2026-03-31 Context-Aware 도구 상태 메시지

## 개요
에이전트의 도구 실행 상태 메시지를 정적 문구에서 실제 파라미터 기반 동적 메시지로 개선. 사용자가 AI가 실제로 무슨 작업을 하고 있는지 맥락을 이해할 수 있도록 함.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| backend/app/agents/a2a_streaming.py | 수정 | context-aware 메시지 생성 시스템 추가 |

## 상세 내용

### 문제
기존에는 도구 실행 시 `TOOL_STATUS_MESSAGES` 딕셔너리에서 정적 메시지만 표시:
- "웹을 검색해보고 있습니다. 조금만 기다려주세요!" (검색어 정보 없음)
- 같은 도구를 다른 파라미터로 2번 호출 시 2번째는 무시됨

### 해결: 3단계 메시지 해석

1. **`TOOL_CONTEXT_GENERATORS`**: 도구별 lambda 함수가 `tool_input`에서 검색어/키워드를 추출하여 동적 메시지 생성
2. **`TOOL_STATUS_MESSAGES`**: 파라미터가 없거나 generator 미등록 도구는 기존 정적 메시지로 폴백
3. **제네릭 폴백**: 어느 쪽에도 없는 미등록 도구용 기본 메시지

### 동적 메시지 지원 도구
| 도구 | 추출 파라미터 | 메시지 예시 |
|------|-------------|-----------|
| tavily_search | query | "'VPN 접속 오류' 에 대해 웹에서 검색하고 있습니다..." |
| search_hr/ac/it/safety_docs | query | "'연차 규정' 관련 인사 문서를 찾아보고 있습니다..." |
| search_user_files | query | "'프로젝트 계획' 관련 파일을 검색하고 있습니다..." |
| search_workspace_docs | query | "'보고서' 관련 워크스페이스 문서를 검색하고 있습니다..." |
| search_it_voc | keyword | "'VPN 오류' 관련 IT 지원 사례를 찾아보고 있습니다..." |
| search_mail | keyword | "'프로젝트 진행' 관련 메일을 검색하고 있습니다..." |
| fetch | (없음) | "웹 페이지 내용을 가져오고 있습니다..." |

### 중복 억제 방식 변경
- **기존**: 도구 이름 기준 (`tool_name not in tool_calls_made`) → 같은 도구 2번째 호출 무시
- **개선**: 메시지 내용 기준 (`status_msg not in tool_messages_sent`) → 같은 도구라도 다른 쿼리면 둘 다 표시

### 도구 완료 메시지 차별화
- **기존**: 모든 도구 동일 "취합 완료! 응답 생성 중..."
- **개선**: `TOOL_COMPLETION_MESSAGES` 딕셔너리로 도구별 완료 메시지 (예: "웹 검색 결과를 정리하고 있습니다...")

### `_truncate()` 헬퍼
- 검색어가 20자 초과 시 "..." 추가 (메시지가 지나치게 길어지는 것 방지)
- 줄바꿈 → 공백 치환

## 결정 사항 및 주의점
- 프론트엔드 변경 없음 (SSE `tool_status` 이벤트 포맷 동일, 메시지 내용만 변경)
- SQL 쿼리 도구(execute_*_query)는 SQL을 노출하지 않고 정적 메시지 사용
- YouTube/메일함 조회 등 파라미터가 민감하거나 의미 없는 도구는 정적 메시지 유지
- `TOOL_CONTEXT_GENERATORS`의 generator가 예외 발생 시 자동으로 정적 메시지 폴백
