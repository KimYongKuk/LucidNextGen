# 2026-03-03 전자결재 알림 문서 접근 개선

## 개요
알림 모달에서 참조/결재대기/수신 문서를 클릭 시, "다른 사용자 문서 접근 불가" 오류가 먼저 발생하던 문제 수정. 사용자가 접근 가능한 문서(참조함, 결재대기함 등)는 즉시 조회되도록 프론트엔드 메시지 포맷과 백엔드 시스템 프롬프트를 개선.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| `frontend/components/notice-toast/notice-modal.tsx` | 수정 | 클릭 시 출처 컨텍스트(참조함/결재대기함/수신문서함) + doc_id를 메시지에 포함 |
| `backend/app/agents/workers/approval_worker.py` | 수정 | 시스템 프롬프트 규칙 4번 개선 — 접근 가능한 뷰에서 먼저 검색하도록 변경 |

## 상세 내용

### 문제
1. 알림 모달에서 참조 문서 클릭 → `'[제목]' 전자결재 문서 확인해줘` 메시지 전송
2. LLM이 제목만 보고 타인의 문서로 판단 → "다른 사용자 문서 접근 불가" 선제 거부
3. 사용자가 수동으로 "참조함에서 찾아줘" → 그제서야 정상 조회

### 프론트엔드 수정
- `ApprovalSubSection`에 `queryPrefix` prop 추가
- 섹션별 차별화된 메시지 생성:
  - 결재 미결: `내 결재 대기함의 '제목' (문서번호: N) 전자결재 문서 확인해줘`
  - 수신문서: `내 수신문서함의 '제목' (문서번호: N) 전자결재 문서 확인해줘`
  - 참조/열람: `내 참조함의 '제목' (문서번호: N) 전자결재 문서 확인해줘`
- doc_id 포함으로 단건 조회 정확도 향상 (제목 유사 문서 혼동 방지)

### 백엔드 프롬프트 수정
- **규칙 4번 세분화**: 기존 "다른 사람 문서 거절" → 3단계 접근 규칙으로 변경
  - 출처 명시 시 → 해당 뷰에서 즉시 조회
  - 출처 미명시 시 → 참조함 → 결재함 → 대기함 → 기안함 순서로 탐색
  - "타인의 결재함 직접 조회" 요청만 거절 (예: "김철수의 기안함 보여줘")
- **TOOL SELECTION GUIDE**: "내 참조함의", "내 결재 대기함의", "내 수신문서함의" 패턴 → doc_id 기반 즉시 단건 조회 가이드 추가

## 결정 사항 및 주의점
- 보안 자체는 유지됨: `prepare_tools()`의 employee_number 강제 주입으로 타인 결재함 SQL 접근은 여전히 불가
- LLM이 "거절 먼저, 검색 나중"이 아닌 "검색 먼저, 불가능하면 거절"로 동작하도록 프롬프트 순서를 변경한 것이 핵심
- doc_id가 메시지에 포함되므로 LLM이 정확한 단건 조회 SQL을 바로 생성 가능
