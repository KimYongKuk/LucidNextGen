# 2026-04-06 캘린더/예약 Worker 통합 테스트 및 버그 수정

## 개요
CalendarWorker와 ReservationWorker를 실제 환경에서 테스트하면서 발견된 다수의 버그를 수정하고, 기능을 보강하였다. 특히 LLM이 예약 현황 분석 시 시간대 교차를 잘못 판단하는 문제를 서버 측 계산으로 해결하였고, CalendarWorker에 회의실 예약 도구를 추가하여 일정+예약 동시 등록을 지원한다.

## 상세 내용

### ReservationWorker 수정

#### find_available_rooms 도구 추가
- LLM이 get_daily_reservations 결과에서 시간대 교차 분석을 잘못하는 문제 (Sonnet이 32건 예약 데이터에서 빈 회의실을 잘못 판정)
- 서버 측에서 시간 겹침을 코드로 직접 계산하여 ✅ 비어있음 / ❌ 예약됨 분류 후 반환
- 겹침 판정: `NOT (req_end <= r_start OR r_end <= req_start)` — 문자열 비교로 확정적 계산

#### create_reservation 충돌 사전 검증
- 예약 직전에 daily API를 재조회하여 해당 item_id의 시간 겹침 체크
- LLM이 빈 시간 분석을 틀려도 예약 전에 잡아냄
- 충돌 시 "이미 예약이 있습니다 + 충돌 예약 정보" 반환

### CalendarWorker 수정

#### get_my_calendars — user API 기반으로 변경
- 기존: `/api/calendar/feed` (서비스 계정 세션 기준) → 사용자 본인 캘린더 미반환
- 수정: `/api/calendar/user/{goUserId}/calendar` (사용자별 조회) → 본인 캘린더 정상 반환
- 관심 캘린더는 여전히 feed API 사용 (서비스 계정의 관심 캘린더)

#### get_calendar_events — URL 인코딩 수정
- `+09:00`의 `+`가 URL에서 공백으로 해석되는 문제
- `urllib.parse.quote`로 `%2B09:00`으로 인코딩

#### create_event — API 호환 수정
- `timeType`: `"normal"` → API 거부 → `"timed"` (비종일) / `"allday"` (종일)
- `type`: `"normal"` 필드 필수 — 누락 시 BindException
- 소유권 검증: feed API 대신 user API 기반으로 변경 (본인 캘린더 정확 판정)

#### attendee_names — 사내 참석자 이름 검색
- `attendee_names` 파라미터 추가 (콤마 구분 이름)
- `v_user_info_mapping` DB에서 이름 검색 → GO user ID + email 자동 연결
- `login_id@landf.co.kr`로 이메일 생성
- 동명이인 시 첫 번째 결과 사용 + 로그, 미발견 시 결과에 안내

#### 일정+회의실 동시 등록
- CalendarWorker에 `find_available_rooms`, `create_reservation` 도구 추가
- 워크플로우: 조직도 검색 → 캘린더 조회 → 빈 시간 분석 → 빈 회의실 → create_event + create_reservation
- `execute_org_chart_query` 도구 추가 (팀/파트 인원 조회용)

### 인텐트 분류 개선
- calendar + reservation 키워드 동시 매칭 시 calendar 우선 (CalendarWorker가 회의실 예약도 처리 가능)
- quick_classify와 LLM 프롬프트 양쪽에 규칙 적용

## 결정 사항 및 주의점
- CalendarWorker와 ReservationWorker는 분리 유지. 통합 시 시스템 프롬프트 비대화 + 도구 혼동 리스크
- CalendarWorker는 예약 도구 2개만 빌려씀 (find_available_rooms, create_reservation)
- 서비스 계정의 feed API는 사용자별 관심 캘린더를 정확히 반환하지 못함 — user API 필수
- a2a_streaming.py의 예약/캘린더 전문 로깅은 디버깅용 — 안정화 후 제거 가능
