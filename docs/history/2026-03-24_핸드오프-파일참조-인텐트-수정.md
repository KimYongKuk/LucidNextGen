# 2026-03-24 HANDOFF 마커 감지 실패 + 파일 참조 인텐트 오분류 수정

## 개요
사용자가 파일 업로드 후 "outline 위키용 markdown 변환"을 요청했으나, "outline" 키워드 때문에 OutlineWorker로 라우팅되고, OutlineWorker의 HANDOFF 요청도 감지되지 않아 응답 실패한 버그 수정.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| backend/app/agents/orchestrator.py | 수정 | `_extract_text()`에 list content 처리 추가 |
| backend/app/agents/intent_classifier.py | 수정 | Step 1에 업로드 파일 명시 참조 패턴 추가 |

## 상세 내용

### 버그 1: `_extract_text`가 list content 미처리 (HANDOFF 감지 실패)
- **원인**: Bedrock Converse API + LangChain AWS에서 `AIMessageChunk.content`가 `[{"type": "text", "text": "..."}]` 형태(list)로 올 수 있음
- **문제**: `_extract_text`가 `isinstance(content, str)`만 체크 → list content 무시 → `collected_text` 항상 빈 문자열
- **영향**: HANDOFF 마커 감지 실패, NO_RESULTS 마커 감지도 잠재적 실패
- **수정**: `a2a_streaming.py`와 동일한 list content 처리 로직 추가

### 버그 2: 파일 참조 메시지가 outline으로 오분류
- **원인**: "업로드한 파일을 outline 위키용 markdown으로 변환해줘" → "outline" 키워드 매칭 → `Intent.OUTLINE`
- **문제**: 사용자 의도는 파일 읽기+변환(USER_FILES)인데 OutlineWorker로 라우팅 (파일 읽기 도구 없음)
- **수정**: Step 1(100% 확실 패턴)에 업로드 파일 명시 참조 패턴 추가
  - `(업로드한 파일|올린 파일|첨부 파일|이 파일|그 파일|방금 파일)` + `has_files=True` → `USER_FILES`
  - 다른 도메인 키워드(outline, 위키 등)보다 우선 적용

## 결정 사항 및 주의점
- `_extract_text`의 list content 미처리는 `a2a_streaming.py`에서는 이미 처리하고 있었으나, orchestrator에서 누락된 것
- 파일 참조 패턴은 `has_files=True`일 때만 활성화되므로, 파일 없이 "파일"을 언급하는 일반 대화에는 영향 없음
