# 2026-04-07 워크스페이스 인텐트 오버라이드 제거

## 개요
워크스페이스 세션에서 `direct` 인텐트를 `user_files`로 강제 전환하던 오버라이드를 제거하고, 워크스페이스 컨텍스트를 모든 워커에 자연스럽게 주입하는 구조로 변경.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| backend/app/agents/intent_classifier.py | 수정 | 381행 `direct → user_files` 강제 오버라이드 제거 |
| backend/app/agents/workers/direct_worker.py | 수정 | `shared_tool_names`에 `search_workspace_docs` 추가 |

## 상세 내용

### 문제
워크스페이스(파일 첨부) 세션에서 "아키텍처 그려줘", "PDF로 만들어줘" 같은 시각화/생성 요청 시:
1. LLM이 `direct`로 분류 (차트/PDF 요청은 direct로 분류하는 것이 규칙)
2. `intent_classifier.py` 381행에서 `workspace_has_files and direct → user_files`로 강제 전환
3. `UserFilesWorker`가 처리 — 이 워커는 문서 **검색**이 본업이라 PDF 생성을 제대로 수행하지 못함
4. LLM이 도구를 호출하지 않거나, 호출해도 다운로드 링크를 포맷하지 않음

### 근본 원인
워크스페이스는 **환경**(파일, instructions)이지 특정 워커가 아님. 인텐트를 강제 변경하는 것은 워커의 역할을 뒤섞는 부작용을 만듦.

### 해결 방식
- `BaseWorker.build_system_prompt()`가 이미 모든 워커에 워크스페이스 컨텍스트(instructions, 메모리, 파일 목록, 검색 도구 안내)를 자동 주입하고 있음
- 인텐트 오버라이드 없이도 각 워커가 워크스페이스를 인식함
- `DirectResponseWorker`에 `search_workspace_docs`를 공유 도구로 추가하여, 워크스페이스 문서 검색도 가능하게 함

### 변경 전/후 라우팅
| 요청 (워크스페이스) | 변경 전 | 변경 후 |
|---|---|---|
| "안녕" | direct → **user_files** (강제) | direct → DirectResponseWorker (workspace context 주입) |
| "PDF 만들어줘" | direct → **user_files** (강제) | direct → DirectResponseWorker (shared tool로 PDF 생성) |
| "파일에서 찾아줘" | user_files | user_files (변경 없음) |
| "메일 확인" | mail | mail (변경 없음) |

## 결정 사항 및 주의점
- `DirectResponseWorker`에 `search_workspace_docs`를 **shared_tool**로 추가 (전용 도구가 아님). 워크스페이스 세션이 아닌 경우 도구가 로드되어도 사용하지 않음.
- `BaseWorker.build_system_prompt()`의 606행 파일 컨텍스트 주입이 핵심 — `has_files or workspace_has_files` 조건으로 모든 워커에 검색 도구 안내가 들어감.
- orchestrator의 `_WS_FIRST_ELIGIBLE` workspace-first 라우팅(Phase 1.8)은 별도 메커니즘으로 그대로 유지.
