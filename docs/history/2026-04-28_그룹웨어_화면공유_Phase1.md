# 2026-04-28 그룹웨어 위젯 화면 공유 Phase 1 (DOM 기반)

## 개요
다우오피스 그룹웨어에 임베드된 Lucid 위젯이 사용자가 보고 있는 부모 페이지를 자연스럽게 인지할 수 있도록 "현재 화면 공유" 기능 도입. Gemini 사이드패널의 "[탭이름] 공유 중" 칩과 동일한 UX. Phase 1은 DOM 텍스트 추출 기반(Vision/스크린샷은 Phase 2 예정).

## 상세 내용

### UX 모델
Gemini 사이드패널과 동일한 ambient 공유 모델:
- **기본 ON**: 위젯 mount 시 부모 페이지 메타정보(title/url) 받아 input 위에 chip 표시 → 모든 메시지에 page_context 자동 첨부
- **X 클릭 → OFF**: chip 사라지고 input 위에 작은 "화면 공유 켜기" 링크 노출. 다음 메시지부턴 page_context 미첨부
- **부모 페이지 navigate 감지**: URL이 바뀌면 자동으로 ON 상태로 복귀하며 chip 갱신

chip 본체 클릭은 무반응(X 버튼만 동작). 사용자 결정 사항(Gemini와 동일).

### 데이터 흐름
```
[부모 페이지 navigate (history.pushState/replaceState/popstate/hashchange)]
    ↓
[widget JS: postMessage → iframe {type: lucid-page-context, url, title}]
    ↓
[iframe (EmbedChat): chip 갱신, parentPage state 업데이트]
    ↓
[사용자가 메시지 전송]
    ↓
[useSimpleChat.sendMessage → getPageContext 콜백 호출]
    ↓
[EmbedChat: postMessage → widget {type: lucid-request-page-content, requestId}]
    ↓
[widget: extractPageContent() → postMessage 응답 {success, url, title, content}]
    ↓
[EmbedChat: Promise resolve → useSimpleChat이 page_context 본문 첨부]
    ↓
[백엔드 chat.py → stream_a2a_response → RequestContext.page_context]
    ↓
[base_worker.build_system_prompt에 "CURRENT PAGE CONTEXT" 섹션 주입]
    ↓
[LLM이 화면 관련 질문에 컨텍스트 활용]
```

### 위젯 JS 추가 컴포넌트

**`installNavigationHook()`**: SPA navigation 감지
- `history.pushState`/`replaceState` 원본 함수를 wrap → 호출 후 `setTimeout(send, 0)`로 다음 tick에 송신 (`document.title` 갱신 대기)
- `popstate`, `hashchange` 이벤트 리스너 추가
- `<title>` 변경 MutationObserver 추가 (SPA가 navigate 후 title을 비동기로 갱신하는 경우)
- `window.__lucidGwNavHookInstalled` 플래그로 중복 설치 방지

**`extractPageContent()`**: 부모 DOM 텍스트 + 표 마크다운 변환
- `document.body.cloneNode(true)`로 클론 (원본 DOM 변경 방지)
- 제외 selector: `script`, `style`, `noscript`, `svg`, `iframe`, `#lucid-gw-container`, `#lucid-sm-container`, `input[type="password"]`, `[autocomplete*="cc-"]`, `[aria-hidden="true"]`
- `<table>` 요소는 `tableToMarkdown()`으로 변환 후 본문에서 제거
- `innerText` 추출 후 공백/개행 정규화
- 8,000자 cap (`PAGE_CONTENT_MAX_CHARS`)

**postMessage 프로토콜**:
- 부모 → iframe: `{type: 'lucid-page-context', url, title}` (탭 정보, mount/navigate 시)
- 부모 → iframe: `{type: 'lucid-page-content', requestId, success, url, title, content}` (DOM 추출 응답)
- iframe → 부모: `{type: 'lucid-request-page-context'}` (mount 시 race 회피용 동기화 요청)
- iframe → 부모: `{type: 'lucid-request-page-content', requestId}` (메시지 전송 직전 본문 요청)

### EmbedChat 상태 관리

```typescript
const [parentPage, setParentPage] = useState<{url, title} | null>(null);
const [pageShareEnabled, setPageShareEnabled] = useState(true);
const pageShareEnabledRef = useRef(true);  // 콜백 내 stale closure 방지
const parentPageRef = useRef(null);          // 동상
```

`getPageContext` 콜백:
- `pageShareEnabled` false 또는 parentPage null → `null` 반환 (page_context 미첨부)
- postMessage 요청 → 응답 대기 (3초 타임아웃)
- 타임아웃 시 메타정보(title/url)만이라도 첨부 (content는 빈 문자열)

자동 재활성화: 부모 페이지 URL 변경 감지 시 `setPageShareEnabled(true)` — Gemini와 동일 동작.

### 백엔드 시스템 프롬프트 주입

`base_worker.build_system_prompt`에서 `context.page_context`가 있으면 다음 섹션을 prompt 최상단에 prepend:

```
## CURRENT PAGE CONTEXT — 사용자가 지금 보고 있는 화면

**페이지 제목**: {title}
**URL**: {url}

**페이지 본문 (텍스트 추출):**
```
{content}
```

**활용 규칙:**
1. 사용자가 현재 화면과 무관한 일반 질문을 하면 위 컨텍스트를 무시하세요.
2. 화면 관련 질문이면 제목/URL/본문을 근거로 답변하세요.
3. 본문에 없는 정보를 지어내지 마세요.
4. 본문이 잘려있을 수 있습니다(`[잘림]` 표시).
```

DirectWorker뿐 아니라 모든 워커가 자동으로 활용 가능 (예: ApprovalWorker가 결재 화면을 보면서 답변, MailWorker가 메일 화면 분석 등).

## 결정 사항 및 주의점

- **chip UX는 Gemini 방식 (옵션 a)**: chip 본체 클릭 무반응, X 버튼만 동작. 사용자 confirm.
- **DOM 추출 lazy**: 페이지 변경 시점에 본문 추출하지 않고 메시지 전송 시점에만 추출. 페이지 보기만 하다 위젯 안 쓰면 0 비용.
- **8,000자 cap**: 평균 응답 토큰 ~2-3K. iframe 중첩이 많은 페이지는 셸만 잡힐 수 있음 (Phase 2 또는 후속에서 same-origin iframe 재귀 추출 고려).
- **민감 입력 마스킹**: `input[type="password"]`, `[autocomplete="cc-*"]` selector로 추출에서 제외. 화면에 표시된 패스워드 placeholder도 자동 제거.
- **타임아웃 3초**: DOM 추출은 보통 50ms 이내인데, 거대한 DOM(>10MB)일 때 보호. 타임아웃 시 메타정보만 첨부해서 LLM이 최소한 "어떤 페이지인지"는 알 수 있게.
- **자동 재활성화 트리거는 URL 기반**: 같은 URL 안에서 SPA가 부분 갱신만 해도 chip은 그대로(이미 켜진 상태) → 추가 갱신만 발생. 다른 URL로 가면 강제 재활성화.
- **iframe 호스트만 동작**: `chatMode === "groupware_embed"` 가드. 본 웹 UI나 wiki embed에서는 chip 미표시.
- **Phase 2 예고**: 이 구조 위에 html2canvas 스크린샷 + Vision 모델 폴백 추가 예정. 트리거는 (a) 키워드 자동 분기 ("차트/그래프/이미지" 등) 또는 (b) DOM 결과가 부실할 때.
