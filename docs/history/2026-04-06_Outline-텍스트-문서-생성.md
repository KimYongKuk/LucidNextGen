# 2026-04-06 L&F Wiki 텍스트 직접 문서 생성/수정 도구 추가

## 개요
L&F Wiki MCP 서버에 `create_document`, `update_document` 도구를 추가하여, 파일 업로드 없이 마크다운 텍스트만으로 위키 문서를 생성/수정할 수 있게 했다.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| backend/app/mcp_servers/outline_server/server.py | 추가 | `create_document`, `update_document` MCP 도구 추가 |
| backend/app/agents/workers/outline_worker.py | 수정 | tool_names에 새 도구 등록, 시스템 프롬프트 워크플로우 가이드 추가, `create_document` 쓰기 권한 검증 |
| backend/mcp_config.json | 수정 | outline_server description 업데이트 |

## 상세 내용

### 문제
- 기존에는 `publish_file_to_wiki`만 존재하여, 반드시 업로드된 파일이 있어야만 위키 문서 생성 가능
- Outline API 자체는 `documents.create`로 텍스트 직접 생성을 지원하지만, MCP 도구로 노출되지 않았음

### 추가된 도구

#### create_document
- 파라미터: `title` (필수), `text` (필수, 마크다운), `collection_id` (필수), `parent_document_id` (선택), `publish` (기본 True)
- 내부: `_outline_request("documents.create", payload)` 호출
- 접근 제어: `_get_collection_access()`로 쓰기 권한 사전 검증

#### update_document
- 파라미터: `document_id` (필수), `text` (필수), `title` (선택), `append` (기본 False)
- `append=True`: 기존 본문 끝에 추가, `False`: 전체 교체
- 내부: `_outline_request("documents.update", payload)` 호출

### OutlineWorker 변경
- `tool_names`: `create_document`, `update_document` 추가
- `prepare_tools`: `create_document`에 대한 쓰기 권한 검증 분기 추가 (`_ACCESS_CONTROLLED_WRITE_TOOLS`)
- 시스템 프롬프트: 워크플로우 A(파일→위키), B(텍스트→위키), C(문서 수정) 3가지로 정리
- TOOL SELECTION GUIDE에 새 도구 매핑 추가

### BaseWorker 핸드오프 타겟 프롬프트 추가
- `is_handoff_target=True`일 때 "선행 데이터 수집 역할" 프롬프트 주입
- "할 수 있는 것만 하고, 못 하는 부분은 무시하라"는 지시
- 핸드오프 루프(WebSearchWorker가 검색 거부) 문제 해결

## 결정 사항 및 주의점
- `update_document`는 접근 제어 대상에 포함하지 않음 (document_id 기반이라 collection_id가 없음, Outline API 자체에서 권한 체크)
- `create_document`는 `publish_file_to_wiki`와 달리 `user_id` 주입이 불필요 (파일 탐색이 없으므로)
- 핸드오프 타겟 프롬프트는 모든 워커에 공통 적용 (BaseWorker 레벨)
