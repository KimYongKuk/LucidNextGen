# 2026-03-31 WORKS 서비스데스크 VOC 자동 등록

## 개요
IT 지원 챗봇 대화 중 사용자가 직접 해결하기 어려운 문제일 때, WORKS 서비스데스크(앱릿 934)에 VOC를 자동 등록하는 기능을 추가했다. SSO 내부 API를 통해 담당 부서원 자동 배정까지 포함.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| backend/app/mcp_servers/works_it_mcp_server.py | 수정 | `register_works_voc` MCP 도구 추가, SSO API 호출, 담당자 자동 배정 |
| backend/app/agents/workers/it_support_worker.py | 수정 | 도구 목록에 register_works_voc 추가, WORKS 등록 프롬프트, prepare_tools 보안 래핑 |
| backend/.env | 수정 | LFON_WORKS_TOKEN, LFON_BASE_URL, WORKS_APPLET_ID, LFON_SSO_USERNAME/PASSWORD 추가 |

## 상세 내용

### 플로우
```
사용자 IT 질문
  → ITSupportWorker 병렬 조회 (wiki, IT VOC, 임베딩 문서)
  → 답변 제공 (AI 분석 결과 포함)
  → 해결 어려우면: "서비스데스크(WORKS)에 등록해드릴 수 있습니다"
  → 사용자 승인 ("등록해줘")
  → register_works_voc 도구 호출
  → 시스템명 → 담당 부서 자동 결정 → 부서원 조회 → SSO API로 VOC 생성 + 담당자 배정
```

### register_works_voc MCP 도구
- **Parameters**: employee_number(사번), title(요약), details(상세), system_name(시스템명)
- **내부 로직**:
  1. `v_user_info_mapping`에서 요청자 정보 조회 (user_id, login_id, name, dept_id, dept_name)
  2. 시스템명 → 시스템 코드 → 담당 부서명 자동 매핑
  3. `v_user_info_mapping`에서 담당 부서 멤버 조회
  4. SSO 로그인 → `POST /api/works/applets/934/docs` 호출 (담당자 포함)
  5. SSO 실패 시 OpenAPI 폴백 (담당자 없이 생성)

### 두 가지 API 방식
| | SSO API (Primary) | OpenAPI (Fallback) |
|---|---|---|
| 엔드포인트 | `/api/works/applets/934/docs` | `/openapi/works/applets/doc` |
| 인증 | SSO 쿠키 (로그인 필요) | JWT 토큰 |
| 담당자 지정 | `_8uzx0pk1u` 필드로 가능 | 불가 |
| 회사 필드 | `_awrf64ysv` = "0" | `_affiliation` = "엘앤에프" |
| 시스템 필드 | `_o9nnudfsi` = 코드번호("1") | `_o9nnudfsi` = 텍스트("LFON") |

### 보안
- `prepare_tools()`에서 `register_works_voc`의 `employee_number`를 SSO 인증 사번으로 강제 주입
- `object.__setattr__` + `_unwrapped_ainvoke` 패턴 (approval_worker와 동일)
- SSO 쿠키는 프로세스 수명 캐싱, 세션 만료 시 자동 재로그인

### 환경변수
| 변수 | 설명 |
|------|------|
| LFON_WORKS_TOKEN | LFON OpenAPI JWT 토큰 (폴백용) |
| LFON_BASE_URL | LFON 그룹웨어 URL (기본: https://lfon.landf.co.kr) |
| WORKS_APPLET_ID | WORKS 앱릿 ID (운영: 934) |
| LFON_SSO_USERNAME | SSO 로그인 ID (담당자 지정용) |
| LFON_SSO_PASSWORD | SSO 로그인 비밀번호 |

### 상태 전환 (접수 → 담당자지정)
VOC 생성 후 자동으로 2단계 상태 전환:
1. 접수: `PUT /api/works/applets/934/docs/{docId}/actions/2545` (body: `{}`)
2. 담당자지정: `PUT /api/works/applets/934/docs/{docId}/actions/2619` (body: `{}`)
- doc_id는 SSO 생성 응답에서 추출
- SSO 쿠키 재사용

### 인텐트 분류 Follow-up 규칙
`intent_classifier.py`에 follow-up 규칙 추가:
- quick_classify에서 새 도메인 키워드가 매칭되지 않으면 + previous_intent 존재 → 이전 인텐트 유지
- "등록해줘", "진행해", "ㅇㅇ" 등 어떤 follow-up 메시지든 길이/키워드 무관하게 동작
- 명시적 도메인 키워드(메일, 결재, 게시판 등)가 있을 때만 주제 전환

## 결정 사항 및 주의점
- SSO API를 1차로 사용하여 담당자 배정 포함, 실패 시 OpenAPI 폴백 (담당자 없이)
- SSO 계정은 서비스 계정 사용 권장 (개인 계정 비밀번호 변경 시 장애)
- `v_user_info_mapping` 뷰에 부서 멤버가 없는 경우 담당자 빈 배열로 전달 (수동 배정 필요)
- 앱릿 934의 SSO 필드명(`_awrf64ysv`)과 OpenAPI 연동항목 파라미터명(`_affiliation`)이 다름 주의
- `register_works_voc`의 employee_number는 default="auto"로 설정 — LLM이 사번 없이도 호출 가능, prepare_tools()가 실제 사번 주입
- 향후 자동 처리 가능 케이스(비밀번호 초기화, 메일 용량 등) API 확보 시 MCP 도구 추가로 확장 예정
