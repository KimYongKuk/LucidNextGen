# 2026-03-30 MCP 캐시 리프레시 행(hang) 수정

## 개요
MCP 도구 캐시 리프레시 시 서버 서브프로세스 동시 스폰 실패로 인해 채팅 요청이 무한 대기(행)에 빠지는 문제를 수정.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| backend/mcp_config.json | 수정 | excel_server command를 절대 경로로 변경 |
| backend/app/adapters/mcp_adapter.py | 수정 | 영구실패 블랙리스트 + 개별 서버 타임아웃 추가 |

## 상세 내용

### 근본 원인
1. `excel_server`의 `command: "excel-mcp-server"`가 NSSM 서비스(LocalSystem 계정)의 PATH에 없어서 매번 `FileNotFoundError` 발생
2. 실패 시 캐시 TTL을 1시간 → 60초로 단축하는 로직이 **영구 실패도 일시 실패로 취급**
3. 매 60초마다 15개 MCP stdio 서브프로세스를 동시 스폰 → Windows 리소스 경쟁 → 일부 서버 ExceptionGroup으로 즉시 실패, 일부는 initialize() 단계에서 무한 대기
4. `asyncio.gather()`가 대기 중인 서버 때문에 반환 불가 → 전체 요청 행

### 수정 1: excel_server 절대 경로
- `"excel-mcp-server"` → `"C:\\...\\Scripts\\excel-mcp-server.exe"` 절대 경로
- LocalSystem 계정에서도 바이너리를 찾을 수 있도록 함

### 수정 2: MCP Adapter 캐시 로직 개선
- **영구 실패 블랙리스트**: `FileNotFoundError`, `PermissionError` 등은 `_blacklisted_servers`에 등록, 프로세스 재시작 전까지 재시도 안 함
- **ExceptionGroup 언래핑**: MCP 클라이언트가 ExceptionGroup으로 감싸는 에러를 풀어서 실제 에러 타입 판별
- **개별 서버 타임아웃 (10초)**: `asyncio.wait_for()`로 hang 방지, 하나가 멈춰도 전체 영향 없음
- **TTL 단축 조건 변경**: 블랙리스트 제외한 일시적 실패만 TTL 단축 트리거

## 결정 사항 및 주의점
- 블랙리스트는 프로세스 수명(클래스 변수) → 서버 재시작 시 자동 초기화
- 타임아웃 10초는 Windows에서 Python 서브프로세스 스폰 + MCP initialize에 충분한 시간 (정상 시 ~2초)
- `tavily-mcp.cmd`도 npm 경로이므로 향후 동일 문제 가능성 있음 (현재는 시스템 PATH에 포함되어 정상)
