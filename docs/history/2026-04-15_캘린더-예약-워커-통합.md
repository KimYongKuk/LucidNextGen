# 2026-04-15 캘린더·예약 워커 통합

## 개요
CalendarWorker와 ReservationWorker를 하나의 CalendarWorker로 통합.
"내 일정 확인 → 회의실 예약" 같은 크로스 도메인 요청을 한 워커에서 처리 가능하도록 함.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| `backend/app/agents/workers/calendar_worker.py` | 수정 | 예약 도구 5개 추가, 시스템 프롬프트에 예약 워크플로우·사업장 목록 통합, prepare_tools에 예약 보안 래핑 추가 |
| `backend/app/agents/state.py` | 수정 | Intent.RESERVATION → "CalendarWorker"로 라우팅 변경 |

## 상세 내용

### 통합 전후 비교
| | 통합 전 | 통합 후 |
|---|---|---|
| CalendarWorker 도구 | 캘린더 7 + 조직도 1 + 예약 2 = 10개 | 캘린더 7 + 조직도 1 + 예약 7 = 15개 |
| ReservationWorker | 예약 7개 (별도 워커) | 미사용 (파일 유지, 라우팅 제거) |
| Intent.RESERVATION | → ReservationWorker | → CalendarWorker |
| Intent.CALENDAR | → CalendarWorker | → CalendarWorker (동일) |

### 추가된 예약 도구 (5개)
- `get_sites`: 사업장 목록 조회
- `get_rooms`: 사업장별 회의실 목록
- `get_daily_reservations`: 일별 예약 현황
- `get_my_reservations`: 내 예약 목록
- `cancel_reservation`: 예약 취소

### prepare_tools 보안 래핑 확장
- 기존: 캘린더 7개 + create_reservation
- 추가: get_my_reservations, cancel_reservation (employee_number 강제 주입)

### max_agent_steps
- 20 → 24 (일정 조회 + 빈 회의실 검색 + 예약 등록까지 충분한 단계)

### ReservationWorker 처리
- 파일 유지 (rollback 안전망)
- state.py에서 라우팅만 제거
- __init__.py import 유지

## 결정 사항 및 주의점
- intent_classifier의 reservation 키워드/LLM 분류는 그대로 유지 — Intent.RESERVATION이 CalendarWorker로 라우팅됨
- groupware_embed 모드에서도 CALENDAR, RESERVATION 둘 다 CalendarWorker로 처리
- 시스템 프롬프트가 길어졌으나 Sonnet 기준 허용 범위 내
