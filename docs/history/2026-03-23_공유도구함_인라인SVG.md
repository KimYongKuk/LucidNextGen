## 개요
VisualizationWorker를 제거하고, 차트/PDF/DOCX 도구를 공유 도구함(shared_tool_names)으로 분리하여 여러 에이전트가 직접 사용할 수 있도록 변경. 동시에 인라인 SVG 스트리밍 렌더링을 도입하여 모든 에이전트가 다이어그램/인포그래픽을 텍스트 응답에 자연스럽게 포함할 수 있도록 함.

## 상세 내용

### 시각화 3계층 구조
1. **인라인 SVG** (모든 에이전트): LLM이 `<svg>` 태그를 텍스트에 직접 출력 → 프론트엔드가 실시간 렌더링. 플로우차트, 인포그래픽, 타임라인 등 Recharts로 커버 불가한 시각화 담당.
2. **Recharts 차트** (공유 도구): MCP 차트 도구 → JSON 반환 + PNG 자동 저장 → Recharts 인터랙티브 렌더링. 라인/막대/파이/복합 등 데이터 차트 담당.
3. **파일 산출물** (공유 도구): PDF/DOCX MCP 도구. 다운로드 가능한 문서 생성.

### 공유 도구함 (shared_tool_names)
- BaseWorker에 `shared_tool_names` 프로퍼티 추가 (기본: 빈 리스트)
- `filter_tools()`가 `tool_names + shared_tool_names`를 병합하여 필터링
- opt-in 대상: DirectWorker, WebSearchWorker, UserFilesWorker, CorpRAGWorker
- 공유 도구: create_line/bar/pie/multi_chart, create_document_pdf, create_table_spec_pdf, create_document_docx

### DirectWorker 조건부 ReAct 전환
- 기존: 도구 0개 → 직접 LLM 스트리밍 (create_react_agent 미사용)
- 변경: 공유 도구 사용 가능 → BaseWorker의 ReAct 루프 위임, 도구 없으면 기존 직접 스트리밍 유지

### 차트 MCP 통합 모드
- `output_mode` (display/file) 파라미터 제거
- 항상 Recharts JSON (프론트엔드 렌더링) + PNG 자동 저장 (PDF 임베딩용)
- LLM의 모드 선택 부담 제거

### 파일 산출물 출력 규칙
- 시스템 프롬프트에 "PDF/DOCX 도구 호출 후 응답에 `파일명: xxx.pdf` 형식 포함" 규칙 추가
- 프론트엔드 `processPDFContent`/`processDocxContent`가 이 패턴을 감지하여 다운로드 링크 자동 생성
- workerName 패턴: `visualization|direct|websearch|userfiles|corprag` 모두 허용

### SVG 스트리밍 렌더링 (svg-stream-block.tsx)
- **div + dangerouslySetInnerHTML 방식**: iframe 제거 — 로드 타이밍, 빈 화면, 배경 불일치 문제 해결
- **DOMPurify 동적 로드**: `useEffect` 내 `import('dompurify')` → SSR 에러 방지, 정제 완료 후 state 반영
- **SvgRenderer memo**: 내부 SVG div를 `React.memo`로 감싸 동일 HTML이면 DOM 재생성 방지 (떨림 방지)
- **투명 배경**: 채팅 UI 다크 테마와 자연스럽게 통합
- **오버레이 메뉴**: 헤더 바 없이 hover 시 `...` 버튼 → Copy SVG / Download / Expand 드롭다운

### HTML 위젯 렌더링 (html-widget-block.tsx) — 신규
- **iframe sandbox 격리**: `<style>` 충돌 방지, `<script>` 허용 (인터랙티브 가능)
- **스트리밍**: contentDocument.body.innerHTML RAF 업데이트 (iframe shell 고정, body만 갱신)
- **실시간 높이 갱신**: 콘텐츠 주입 직후 `scrollHeight` 측정 → iframe이 콘텐츠 성장에 따라 실시간으로 늘어남 (500ms 폴링 방식 제거)
- **CSS 변수 기반 테마 대응**: iframe shell에 `--w-bg`, `--w-text`, `--w-card` 등 CSS 변수 정의 → `useTheme()` 감지 → `#theme-vars` style 태그만 갱신 (iframe 재로드 없이 즉시 전환). iframe 최초 로드 시점(`handleIframeLoad`)에서도 현재 테마 즉시 적용.
- **LLM CSS 변수 사용**: 시스템 프롬프트에서 `var(--w-card)`, `var(--w-positive)` 등 CSS 변수 사용 지시 (하드코딩 색상 금지) → 라이트/다크 테마 자동 대응
- **감지 마커**: LLM이 `<lucid-html>...</lucid-html>` 래퍼로 출력 → splitVisualBlocks가 감지
- **용도**: 비교 대시보드, 카드 그리드, 테이블+바 혼합 등 복합 레이아웃

### 시각화 3모드 체계 (시스템 프롬프트)
- **축이 있으면 → Recharts**: 수치 데이터 5개+ 포인트, 추이/분포 차트
- **화살표가 있으면 → SVG**: 프로세스/구조/관계 도식
- **행/열이 있으면 → HTML 위젯**: 비교표, 카드 그리드, 대시보드형 레이아웃
- 회색지대: 항목 5개 이하 단순 바 → HTML, 5개 초과 → Recharts
- HTML 위젯은 `<style>` 먼저 → HTML → `<script>` 마지막 순서로 스트리밍 친화적 구성

### 스트리밍 마커 필터
- `<!--HANDOFF:...-->`, `<!--NO_RESULTS-->` 마커가 a2a_streaming.py에서 스트리밍 시점에 제거됨
- 기존에는 DB 저장 시점에서만 제거 → UI에 노출되던 버그 수정

## 결정 사항 및 주의점
- `visualization_worker.py`는 삭제하지 않고 파일로 보존 (참조용). __init__.py에서만 제거.
- `svg_generator/server.py`는 mcp_config.json에서 비활성화. 파일은 보존.
- PPTWorker, XlsxWorker는 복잡한 전용 저작 환경이므로 공유 도구로 분리하지 않음.
- 인텐트 분류에서 "차트 만들어줘" 등은 이제 direct로 분류됨 → DirectWorker가 공유 도구로 직접 처리.
- SVG는 토큰 소비가 큼 (800~4,000 토큰/건). 데이터 차트는 Recharts 도구 사용 권장.
- HTML 위젯은 항상 다크 테마 기본값 → CSS 변수로 라이트 테마도 대응 (LLM이 CSS 변수 사용 시).
- SVG는 LLM이 색상을 하드코딩하므로 테마 대응 불가 (다크 전용). HTML 위젯만 테마 전환 지원.
