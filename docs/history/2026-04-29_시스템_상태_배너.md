# 2026-04-29 시스템 상태 배너 (Bedrock Throttling 사용자 알림)

## 개요

AWS Bedrock 일일 토큰 한도 초과 등으로 응답 처리가 지연될 때, 프론트 공통 레이아웃 최상단에 안내 배너를 노출. 트리거는 "직전 5분 이내 throttling 발생 여부"이며, 자연스럽게 자동 해제된다.

배경: 2026-04-28 13:39 `us.anthropic.* → global.anthropic.*` 폴백, 19:03 무렵 `global.*`마저 일일 한도 소진. 사용자는 "왜 응답이 안 나오지?" 만 알 수 있어 일관된 안내가 필요.

## 상세 내용

### 트리거 조건

`is_fallback_active = True` 만으로는 불충분. 오늘(04-28) 13:39~19:00 동안은 `global.*`로 정상 응답되고 있었기 때문에 그 시간 내내 배너를 띄우면 거짓 경보다.

→ **"직전 5분 이내 throttling 발생"** 으로 한정. 양쪽 프로필 모두 한도 소진된 상황 + 일시적 throttle만 정확히 포착.

### 호출 지점

`is_throttling_error(e)`로 식별 가능한 LLM 호출 실패 지점에 모두 `record_throttling()` 추가:

1. `base_worker.py:1440` — 워커 응답 LLM (us↔global swap retry 직전)
2. `planner.py:391` — Planner LLM (Sonnet, intent 결정 단계)
3. `intent_classifier.py:559` — 인텐트 분류기 LLM (Haiku, planner fallback 시)

`bedrock_service.py` / `openapi_bedrock_service.py`는 외부 API 호환 레이어라 사용자 채팅 경로와 분리되어 이번 범위 제외.

### API

```
GET /api/v1/system/status
→ 200 OK
{
  "degraded": false,
  "message": "",
  "since": null
}

# Throttling 5분 이내
{
  "degraded": true,
  "message": "현재 루시드AI의 처리 작업량이 많아 지연될 수 있습니다.",
  "since": "2026-04-28T19:03:51+09:00"
}
```

- 인증 불필요 (공개 상태값)
- `RegionFallbackManager` 메모리 값만 읽으므로 응답 1ms 이내
- `since`는 ISO 8601(KST) — 향후 관리자 페이지에서 활용 가능

### 프론트 동작

`frontend/components/system-status-banner.tsx`:
- `getApiUrl()` 사용 (운영: nginx same-origin, 개발: `NEXT_PUBLIC_BACKEND_URL`)
- `setTimeout` 기반 60초 폴링 (`POLL_INTERVAL_MS`)
- `degraded=false`일 때 컴포넌트 자체가 `null` 반환 → 레이아웃 영향 0
- 노출 시 `sticky top-0 z-50` 호박색 띠 + `AlertTriangle` 아이콘
- 다크 모드 대응 (`dark:bg-amber-950/40`)
- `role="status"` `aria-live="polite"` 접근성

루트 레이아웃에 삽입했으므로 메인 챗 / 그룹웨어 위젯(`/embed/gw`) / 위키 임베드(`/embed/wiki`) / 관리자 페이지 등 모든 경로에 자동 적용.

## 결정 사항 및 주의점

- **5분 윈도우**: 너무 길면 복구 후에도 배너 잔존, 너무 짧으면 깜빡거림. 환경변수 `DEGRADED_WINDOW_SECONDS`로 조정 가능
- **메시지 톤**: "AWS 한도 초과" 같은 내부 사정은 노출 안 함. "처리 작업량이 많아 지연" 정도가 사용자 친화적
- **메모리 only**: `_last_throttle_at`은 파일에 저장 안 함. 재시작 시 초기화돼도 다음 throttle 발생 시 즉시 재기록되므로 문제 없음
- **Blue/Green**: 두 슬롯이 각자 메모리 카운터를 가짐. 비활성 슬롯도 동일하게 동작하지만 트래픽이 없어 자연 OFF — 의도된 동작
- **위젯 영향**: 그룹웨어 위젯에도 같이 보임. 화면이 작은 환경이라 2줄 정도 차지하지만, 사용자 인지가 목적이므로 의도적 노출
- **확장성**: 추후 점검/유지보수 공지 등 다른 사유로도 `degraded=true` 만들고 싶으면 `region_mgr` 외 다른 source를 OR로 합치면 됨
