# 2026-03-10 조직도 계층 조회 (Hierarchical Org Chart)

## 개요
조직도 조회 시 상위 부서를 검색하면 하위 부서까지 포함하여 전체 조직도를 조회하도록 개선. 기존에는 `부서 ILIKE '%키워드%'`로만 검색하여 직속 부서만 반환되었으나, `부서ID` + `부서경로` 컬럼을 활용한 2단계 계층 조회 패턴을 도입.

## 상세 내용

### 문제
- `v_org_chart` 뷰에 `부서경로`는 있었으나 `부서ID`가 없었음
- `부서경로`는 부모까지의 경로만 포함하고 자기 자신의 ID를 포함하지 않음
- 같은 부모 밑의 형제 부서들이 동일 경로를 공유 → 형제 구분 불가
- 예: 인사팀(ID=50)과 총무팀(ID=52) 모두 경로=10:12:199

### 부서경로 + 부서ID 구조
```
10:12:199         ← HR부문 (부모 경로)
  ├── 인사팀 (ID=50, 경로=10:12:199) ← 형제
  ├── 총무팀 (ID=52, 경로=10:12:199) ← 형제
  └── 하위:
      ├── 10:12:199:50 → 인사 하위 (교육, 급여/근태 등)
      └── 10:12:199:52 → 총무 하위 (공장총무파트 등)
```

### 계층 조회 패턴 (부서ID 기반)
1. **Step 1**: 부서명으로 `부서ID` + `부서경로` 조회
   ```sql
   SELECT DISTINCT 부서ID, 부서, 부서경로 FROM v_org_chart WHERE 부서 ILIKE '%인사팀%'
   → 부서ID=50, 경로=10:12:199
   ```

2. **Step 2**: 부서ID 기반 정확한 계층 조회
   ```sql
   WHERE 부서경로 LIKE '10:12:199:50:%'   -- 하위 부서
      OR 부서경로 = '10:12:199:50'          -- 직속 자식 레벨
      OR 부서ID = 50                        -- 자기 자신
   ```

### 검증 결과
| 부서 | 결과 | 형제 분리 |
|------|------|-----------|
| 인사팀 | 인사팀 + 교육/급여/인력운영/조직문화/평가 | ✅ 총무 없음 |
| IT센터 | IT센터 + IT운영팀/IT인프라팀 + DA/DX/ERP파트 | ✅ 보안팀 없음 |
| 총무팀 | 총무팀 + 공장총무/본사총무/소통관리/시설관리 | ✅ 인사 없음 |

## 결정 사항 및 주의점
- DBA에게 `v_org_chart` 뷰에 `d.id AS "부서ID"` 컬럼 추가 요청 필요 (완료)
- `부서ID`는 ALLOWED_COLUMNS에 포함되어 쿼리 결과로 반환 가능하지만, 사용자 응답에는 포함하지 않음 (내부 키)
- LLM이 2단계 tool call 필요 (기본 agent step 20이므로 충분)
