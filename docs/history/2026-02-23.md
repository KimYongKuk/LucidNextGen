# 2026-02-23 업데이트 내역

## 1. 전자결재 조회 모듈 (신규)

사용자의 전자결재 문서를 자연어로 조회하는 기능 추가.

### 주요 파일
- `backend/app/agents/workers/approval_worker.py` - ApprovalWorker (Sonnet)
- `backend/app/mcp_servers/approval_mcp_server.py` - 전자결재 MCP 서버 (FastMCP, stdio)

### 기능
- **결재 대기함 조회**: 미결재 문서 확인
- **기안함 조회**: 본인이 기안한 문서 목록
- **결재 완료함**: 처리 완료된 결재 문서
- **참조함**: 참조로 수신된 문서
- **부서 문서함**: 부서 수신/발신 문서
- **결재 병목 분석**: 결재선에서 지연되고 있는 건 확인

### 아키텍처
- **Dual-mode**: 사전 조회(prefetch) 성공 시 시스템 프롬프트에 사용자 정보 주입, 실패 시 LLM이 `get_user_approval_info` 도구 직접 호출
- **보안**: `prepare_tools()`에서 `object.__setattr__` 패턴으로 employee_number 강제 주입
- **DB**: PostgreSQL TIMS DB의 `v_appr_*` VIEW 9종 활용
- **스키마 가이드**: `backend/metadata/MCP_GW_APPR.md` 로드하여 LLM 시스템 프롬프트에 주입
- **DateStyle**: `asyncpg.create_pool` init 콜백에서 `SET DateStyle = 'ISO, YMD'` 자동 설정
- **On/Off**: `APPROVAL_WORKER_ENABLED` 환경변수로 즉시 비활성화 가능

### 도구
| 도구 | 설명 |
|------|------|
| `get_user_approval_info` | 사용자 결재 기본 정보 조회 (login_id, dept_id) |
| `execute_approval_query` | 범용 SQL 쿼리 실행 (IT VOC와 동일 패턴) |

## 2. 메일 조회 모듈 (신규)

사용자의 사내 메일함을 자연어로 조회하는 기능 추가.

### 주요 파일
- `backend/app/agents/workers/mail_worker.py` - MailWorker (Sonnet)
- `backend/app/mcp_servers/mail_server/server.py` - 메일 MCP 서버 (FastMCP, stdio)

### 기능
- **받은편지함/보낸편지함 조회**: `get_inbox_mail`, `get_sent_mail`
- **메일 검색**: `search_mail` (키워드 기반)
- **안 읽은 메일 조회**: `get_unread_mail`
- **메일함 목록**: `get_mail_folders`

### 아키텍처
- 사번(user_id) → PostgreSQL VIEW(`v_mail_user_mapping`) → message_store → JSP 엔드포인트 호출
- **보안**: 사번은 SSO 세션에서 자동 주입 (사용자 변경 불가)
- **성능**: message_store 경로 프로세스 수명 캐싱
- **On/Off**: `MAIL_WORKER_ENABLED` 환경변수로 즉시 비활성화 가능

## 3. 인텐트 분류기 개선

### 변경 파일
- `backend/app/agents/intent_classifier.py`
- `backend/app/agents/state.py`

### 변경 내용
- **Intent 추가**: `Intent.MAIL`, `Intent.APPROVAL` Enum 및 Worker 매핑 추가
- **Quick classify 확장**: 메일/전자결재 키워드 정규식 패턴 추가
- **워크스페이스-인식 라우팅(Workspace-Aware Routing)**: 워크스페이스 이름/설명/파일명을 기반으로 사용자 질문이 워크스페이스 주제와 관련 있는지 판단 후 인텐트 결정
  - 예: 워크스페이스 "분기 실적" + 질문 "안 읽은 메일" → `mail` (워크스페이스 주제와 무관)
  - 예: 워크스페이스 "메일 관리" + 질문 "메일 현황 정리" → `user_files` (워크스페이스 파일 우선 검색)
- **LLM 분류기 프롬프트**: 워크스페이스 메타데이터(이름, 설명, 파일명 목록) 전달하여 정확도 향상
- **"결산" vs "결재" 구분**: "결산"(settlement)은 `acct_support`, "결재"(approval)은 `approval`로 명확 분리
