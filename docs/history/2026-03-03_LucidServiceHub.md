# 2026-03-03 Lucid AI 서비스 허브 구상

## 개요
사내 각 부서가 자체적으로 구축한 자동화/AI 서비스를 Lucid 플랫폼에 통합하여, 전 직원이 자연어로 실행·조회·검증할 수 있는 "사내 AI 서비스 허브" 구조에 대한 브레인스토밍.

## 핵심 컨셉

> **"만드는 건 각자, 쓰는 건 루시드에서."**

- Lucid는 서비스를 **만드는 곳이 아니라, 붙이고 쓰는 곳**
- 최상위 플랫폼은 Lucid이며, 에이전트 플랫폼 등 다른 시스템도 Lucid 아래에 위치
- 당장 구현은 아니고, RPA/VDI 자동화가 활성화되는 시점에 확장 예정

## Lucid의 4가지 역할

| 역할 | 설명 |
|------|------|
| **프론트도어** | 사용자는 루시드만 알면 됨 |
| **레지스트리** | 어떤 서비스가 있는지 통합 관리 |
| **라우터** | 사용자 요청 → 적합한 서비스 자동 연결 |
| **통합 UX** | 어떤 도구로 만들었든 동일한 자연어 대화 경험 |

## 서비스 생성 환경

각 부서가 자유롭게 선택하여 구현:

| 도구 | 대상 | 특징 |
|------|------|------|
| AI Agent Platform | 현업 (노코드/바이브코딩) | AI 에이전트 구축 |
| n8n / Make / Zapier | 현업 | 워크플로우 자동화 |
| PAD / VBA | 현업 | 기존 매크로 활용 |
| Python / MCP 직접 개발 | IT/개발팀 | 고도화 서비스 |

- 노코드 기반 AI Agent 플랫폼이 별도로 도입 예정
- 이 에이전트 플랫폼도 VDI와 통신하여 워크플로우를 실행하게 됨
- 에이전트 플랫폼 역시 Lucid 아래에 있는 하나의 서비스 채널

## 서비스 3가지 유형

### ① Agent형
- AI 에이전트가 대화형으로 응답
- 예시: "유가 분석 에이전트", "채용 FAQ 봇"
- 질문 → 응답 → 후속 질문 가능

### ② Workspace형
- 부서 지식베이스를 전사에 공유
- 예시: "K-IFRS 회계기준 가이드", "인사규정 안내"
- 문서 기반 RAG 검색 + 자연어 요약
- **현재 루시드 구조에서 가장 빠르게 실현 가능** (워크스페이스 공유 기능 추가만 하면 됨)

### ③ Trigger형
- 기존 매크로/워크플로우를 원격 실행
- 예시: "SAP 원가 데이터 추출", "월간 매출 집계", "외화자금 분석"
- 실행 → 대기 → 완료 시 산출물 전달
- 비동기 처리 필요 ("완료되면 알려드릴게요" 패턴)

## 전체 아키텍처

```
              Lucid (최상위 통합 플랫폼)
                │
         ┌──────┼──────────┬─────────────┐
         │      │          │             │
         ▼      ▼          ▼             ▼
      Agent   노코드      VDI 데몬       직접 연동
      플랫폼  Agent       (어댑터)       (REST API)
              플랫폼      │
                          ├─ PAD/RPA 실행
                          ├─ VBA 매크로
                          ├─ Python 스크립트
                          └─ n8n 워크플로우
```

## VDI 데몬 (어댑터 레이어)

- PAD, VBA 등 자체 HTTP 서버를 띄울 수 없는 도구를 위한 어댑터
- 각 VDI에 경량 에이전트(데몬)가 상주
- REST API로 외부 요청 수신 → 로컬 실행 → REST로 결과 반환
- **Lucid 전용이 아님**: 에이전트 플랫폼 등 다른 시스템도 동일한 데몬을 통해 VDI와 통신
- 따라서 이 데몬은 **사내 VDI 자동화 인프라의 공통 레이어**

```
노코드 Agent 플랫폼 ──→ VDI 데몬 ──→ 로컬 스크립트/매크로 실행
Lucid               ──→ VDI 데몬 ──→ 로컬 스크립트/매크로 실행
기타 시스템          ──→ VDI 데몬 ──→ 로컬 스크립트/매크로 실행
```

## 표준 서비스 스펙 (REST API 계약)

어떤 도구로 만들었든 아래 3개 API만 맞추면 Lucid에 연결 가능:

```
1. POST /execute        — 서비스 실행 요청 (파라미터 전달)
2. GET  /status/{job_id} — 진행 중 / 완료 / 실패
3. GET  /result/{job_id} — 텍스트 응답 또는 파일 경로
```

서비스 등록 메타 정보:
```yaml
service:
  name: "서비스명"
  description: "설명 (LLM 라우팅에 사용)"
  type: agent | workspace | trigger
  owner: "등록자 (부서, 이름)"
  endpoint: "http://vdi-xxx:9000"
  access:
    departments: [재무, 기획]  # 또는 all
    level: 직급 제한 등
  inputs: 입력 파라미터 정의
  outputs: 산출물 형식 (text, file, json)
```

## 실제 적용 시나리오: 외화자금계획

AS-IS: 담당자가 수동 경제지표 검색 → 환율 분석 → 보고서 작성 (연간 74,880분)

TO-BE:
1. 외화자금 분석 서비스를 별도 플랫폼에서 구축
2. Lucid에 Trigger형으로 등록
3. "금주 외화자금 분석 실행해줘" → 자동 실행 → 보고서 다운로드
4. "지난주 대비 환율 리스크 변동 요약해줘" → 산출물 기반 자연어 검증
5. 재무팀뿐 아니라 기획/영업도 동일하게 사용 가능

Lucid의 두 가지 모드:
- **실행**: 서비스를 트리거하고 결과 받아옴
- **조회**: 이미 나온 산출물에 대해 자연어 질의 (파일 직접 안 열어도 됨)

## 시나리오별 유효성 평가

| 시나리오 | 유형 | 유효성 | 비고 |
|---------|------|--------|------|
| 유가 분석 에이전트 (재무) | Agent | 매우 유효 | 데이터 수집+분석, AI Agent 최적 사용처 |
| 회계기준 가이드 (회계) | Workspace | 매우 유효 | 현재 구조에서 바로 가능 |
| 채용 FAQ 봇 (인사) | Agent | 유효 | 노코드 빌더 전형적 성공 사례 |
| SAP 원가 추출 (구매) | Trigger | 유효 (주의) | VDI 상시 가동, 비동기 처리, 큐잉 필요 |
| 리드 알림 워크플로우 (영업) | Trigger | 검토 필요 | 능동 알림(Push)까지 스코프에 넣을지 결정 |

## 고려사항 (향후 설계 시)

### 보안/권한
- 서비스별 접근 가능 부서/직급 설정
- 데이터 민감도 분류
- 감사 추적 (누가 어떤 서비스를 언제 호출했는지)

### 품질 관리
- 등록 전 승인 절차 (부서장 or 관리자)
- 등록자 = 책임자 명시
- 사용자 피드백/신고 기능

### 운영 관리
- 주기적 헬스체크 (서비스 생존 확인)
- N개월 미사용 시 자동 비활성화
- 등록자 퇴사 시 인수인계 프로세스

### Trigger형 특수 고려
- VDI 꺼져있을 때: "현재 서비스 불가" 안내
- 실행 시간이 긴 경우: 비동기 + "완료되면 알려드릴게요"
- 동시 요청: 큐잉 필요
- SAP 세션 등 전제 조건: 서비스 계정 필요 여부

## 업계 검증

이 구조는 엔터프라이즈 자동화 업계의 표준 패턴:

| 제품 | 중앙 | 원격 | 통신 |
|------|------|------|------|
| UiPath | Orchestrator | Robot (VDI) | REST |
| Automation Anywhere | Control Room | Bot Agent (VDI) | REST |
| Blue Prism | Control Room | Runtime Resource (VDI) | REST |
| Jenkins | Master | Agent | REST |
| Kubernetes | Control Plane | Kubelet | REST |

Lucid 차별점:
- 실행 대상: RPA뿐 아니라 AI Agent, 워크플로우, 매크로 등 전부
- 인터페이스: 전용 UI가 아닌 **자연어 대화**
- 비용: 별도 라이선스 없는 자체 구축
