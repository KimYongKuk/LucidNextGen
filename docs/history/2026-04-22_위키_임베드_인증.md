# 2026-04-22 위키(Outline) 임베드 iframe 인증 연동

## 개요

Outline 위키 문서에 임베드된 Lucid AI 채팅 iframe이 오늘 새벽 배포된 JWT 인증 의무화로 인해 401 에러 발생. Cross-origin iframe 특성상 쿠키 전달 불가 → 그룹웨어 위젯과 동일한 **AES 암호화 토큰** 방식으로 해결. Outline 측은 Node.js로 토큰 생성, Lucid 측은 기존 `widget_auth.py` 재사용.

같은 날 해결 과정에서 `/embed` 경로의 Next.js redirect loop 이슈도 발견해서 `/embed/wiki` sub-path로 이동.

## 변경 파일 요약

| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| `frontend/app/embed/page.tsx` | 이동 | `frontend/app/embed/wiki/page.tsx`로 이동 — redirect loop 해소 |
| `frontend/app/embed/layout.tsx` | 이동 | 각 sub-path로 분리 (`wiki/layout.tsx`, `gw/layout.tsx`) |
| `frontend/app/embed/wiki/page.tsx` | 수정 | `token` URL 파라미터 추출 + `widgetAuthToken` prop 전달 |
| Outline 서버 코드 | 수정 필요 (외부) | iframe src 생성 시 AES 토큰 생성 후 `?token=` 추가 |

## 아키텍처

### 이전 (문제 상태)
```
[Outline 위키 문서]
  <iframe src="https://lucid.landf.co.kr/embed?empno=A2304013">
      ↓
  /embed → 301 Redirect Loop (Next.js 버그)
      ↓
  chrome-error → iframe broken (빈 X 아이콘)
```

### 이후 (수정)
```
[Outline 서버]
  generateLucidWikiToken(user.email)
    → AES-128 ECB 암호화 "email|timestamp"
    → URL-encoded base64
  iframe src = `/embed/wiki?token=${token}&sid=${uuid}`
      ↓
[Lucid /embed/wiki/page.tsx]
  URL에서 token 추출
    → EmbedChat(widgetAuthToken={token})
      ↓ useSimpleChat
  X-Widget-Auth 헤더로 chat API 호출
      ↓
[Backend widget_auth.py]
  복호화 → email → v_user_info_mapping → 사번 (A2304013)
      ↓
  인증된 사번으로 Worker 실행 (OUTLINE + DIRECT만 허용)
```

### redirect loop 원인과 해결

**원인:** Next.js에서 `app/embed/` 디렉토리에 서브라우트(`/embed/gw`)가 존재하면, `/embed`(no slash) ↔ `/embed/`(slash) 간 301/308 무한 리다이렉트 발생. `layout.tsx`만 있고 `page.tsx`가 없으면 Next.js가 여전히 route로 인식.

**해결:**
1. `app/embed/page.tsx` → `app/embed/wiki/page.tsx` 이동
2. `app/embed/layout.tsx` → `app/embed/wiki/layout.tsx`, `app/embed/gw/layout.tsx` 양쪽에 복사
3. `/embed/` 디렉토리에서 page.tsx, layout.tsx 둘 다 제거 → Next.js가 `/embed`를 더 이상 route로 인식 안 함

## 상세 내용

### 1. Lucid 측 (`/embed/wiki/page.tsx`)

```tsx
const widgetAuthToken = useMemo(() => {
  return searchParams.get("token") || undefined;
}, [searchParams]);

const userId = useMemo(() => {
  return searchParams.get("empno") || getUserId() || "wiki_user";
}, [searchParams]);

// ...

<EmbedChat
  userId={userId}
  widgetAuthToken={widgetAuthToken}
  chatMode="outline_embed"
/>
```

- `token`이 있으면 `widgetAuthToken`으로 사용 → `X-Widget-Auth` 헤더 전송
- 없으면 `empno`만 (레거시 fallback, 현재 무인증 → 401)

### 2. Outline 측 (Node.js)

```javascript
const crypto = require('crypto');

const WIDGET_KEY = process.env.LUCID_WIDGET_KEY || 'landf01234567890';

function generateLucidWikiToken(identifier) {
  const payload = `${identifier}|${Date.now()}`;
  const cipher = crypto.createCipheriv(
    'aes-128-ecb',
    Buffer.from(WIDGET_KEY, 'utf8'),
    null
  );
  cipher.setAutoPadding(true);  // PKCS7
  const encrypted = Buffer.concat([
    cipher.update(payload, 'utf8'),
    cipher.final()
  ]).toString('base64');
  return encodeURIComponent(encrypted);
}

// iframe src 생성
const token = generateLucidWikiToken(currentUser.email);
const iframeSrc = `https://lucid.landf.co.kr/embed/wiki?token=${token}&sid=${uuid}`;
```

### 3. 암호화 호환성 (Java JSP / Python / Node.js 삼각 검증)

| 언어 | 구현 |
|------|------|
| Java (JSP, 그룹웨어) | `Cipher.getInstance("AES/ECB/PKCS5Padding")` |
| Node.js (Outline 위키) | `crypto.createCipheriv('aes-128-ecb', ...)` + `setAutoPadding(true)` |
| Python (Lucid widget_auth) | `AES.new(key, MODE_ECB)` + `pad(data, AES.block_size)` |

세 구현체 모두 동일 AES-128 ECB + PKCS7 → **상호 복호화 가능**. 같은 `WIDGET_SHARED_KEY` (`landf01234567890`).

## 인텐트 제한 (outline_embed 모드)

위키 임베드에서는 **OUTLINE + DIRECT만 허용** (orchestrator.py):
- 비-DIRECT 인텐트 → 강제 OUTLINE 전환
- DIRECT + 검색성 질문 → OUTLINE으로 전환
- 메일/결재/일정 등 개인 데이터 Worker는 **접근 불가** (보안 경계)

따라서 위키에서 사번이 인증되어도 접근 범위는 **위키 문서 검색**으로 제한됨.

## 결정 사항 및 주의점

### 결정
- **iframe 인증은 AES 토큰 방식 재사용**: 그룹웨어 위젯 + JSP에서 이미 검증된 패턴. Outline Node.js도 호환.
- **사번 대신 email 권장**: Outline이 사번은 직접 모름(Outline 사용자 = email 기반). Lucid 측 `_resolve_to_sabun()`이 email → login_id → 사번 변환.
- **URL에 token + empno 병존 허용**: legacy fallback 유지 (Outline 업데이트 전 임시 동작 보장).

### 주의점
- **토큰 TTL 8시간**: Outline 페이지 오래 열어둬도 안전. 그 이후는 Outline이 다시 iframe 렌더링해서 새 토큰 발급.
- **cross-origin iframe 쿠키 전송 안 됨**: `auth_token` 쿠키가 lucid 도메인이라도 Outline iframe 안에서는 SameSite 정책으로 전송 안 됨. X-Widget-Auth 헤더만이 유일한 인증 경로.
- **outline_embed 모드의 인텐트 제한 유지**: 위키에서 메일/결재 Worker 사용 시도해도 OUTLINE으로 강제 전환. 보안 경계 의도적.
- **키 공유 주의**: WIDGET_SHARED_KEY가 Lucid 백엔드 + JSP + Outline 3군데 하드코딩. 변경 시 세 곳 동시 업데이트 필요.

## 배포 순서 (실제)

1. Lucid 코드 수정 + 배포 (embed/wiki 이동, token 지원)

3. 위키 페이지 새로고침 → iframe 정상 로드 + 채팅 동작 ✅
