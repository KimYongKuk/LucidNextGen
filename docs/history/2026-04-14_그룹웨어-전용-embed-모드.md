# 2026-04-14 그룹웨어 전용 Embed 모드

## 개요

## 상세 내용

### 아키텍처
```
custom_index_header.jsp (다우오피스)
  ↓ <script> lucid-chat-widget-gw.js 로드
  ↓ LucidChat.init({ apiUrl, userId, position })
  ↓
플로팅 버튼 클릭
  ↓ GO.session().employeeNumber → 사번 추출
  ↓ iframe src="/embed/gw?empno=A2304013"
  ↓
embed/gw/page.tsx (Next.js)
  ↓ chatMode="groupware_embed"
  ↓ useSimpleChat → 백엔드 직접 호출
  ↓
orchestrator.py
  ↓ groupware_embed 모드: 허용 인텐트만 라우팅
  ↓ MAIL, APPROVAL, BOARD, CALENDAR, RESERVATION,
  ↓ IT_SUPPORT, ACCT_SUPPORT, WEB_SEARCH, DIRECT
  ↓
해당 Worker 실행 → SSE 스트리밍 응답
```

### 사번 추출 과정 (해결 과정 포함)
1. **JSP `${userId}`** → 다우오피스 내부 DB 키(337), 사번이 아님
2. **`v_user_info_mapping.user_id`** → go_users.id와 다른 ID 체계, 잘못된 사용자로 매핑됨
3. **`go_users` 직접 조회** → `ai_reader` 계정에 SELECT 권한 없음
4. **`GO.session().employeeNumber`** → 클라이언트에서 사번 직접 추출 (최종 채택)

### iframe lazy-load
- `<head>` 스크립트 실행 시점에는 `GO.session()`이 미초기화
- 해결: 플로팅 버튼 클릭 시 iframe 생성 (그때 GO.session() 확실히 준비됨)

### userId anonymous 문제
- `useSearchParams()` + `useMemo` 조합에서 리렌더 시 searchParams 참조 변경 → empno 소실
- `useCallback` 의존성 추가 → 효과 없음
- `useRef`로 최신값 참조 → 효과 없음
- **`useState` initializer로 마운트 시 1회만 URL에서 추출** → 해결

### nginx CSP 설정
```nginx
location /embed/ {
    proxy_hide_header X-Frame-Options;
    add_header Content-Security-Policy "frame-ancestors https://lfon.landf.co.kr https://*.landf.co.kr http://192.168.90.30:* http://192.168.100.55" always;
}
```

## 결정 사항 및 주의점
- **위젯 on/off**: `C:/Services/LFChatbot_prod/widget/lucid-chat-widget-gw.js` 파일명 변경으로 제어
- **다우오피스 업데이트**: `custom_index_header.jsp` 초기화 가능 → 스크립트 재삽입 (30초)
- **개발서버 테스트**: `http://192.168.100.55` CSP에 추가 필요 (완료)
- **위키 embed 영향 없음**: `embed-chat.tsx` 파라미터화로 기존 `outline_embed` 동작 유지
