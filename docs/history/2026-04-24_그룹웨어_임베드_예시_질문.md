# 2026-04-24 그룹웨어 임베드 예시 질문 박스

## 개요
그룹웨어 embed(`/embed/gw`) 초기 화면에서 사용자가 바로 시도할 수 있는 예시 질문 박스를 2x2 그리드로 노출. 메시지가 하나도 없을 때만 입력창 위에 4개 랜덤 표시. wiki embed와 메인 챗은 영향 없음.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| frontend/components/embed-gw-examples.tsx | 추가 | 그룹웨어 전용 예시 풀(41개) + 2x2 랜덤 노출 컴포넌트 |
| frontend/components/embed-chat.tsx | 수정 | `renderEmptyExamples` 렌더 prop 추가 (messages.length===0일 때 호출) |
| frontend/app/embed/gw/page.tsx | 수정 | `EmbedGwExamples`를 `renderEmptyExamples`로 연결 |

## 상세 내용

### 예시 풀 카테고리 (총 41개)
- IT 문제 해결/매뉴얼 (6)
- 담당자/조직도 — 전 지원부서 (10): IT, HR, 재무/총무, 회계, 구매, EHS, 품질
- 메일 (3)
- 전자결재 (3)
- 사내 공지 / 지원 VOC (3)
- 일정 관리 (6) — CalendarWorker
- 회의실/자산 예약 (6) — ReservationWorker
- 일정+예약 하이브리드 (4) — CalendarWorker 통합 도구 활용

### 컴포넌트 구조
- `EmbedGwExamples`: `chatId` + `onSelect(text)` prop 수신, 마운트 시 풀에서 4개 랜덤 추출
- `chatId` 변경 시(새 대화) useEffect로 재추출
- Framer Motion으로 순차 페이드인 애니메이션

### EmbedChat 확장
`renderEmptyExamples?: (args: { chatId, onSelect }) => ReactNode` prop 추가.
- 메시지가 비었을 때만 호출
- wiki embed(`/embed/wiki`)는 prop 미전달 → 기존 동작 유지

## 결정 사항 및 주의점
- **UI-only 변경 스코프**: 사용자 요청대로 gw embed UI만 커스텀. `EmbedChat`은 prop 기반 확장이므로 wiki embed에 영향 없음
- **하이브리드 예시**: CalendarWorker는 `create_event` + `create_reservation` 도구를 모두 가지므로 한 대화에서 일정 등록 + 회의실 예약 동시 처리 가능. 예시도 이 특성을 반영
- **랜덤 노출 일관성**: `memo` compare로 `chatId` 같으면 리렌더 방지 → 같은 세션에서는 동일한 4개 유지
- **향후 확장**: 채팅 내부 UI까지 커스텀 필요해지면 `EmbedChatGw`로 분리하는 것을 고려 (지금은 gw page + 전용 컴포넌트 레벨에서 충분)
