# 2026-03-30 Outline 컬렉션 문서 수 부정확 수정

## 개요
`list_collections` MCP 도구가 Outline API의 캐시된 `documentCount`를 그대로 반환하여 실제 문서 수와 불일치하는 문제를 수정. 실제 문서 트리를 병렬 조회하여 정확한 카운트를 반환하도록 변경.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| backend/app/mcp_servers/outline_server/server.py | 수정 | `list_collections`에서 `collections.documents` 병렬 호출로 실제 문서 수 조회 |

## 상세 내용

### 문제
- Outline API의 `collections.list` 응답에 포함된 `documentCount`는 캐시된 메타데이터
- 하위 문서(child documents)가 있거나 문서 생성/삭제 직후에는 실제 수와 불일치
- LLM이 "문서 0개"로 잘못 응답 → 사용자가 재질문해야 실제 문서 확인 가능

### 수정 내용
- `list_collections` 호출 시 각 컬렉션의 `collections.documents` API를 **asyncio.gather로 병렬 호출**
- 트리 구조를 재귀적으로 순회하여 정확한 문서 수(`_count_tree`) 계산
- 조회 실패 시 기존 `documentCount`를 폴백으로 사용하되, `documentCountApproximate: true` 플래그 추가

## 결정 사항 및 주의점
- 컬렉션 수가 많을 경우 API 호출이 N+1회로 증가하나, 병렬 실행이므로 latency는 1회 호출 수준
- 100개 컬렉션 limit 내에서는 Outline 서버 부하 문제 없을 것으로 판단
- 폴백 시 `documentCountApproximate` 플래그로 LLM이 부정확함을 인지할 수 있음
