# 2026-04-10 이미지 맥락 유지 수정

## 개요
이미지 공유 후 후속 질문 시 AI가 "이미지를 받은 적 없습니다"라고 응답하는 문제 수정. message_history에 이미지 base64가 포함되지 않아 AI가 이전 분석을 활용하지 않는 현상.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| frontend/hooks/use-simple-chat.ts | 수정 | message_history에 `[이미지 첨부됨]` 힌트 태그 추가 |
| backend/app/agents/workers/base_worker.py | 수정 | 시스템 프롬프트에 이미지 맥락 유지 규칙 추가 |

## 상세 내용

### 문제 현상
- 사용자가 이미지 공유 후 "다시 분석해줘", "좀 더 자세히" 등 후속 질문 시
- AI가 "이미지를 받은 적 없습니다", "이미지가 참조되지 않습니다" 등으로 응답
- message_history는 정상 전송됨 (토큰 수 증가 확인), AI의 이전 분석 텍스트도 포함됨
- 그러나 이미지 자체가 history에 없으므로 AI가 "이미지 없음"에 과하게 집중

### 근본 원인
1. 프론트엔드 message_history에서 이미지 parts는 텍스트만 추출 (base64 포함 시 토큰 폭발)
2. AI가 이전 분석 결과 텍스트는 보고 있지만, "현재 이미지가 없다"는 사실에 초점을 맞춰 거부 응답
3. 시스템 프롬프트에 이미지 맥락 유지 가이드가 없었음

### 수정 방법

**프론트엔드**: 이미지가 포함된 user 메시지에 `[이미지 첨부됨]` 텍스트 태그 추가
```javascript
const hasImages = msg.role === 'user' && msg.parts
  ?.some((part) => part.type === 'file' && part.mediaType?.startsWith('image/'));
const content = hasImages ? `[이미지 첨부됨]\n${textContent}` : textContent;
```

**백엔드**: base_worker.py의 "대화 히스토리 데이터 활용" 규칙에 이미지 맥락 유지 규칙 추가
- `[이미지 첨부됨]` 태그 인식 → 이전 분석 결과 기반으로 후속 질문 답변
- "이미지가 없습니다" 거부 응답 금지

## 결정 사항 및 주의점
- 이미지 base64를 history에 포함하면 토큰 비용이 폭발하므로 텍스트 태그 방식 채택
- AI의 이전 분석 결과가 history의 assistant 메시지에 포함되어 있으므로 재분석은 가능하나, 새로운 관점의 분석은 원본 이미지 재첨부 필요
- stale closure 이슈가 아님 확인 — `onData`/`onFinish`/`onError`가 매 렌더 새 참조이므로 `useCallback` deps 변경 → `sendMessage` 매 렌더 재생성
