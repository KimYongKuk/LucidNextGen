# 2026-03-30 IT VOC → L&F Wiki 자동 축적 시스템

## 개요
IT VOC(v_works_app_934_data) DB의 해결 사례를 매일 자동으로 L&F Wiki "L&F IT 지식베이스" 컬렉션에 축적하는 배치 시스템을 구현했다. 매일 새 문서를 만드는 대신, LLM이 기존 문서를 읽고 신규 건을 병합하여 중복 없이 지식을 보강한다.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| backend/app/services/voc_wiki_service.py | 신규 | 핵심 서비스 (VOC 조회→LLM 분류→LLM 병합→Outline API) |
| backend/app/utils/voc_wiki_scheduler.py | 신규 | CronTrigger 스케줄러 (매일 06:00 KST) |
| backend/app/main.py | 수정 | 스케줄러 등록 (startup/shutdown) |
| backend/migrations/add_voc_wiki_sync.sql | 신규 | 동기화 로그 테이블 |

## 상세 내용

### 아키텍처
```
[CronTrigger 매일 06:00 KST]
  → VOC DB 조회 (asyncpg, since last_sync)
  → Haiku LLM 분류 (시스템/주제, 20건 배치)
  → 시스템/주제별 그룹핑
  → Haiku LLM 병합 (기존 문서 + 신규 건)
  → Outline API 문서 생성/업데이트
  → 부모 목차 갱신
  → MySQL sync_log 저장
```

### 문서 계층 구조
```
📁 L&F IT 지식베이스 (컬렉션)
├── 📄 SAP 지원 사례 모음 (부모 - 목차)
│   ├── 📄 SAP - 로그인/권한 문제 (자식 - 주제별)
│   └── 📄 SAP - 트랜잭션 오류
├── 📄 네트워크 지원 사례 모음
│   └── 📄 네트워크 - VPN 접속 장애
└── ... (LLM이 내용 기반 자율 생성)
```

### LLM 분류 전략
- DB의 `시스템` 컬럼은 hint로만 사용, LLM이 실제 내용 기반 재분류
- 기존 주제 목록을 context로 제공하여 분류 일관성 유지
- 폴백: LLM 실패 시 DB 시스템 + "미분류"

### 핵심 클래스/함수
| 클래스/함수 | 역할 |
|------------|------|
| `VocWikiService` | 메인 서비스 (싱글톤) |
| `VocWikiService.sync()` | 전체 동기화 오케스트레이션 |
| `VocWikiService.classify_entries()` | Haiku LLM 배치 분류 |
| `VocWikiService.merge_into_document()` | Haiku LLM 문서 병합 |
| `VocWikiScheduler` | APScheduler CronTrigger 래퍼 |
| `VocWikiScheduler.run_initial_load()` | 초기 적재 (1주 단위 순차) |

### 환경변수
| 변수 | 기본값 | 설명 |
|------|--------|------|
| VOC_WIKI_SYNC_ENABLED | true | 스케줄러 on/off |
| VOC_WIKI_SYNC_HOUR | 6 | 실행 시각 (KST) |
| VOC_WIKI_COLLECTION_ID | (필수) | Outline 컬렉션 ID |

### Outline API 직접 호출
MCP 서버를 경유하지 않고 서비스에서 Outline HTTP API를 직접 호출한다:
- `documents.create` — 문서 생성
- `documents.update` — 문서 내용 업데이트
- `documents.info` — 문서 조회
- `collections.documents` — 트리 구조 조회

## 결정 사항 및 주의점
- **MCP 서버 미수정**: 배치 서비스가 독립적으로 Outline API 호출 (MCP는 사용자 대화용)
- **초기 적재**: `run_initial_load(months=1)`로 최근 1개월치 수동 실행 필요
- **마이그레이션**: `add_voc_wiki_sync.sql` 수동 실행 필요
- **컬렉션 ID**: Outline에서 "L&F IT 지식베이스" 컬렉션 ID 확인 후 `.env`에 설정 필요
