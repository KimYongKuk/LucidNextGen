# 2026-04-01 캘린더 일정 관리 Worker

## 개요
LFON 그룹웨어 캘린더 REST API를 연동하여 일정 조회/등록/삭제를 수행하는 CalendarWorker를 신규 추가하였다. 내 캘린더, 관심 캘린더, 공개 캘린더 조회를 지원하며, 비공개 일정 필터링과 소유권 검증을 통해 권한 기반 접근 제어를 구현하였다.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| backend/app/mcp_servers/calendar_mcp_server.py | 신규 | MCP 서버 (6개 도구) |
| backend/app/agents/workers/calendar_worker.py | 신규 | Sonnet Worker (보안 래핑 + 시스템 프롬프트) |
| backend/app/agents/state.py | 수정 | Intent.CALENDAR + INTENT_TO_WORKER + WORKER_CAPABILITIES 추가 |
| backend/app/agents/intent_classifier.py | 수정 | quick_classify 패턴 + LLM 분류 설명 + 예시 추가 |
| backend/app/agents/workers/__init__.py | 수정 | CalendarWorker import + 레지스트리 등록 |
| backend/mcp_config.json | 수정 | calendar_server 엔트리 추가 |

## 상세 내용

### MCP 도구 (6개)
| 도구 | 용도 | 보안 |
|------|------|------|
| get_my_calendars | 내 캘린더 + 관심 캘린더 목록 | employee_number 주입 |
| get_user_public_calendars | 특정 사용자의 공개 캘린더 조회 | employee_number 주입 |
| get_calendar_events | 기간별 일정 조회 (다중 캘린더) | private 일정 필터링 |
| get_event_detail | 일정 상세 조회 | private 일정 접근 검증 |
| create_event | 일정 등록 (내 캘린더만) | 소유권 검증 |
| delete_event | 일정 삭제 (내 일정만) | 소유권 + creator 검증 |

### API 엔드포인트
| Method | Endpoint | 용도 |
|--------|----------|------|
| GET | /api/calendar/feed | 내 캘린더 + 관심 캘린더 목록 |
| GET | /api/calendar/user/{goUserId}/calendar | 특정 사용자의 공개 캘린더 |
| GET | /api/calendar/event?timeMin=...&timeMax=...&calendarIds[]=... | 기간별 일정 조회 |
| GET | /api/calendar/{calendarId}/event/{eventId} | 일정 상세 |
| POST | /api/calendar/{calendarId}/event/ | 일정 등록 |
| DELETE | /api/calendar/{calendarId}/event/{eventId}?recurChangeType=... | 일정 삭제 |

### 권한 모델
- **캘린더 레벨**: visibility (public/private/수락후공개)
  - 내 캘린더: 무조건 접근 가능
  - 관심 캘린더: feed에 있으면 접근 가능 (수락 완료)
  - 공개 캘린더: 관심 등록 여부 무관하게 접근 가능
  - 비공개/수락전: 접근 불가
- **일정 레벨**: event.visibility (public/private)
  - 타인 캘린더의 private 일정 → creator 또는 attendee만 상세 조회 가능
  - 그 외 → 시간만 표시 + "🔒 비공개 일정" 마스킹

### 인증 패턴
- 예약 워커와 동일: 서비스 계정 SSO → API 호출, 사번 → GO user.id → 데이터 식별
- prepare_tools()에서 employee_number 강제 주입 (6개 도구 전부)
- v_user_info_mapping VIEW 재사용

### 인텐트 분류
- quick_classify 키워드: 캘린더, 일정 조회/등록/삭제/추가/확인, 내 일정, 오늘/이번주/다음주 일정, 빈 시간, 스케줄
- "회의실 예약" → reservation, "일정/캘린더" → calendar 으로 분리
- CALENDAR_WORKER_ENABLED 환경변수로 on/off 제어

### 캐싱
- 사번 → GO user.id 매핑: 프로세스 수명 캐싱
- 피드(캘린더 목록): 5분 TTL 캐싱 (등록/삭제 시 무효화)

## 결정 사항 및 주의점
- feed API는 로그인 세션 기준이라 서비스 계정으로 호출 시 사용자별 관심 캘린더 구분 불가할 수 있음. API 탐색 후 불가 시 DB VIEW로 전환 예정.
- 일정 등록 시 참석자는 이메일 기반으로 추가 (GO user.id 매핑은 본인만)
- 반복 일정 삭제 시 delete_type 파라미터로 범위 지정 (all/this/following)
- 일정 등록/삭제 전 반드시 사용자 확인 필수 (시스템 프롬프트 규칙)
