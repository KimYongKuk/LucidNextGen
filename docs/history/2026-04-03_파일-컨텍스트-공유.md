# 2026-04-03 파일 컨텍스트 공유 — 모든 워커에서 업로드 파일 접근 가능

## 개요
기존에는 UserFilesWorker만 업로드 파일에 접근할 수 있어, 다른 워커로 라우팅되면 "파일에 접근할 수 없습니다" 응답이 발생했다. 이제 모든 워커에서 파일 검색 도구를 사용할 수 있도록 base_worker에서 자동 주입한다.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| backend/app/agents/workers/base_worker.py | 수정 | 파일 검색 도구 자동 추가 + 시스템 프롬프트 파일 컨텍스트 주입 |
| backend/app/agents/intent_classifier.py | 수정 | has_files + 분석/요약 키워드 → USER_FILES 즉시 라우팅 |

## 상세 내용

### 1. 도구 자동 추가 (base_worker.py stream_response)
- `has_files=True` 또는 `workspace_has_files=True`일 때
- `search_user_files`, `search_workspace_docs`를 자동으로 filtered_tools에 추가
- 이미 보유한 워커(PPTWorker, VisualizationWorker 등)는 중복 방지로 건너뜀
- 로그: `[워커명] [FILE_TOOLS] 파일 검색 도구 자동 추가: [도구명]`

### 2. 시스템 프롬프트 주입 (base_worker.py build_system_prompt)
- 파일 존재 시 "UPLOADED FILES" 섹션 자동 추가
- 업로드 파일명 목록 표시
- 파일 검색 도구 사용법 (session_id, workspace_uuid 포함)
- "파일에 접근할 수 없습니다"라고 답하지 말라는 지시

### 3. 인텐트 분류 개선 (intent_classifier.py)
- `has_files=True` + 분석/요약/정리/확인 등 키워드 → USER_FILES 즉시 반환
- 파일 명시 없이 "분석해줘"만 말해도 파일 우선 접근

## 결정 사항 및 주의점
- 도구 자동 추가는 base_worker 레벨이므로 개별 워커 수정 불필요
- 토큰 증가: 시스템 프롬프트에 ~200토큰 추가 (파일 존재 시만)
- XlsxWorker의 prepare_tools에서 search_user_files/search_workspace_docs의 filepath 검증은 불필요 (이 도구들은 filepath 파라미터 없음)
