# 2026-04-13 시각화 과다 사용 억제 + PPT/XLSX 인텐트 오분류 방지

## 개요
시각화(SVG/HTML 위젯)가 지나치게 자주 생성되는 문제와, PPT/XLSX 인텐트가 사용자 요청 없이 오분류되어 불필요한 파일 생성이 발생하는 문제를 수정.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| backend/app/agents/workers/base_worker.py | 수정 | 시각화 가이드: 텍스트/마크다운 우선 원칙, 파일 산출물 빈 파일 방지 규칙 |
| backend/app/agents/intent_classifier.py | 수정 | PPT quick_classify 생성동사 필수화, LLM 프롬프트 문서생성 규칙 강화 |

## 상세 내용

### 1. 시각화 가이드 변경 (base_worker.py)
- **이전**: "내용의 이해를 돕는다고 판단되면 자연스럽게 포함" → LLM이 과도하게 시각화 생성
- **이후**: "텍스트/마크다운이 기본, 시각화 도구는 명시적 요청 또는 복잡 데이터일 때만"
- "하지 마세요" 섹션 추가: 단순 정보에 SVG/HTML 금지, 3~4개 비교는 마크다운 표 사용
- 회색지대 기준: "HTML" → "마크다운 표"로 변경

### 2. PPT/XLSX 인텐트 분류 강화 (intent_classifier.py)
- **PPT quick_classify**: 키워드만 → 키워드 + 생성동사(만들/생성/작성 등) 필수
  - "PPT 만들어줘" → PPT_GENERATION, "PPT 잘 만드는 법" → LLM 위임
- **LLM 프롬프트 rule 1**: "ALWAYS ppt_generation" → "명시적 생성 요청일 때만"
- **LLM 프롬프트 rule 3**: XLSX도 명시적 생성 요청일 때만
- 예시 추가: 생성이 아닌 PPT/XLSX 언급 케이스

### 3. 빈 파일 생성 방지 (base_worker.py)
- 파일 생성 전: 명시적 요청일 때만 도구 호출, 아니면 텍스트 답변 + 제안
- 도구 호출 후: 반환 결과에서 파일명 확인 필수, 에러 시 거짓 파일명 출력 금지
- 실패 시 "파일 생성에 실패했습니다" 솔직 답변 규칙

## 결정 사항 및 주의점
- 시각화 기능 자체를 제거하지는 않음 — 사용자가 기능 존재를 인지할 수 있도록 팔로우업 제안은 유지
- PPT 키워드가 있지만 생성동사가 없는 경우 LLM에 위임하여 맥락 판단 가능
- "정리해줘"는 PPT 생성동사에 포함하되, 단독 사용 시 direct로 처리
