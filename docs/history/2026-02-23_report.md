# 서비스 레포트 대시보드 구현 (2026-02-23)

## 개요
관리자용 서비스 레포트 대시보드(`/admin/report`)를 v0 목업 기반 UI에서 실제 데이터 연동까지 완성.

## 아키텍처

```
[Frontend: /admin/report]
    ↓ fetchAllReportData() → Promise.all 6개 API 병렬 호출
[Backend: /api/v1/admin/report/*]
    ↓ ReportService (singleton)
[MySQL: chat_log_new] + [ChromaDB: workspace collections] + [Disk: output directories]
```

## Phase 1: 백엔드 데이터 수집 파이프라인

### DB 스키마 변경
- `chat_log_new` 테이블에 3개 컬럼 추가:
  - `intent VARCHAR(20)` — 인텐트 분류 결과
  - `worker_name VARCHAR(30)` — 처리 Worker 이름
  - `response_time_ms INT` — 응답 소요 시간(ms)
- 마이그레이션: `backend/migrations/add_report_columns.sql`

### 데이터 수집 흐름
```
[a2a_streaming.py] intent_classified 이벤트 → classified_intent/classified_worker 캡처
    ↓ _internal_collected 이벤트에 intent, worker_name, response_time_ms, is_error, tools_used 포함
[chat.py] _internal_collected 수신 → metadata에 is_error, tools_used, image_count 추가
    ↓ background_tasks.add_task(_save_chat_log_background, ..., intent, worker_name, response_time_ms)
[chat_log_service.py] save_chat_log()에 3개 파라미터 추가 → INSERT SQL 확장
```

## Phase 2: 백엔드 Report API (6+1 엔드포인트)

| 엔드포인트 | 데이터 소스 | 설명 |
|-----------|-----------|------|
| `GET /overview` | chat_log_new | 총 메시지/세션/사용자 + 일별 추이 |
| `GET /intents` | chat_log_new | 의도별 분포 (count, ratio, intentKey) |
| `GET /intents/detail` | chat_log_new | 특정 의도의 메시지 상세 리스트 (최근 50건) |
| `GET /quality` | chat_log_new | 답변 실패율 + 카테고리별 실패율 + 실패 샘플 |

| `GET /artifacts` | output directories + chat_log_new | 파일 생성물 현황 (PDF/XLSX/PPT/업로드) |
| `GET /performance` | chat_log_new | 응답 성능 (평균/P95, Worker별, 일별 추이) |

## 주요 설계 결정

### 1. 답변 품질 = "답변 실패" 감지
단순 에러가 아닌, Agent가 답변을 제대로 못한 경우를 감지:
- `is_error = true` (시스템 에러)
- `LENGTH(outputLog) < 10` (빈/극히 짧은 응답)
- outputLog에 실패 패턴 매칭 (9개):
  - `찾을 수 없`, `검색 결과가 없`, `관련 정보를 찾지 못`, `관련된 정보가 없`
  - `해당하는 내용을 찾`, `도움을 드리기 어려`, `답변하기 어려`
  - `관련 자료가 없`, `조회 결과가 없`

### 2. 파일 생성물 카운트 = 디렉토리 스캔
chat_log 기반이 아닌 **실제 출력 디렉토리 파일** 기준:
- `backend/data/pdf_output/*.pdf` → PDF 생성 건수
- `backend/data/ppt_output/*.pptx` → PPT 생성 건수
- `backend/data/xlsx_output/*.xlsx` → XLSX 생성 건수
- 파일 modification time으로 날짜 범위 필터링 → 실제 생성 파일만 정확히 카운트

### 3. 워크스페이스 문서 수 = ChromaDB 조회
- `workspace_{uuid}` 컬렉션에서 고유 `file_id` 카운트
- top 10 워크스페이스만 조회하므로 성능 부담 최소

### 4. 의도분류 상세 모달
- 파이차트 섹션 클릭 또는 테이블 행 클릭 → 모달 팝업
- 해당 의도의 최근 50건 메시지 (질문+답변 미리보기+응답시간)
- ESC 키 또는 배경 클릭으로 닫기

### 5. 이미지 업로드 추적
- `chat.py`에서 `request.images`가 있으면 `metadata.image_count` 저장
- report에서 `JSON_EXTRACT(metadata, '$.image_count') > 0` 으로 카운트
- 기존 데이터에는 image_count 없으므로 0으로 표시 (향후 데이터부터 집계)

## Collation 이슈 해결
- `workspaces.uuid`(utf8mb4_unicode_ci)와 `chat_sessions.workspace_id`(utf8mb4_general_ci) 충돌
- `LEFT JOIN workspaces w ON cs.workspace_id = w.uuid COLLATE utf8mb4_general_ci`로 해결

## 사전 요구사항
- DB 마이그레이션 실행 필요: `backend/migrations/add_report_columns.sql`
- 마이그레이션 전 기존 데이터는 intent/worker_name/response_time_ms가 NULL
- 마이그레이션 후 새로 생성되는 채팅 로그부터 데이터 수집 시작
