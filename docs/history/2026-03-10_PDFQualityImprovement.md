# 2026-03-10 PDF 품질 개선

## 개요
PDF 생성기(fpdf2 기반)의 출력 품질을 DOCX 생성기와 동일 수준으로 개선. 여백, 색상, 표 렌더링, 코드 블록 등 전반적 품질 향상.

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| `backend/app/mcp_servers/pdf_generator/server.py` | 수정 | PDF 렌더링 전반 개선 |

## 상세 내용

### 변경 사항 비교

| 항목 | Before | After |
|------|--------|-------|
| **좌우 여백** | 15mm | 25mm (DOCX 2.5cm과 동일) |
| **콘텐츠 영역** | 180mm | 160mm (더 안정된 레이아웃) |
| **표 헤더 색상** | 회색 (74,85,104) | 파란색 (43,108,176) — DOCX와 동일 |
| **표 텍스트** | cell → 길면 잘림 | multi_cell → 자동 줄바꿈 |
| **표 행 높이** | 고정 7mm | 가장 긴 셀 기준 동적 계산 |
| **표 열 너비** | 최소 20mm, 최대 80mm | 최소 18mm, 콘텐츠 비율 기반 |
| **코드 블록 배경** | 어두운 색 (26,32,44) | 밝은 회색 (240,244,248) — DOCX와 동일 |
| **코드 텍스트** | 밝은 색 (226,232,240) | 진한 회색 (45,58,74) — DOCX와 동일 |
| **코드 블록 잘림** | 95자에서 `...` | 제한 없음 |
| **코드 블록 테두리** | 없음 | 있음 (table_border 색상) |
| **부제목** | 미지원 | 지원 (제목 아래 회색 이탤릭) |
| **제목 크기** | 20pt | 22pt (DOCX와 동일) |
| **H2 크기** | 14pt | 15pt (DOCX와 동일) |
| **테이블 테두리** | 연한 회색 | 연한 파란 (#B0C4DE) — DOCX와 동일 |

### 주요 기술 변경

#### 표 렌더링 (render_table)
- `pdf.cell()` → 셀 배경/테두리를 `pdf.rect()`로 먼저 그리고, `pdf.multi_cell()`로 텍스트 렌더링
- 행 높이: `_measure_cell_height()` 함수로 모든 셀의 렌더링 높이를 사전 측정 → 가장 큰 값 사용
- 페이지 넘침 시 헤더 자동 반복 (`_render_header_row` 내부 함수)

#### 코드 블록 (render_code_block)
- 밝은 배경 + 테두리 스타일 (어두운 배경 → 밝은 배경)
- 95자 하드 잘림 제거
- 페이지 넘침 시 새 페이지에서 시작

#### 레이아웃 상수화
- `PAGE_WIDTH`, `MARGIN_LEFT`, `MARGIN_RIGHT`, `CONTENT_WIDTH` 상수로 관리
- 하드코딩된 15/195/180 값 → 상수 참조

### MCP 도구 변경
- `create_document_pdf`: `subtitle` 파라미터 추가 (DOCX의 subtitle과 동일)

## 결정 사항 및 주의점
- fpdf2의 multi_cell은 cell과 달리 Y 위치가 자동으로 이동하므로, 표 렌더링 시 각 셀마다 set_xy로 위치를 명시적으로 지정해야 함
- 코드 블록의 긴 줄은 잘리지만 `...` 표시를 하지 않음 (PDF 특성상 줄바꿈하면 코드 가독성 저하)
- 백엔드 서버 재시작 필요 (MCP 서버 코드 변경)
