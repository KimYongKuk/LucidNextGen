# 2026-04-17 VOC 담당자 직위 필터링 + 다중 부서 매핑

## 개요
IT VOC 자동등록 시 담당자 지정을 두 축으로 정교화:
1. **직위 필터**: 담당자 풀을 `파트장` / `책임` 직위로만 한정 (팀원/NULL 제외)
2. **다중 부서 매핑**: 한 시스템 코드가 **여러 부서**를 공동 담당할 수 있도록 구조 변경 (보안성 검토용)

## 변경 파일 요약
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| backend/app/mcp_servers/works_it_mcp_server.py | 수정 | `_get_dept_members()` 쿼리 직위 JOIN, `SYSTEM_CODE_TO_DEPT`(str) → `SYSTEM_CODE_TO_DEPTS`(tuple) 구조 변경, 호출부 2곳 갱신 |
| (DB) v_org_chart | 수정 (DBA 작업) | `"직위"` 컬럼 추가 (position_id → go_domain_codes JOIN 한글명) |

## 상세 내용

### 직책 vs 직위 (혼동 주의)
- **직책(duty)**: 조직 내 역할. 값: `파트장`, `팀원`, NULL
- **직위(position)**: 인사상 등급. 값: `파트장`, `책임`, NULL 등
- 이번 필터링은 **직위** 기준. 이름이 겹쳐 보이지만(`파트장`) 필드는 별개

### 상수 변경

#### ① 직위 화이트리스트
```python
ASSIGNEE_ALLOWED_POSITIONS = ("파트장", "책임")
```

#### ② 시스템 코드 → 부서 매핑 (단일 → 다중)

Before (str):
```python
SYSTEM_CODE_TO_DEPT = {
    "14": "보안기술팀",  # 보안성 검토
    ...
}
```

After (tuple, 1 시스템 → N 부서 허용):
```python
SYSTEM_CODE_TO_DEPTS = {
    "14": ("보안기술팀", "보안관리파트"),  # 보안성 검토: 기술+관리 공동
    "2":  ("보안기술팀",),                 # DLP
    "3":  ("보안기술팀",),                 # DRM
    "15": ("보안기술팀",),                 # MDM
    "19": ("보안기술팀",),                 # VPN
    ...  # 나머지는 단일 부서 튜플
}
```

산업보안파트(물리보안)는 현재 IT VOC 시스템 코드에 매칭되는 항목이 없어 제외.

### `_get_dept_members()` 쿼리 변경

```sql
-- 다중 부서 + 직위 필터
SELECT m.user_id, m.login_id, m.name, m.dept_id, m.dept_name, m.employee_number
FROM v_user_info_mapping m
JOIN v_org_chart o ON m.user_id = o.user_id
WHERE m.dept_name = ANY($1::text[])
  AND o."직위" = ANY($2::text[])
```

- 시그니처: `_get_dept_members(dept_names: tuple[str, ...] | list[str])`
- 호출부 (성공 응답/폴백 메시지) 2곳의 `dept_name_for_assign` → `dept_names_for_assign`, 표시는 `/`로 join

## 결정 사항 및 주의점

- **시스템별 직위 차별화 없음**: 모든 시스템 동일하게 "파트장/책임". 필요 시 `SYSTEM_CODE_TO_ALLOWED_POSITIONS` 별도 도입 가능
- **Fallback 없음**: 담당자 0명이면 빈 배열로 VOC 등록됨 (사용자 명시적 결정)
- **works_acct는 미적용**: 회계 VOC 자동등록 로직 자체가 없음
- **산업보안파트 제외**: 물리보안 중심 부서라 IT VOC 시스템 코드(0~19)에 매칭 항목 없어 보류
- **v_org_chart 의존성**: `"직위"` 컬럼 삭제/변경 시 담당자 0명으로 빠짐. 로그 모니터링 포인트: `[IT VOC MCP] 부서 멤버 조회: {부서들} (파트장/책임) → 0명`
- **구조 확장성**: 향후 다른 시스템 코드도 다중 부서 담당으로 바꾸고 싶으면 튜플에 부서명만 추가
