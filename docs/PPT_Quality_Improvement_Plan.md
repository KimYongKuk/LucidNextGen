# PPT 콘텐츠 퀄리티 개선 계획

## 현재 상태

### 코드 지원 현황
PPT 생성 서버(`ppt_generator/server.py`)는 4가지 shape 타입을 지원:

| shape 타입 | 구현 상태 | 실제 산출물 활용도 |
|-----------|----------|------------------|
| `textbox` | 완성 | **높음** — 거의 모든 슬라이드에서 사용 |
| `table` | 완성 (헤더 색상, 교대행, 셀 병합) | **높음** — 데이터 정리 시 주로 사용 |
| `chart` | 구현됨 (line/column/bar/pie/area/scatter) | **낮음** — LLM이 거의 사용하지 않음 |
| `image` | 구현됨 (chart_output 경로 참조) | **낮음** — 경로 참조 한계로 사실상 미사용 |

### 핵심 문제
**코드 기능 부족이 아니라, LLM이 텍스트+테이블만 생성하는 경향이 문제.**

원인:
1. **차트**: LLM이 `categories`/`series` 데이터를 정확히 구조화해야 하는데, 시스템 프롬프트에 예시·규칙이 부족
2. **이미지**: 서버에 이미 존재하는 파일 경로를 알아야 해서 사실상 사용 불가
3. **레이아웃 다양성 부재**: LLM이 "텍스트 박스 하나 + 테이블 하나"로 안전하게 가려는 경향
4. **슬라이드 구성 패턴 미정의**: 어떤 콘텐츠에 어떤 레이아웃을 써야 하는지 가이드 없음

### 산출물 특징
- 대부분 **텍스트 위주 + 테이블 위주**로만 구성
- 차트, 다이어그램, 시각적 강조 요소 부족
- 모든 슬라이드가 비슷한 단조로운 레이아웃

## 개선 방향

### 1. 목표 산출물 샘플 (최우선)
- "이런 PPT를 만들고 싶다"는 **실제 .pptx 예시 파일 2~3개** 확보
- 좋은 슬라이드 vs 부족한 슬라이드 구체적 정의
- 샘플 기반으로 LLM 프롬프트 개선

### 2. 슬라이드 구성 패턴 정의
콘텐츠 유형별 레이아웃 패턴을 사전 정의:

```
예시:
- "비교 분석" → 좌우 2단 레이아웃 + 상단 제목
- "트렌드"    → 상단 제목 + 차트 + 하단 핵심 요약
- "프로세스"  → 좌→우 화살표 박스 3~5개
- "KPI 대시보드" → 상단 수치 카드 + 하단 차트/테이블
- "Before/After" → 좌우 대비 박스
```

→ 메타데이터 또는 시스템 프롬프트에 패턴 주입

### 3. 차트 사용 기준
- 수치 비교 3개 이상 → bar/column chart
- 시계열 데이터 → line chart
- 비율/구성 → pie chart
- 차트 선호 스타일 (색상, 범례 위치, 데이터 라벨 등)

### 4. 시각 요소 규칙
- **강조 박스**: 색상 배경 + 텍스트 (예: 핵심 메시지, 인사이트)
- **컬러 코딩**: 긍정=초록, 부정=빨강, 중립=회색, 강조=오렌지
- **아이콘/도형**: 체크마크, 화살표, 원형 번호 등 활용 규칙
- **구분선/여백**: 콘텐츠 영역 간 시각적 분리

### 5. 콘텐츠 밀도 기준
- 슬라이드당 텍스트 최대 줄 수
- "한 슬라이드 = 하나의 메시지" 원칙 여부
- 불릿 포인트 최대 개수
- 테이블 최대 행/열 수

---

## 기대 효과
- LLM이 콘텐츠 유형에 맞는 레이아웃을 자동 선택
- 텍스트+테이블 일변도 → 차트·강조 박스·다단 레이아웃 혼용
- 템플릿 디자인 톤에 맞는 일관된 시각적 품질
